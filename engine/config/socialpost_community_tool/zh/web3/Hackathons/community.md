# Web3 黑客松 - 社区类活动知识库

## 📌 模块激活条件

当活动主题涉及以下关键词时，应用本模块规则：
- 社区黑客松、生态 Hackathon、Builder Jam
- DAO 黑客松、社区共建赛、开源贡献赛
- 主题创意赛、Meme/NFT 创作马拉松

---

## 🎯 场景特征概述

| 特征维度 | 具体描述 |
|----------|----------|
| 形式 | 线上/线下开发 + 社区投票 + 展示交流 + 派对庆祝 |
| 时长 | 24-72 小时，氛围轻松 |
| 参与方 | 社区成员、Builder、创作者、生态项目 |
| 核心目标 | 社区活跃、生态建设、Builder 连接、文化传播 |

---

## 👥 受众画像（按优先级）

| 优先级 | 受众类型 | 关注点 |
|--------|----------|--------|
| P0 | 社区 Builder | 项目实践、社区认可、合作机会 |
| P1 | 生态开发者 | 生态贡献、官方激励、曝光机会 |
| P2 | 创意爱好者 | 创作表达、社区互动、奖励福利 |
| P3 | 社区新人 | 参与体验、融入社区、学习机会 |

---

## 🔀 风格选择指引

| 活动类型 | 推荐风格 | 核心侧重 |
|----------|----------|----------|
| 生态共建赛 | 热情活力型 (B) | 社区氛围、共建精神 |
| DAO 黑客松 | 热情活力型 (B) | 治理创新、社区参与 |
| 创意/Meme 赛 | 福利导向型 (D) | 趣味参与、奖励激励 |
| 开源贡献赛 | 悬念引导型 (C) | 技术贡献、生态价值 |

---

## 📝 内容必含元素

### 1. 社区氛围（至少2处）
- "和 [数量]+ Builder 一起 Build"
- "[社区名] 年度最大规模共建活动"

### 2. 参与激励（至少1处）
- "社区投票选出最受欢迎项目"
- "参与即可获得专属 POAP/NFT"

### 3. 生态价值（至少1处）
- "优秀项目纳入生态官方支持"
- "获得社区 Grant/Treasury 资助"

### 4. 社区精神（建议包含）
- "Let's BUIDL together!" / "WAGMI"

## ❓ 常见问答模板

| 问题 | 回答 |
|------|------|
| 非技术背景可以参加吗？ | 可以！创意、设计、运营都需要 |
| 必须用特定技术吗？ | 建议基于生态技术栈，但不强制 |
| 社区投票怎么参与？ | 持有社区 NFT/Token 即可投票 |
| POAP 怎么领取？ | 完成提交后自动发放到钱包 |

## ⚠️ 禁止事项

- ❌ 将社区活动过度商业化
- ❌ 承诺确定的 Token 空投
- ❌ 忽视社区投票的公平性
- ❌ 过度强调奖金而忽视共建精神
