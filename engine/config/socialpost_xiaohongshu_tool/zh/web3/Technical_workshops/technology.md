# Web3技术工作坊场景知识库 - 技术类

## 场景概述
技术类Web3工作坊面向有一定Solidity/Rust基础的开发者，聚焦特定技术方向的深度学习和生产级实践。这类活动强调技术深度、合约安全、Gas优化和链上架构设计，是进阶开发者提升专业技能、解决复杂链上问题的核心学习场景。

---

## 场景特征
- **规模**：通常15-40人，保证深度交流和Code Review
- **时长**：1-2天，深度学习需要时间
- **形式**：原理讲解 + 源码/合约分析 + 高级实操 + Code Review
- **参与者**：有经验的合约开发者、协议工程师、安全研究员、技术架构师
- **特点**：技术门槛高、内容深度大、代码质量要求高

## 课程设计建议

### 内容规划
- 确保内容有足够的技术深度（EVM级/源码级）
- 平衡原理讲解和合约实操练习
- 准备顶级协议的源码级分析（Uniswap/Aave/MakerDAO）
- 设计有挑战性的实操任务（Bug修复/Gas优化/安全审计）
- 留足讨论、Code Review和答疑时间

### 讲师要求
- 必须有Mainnet生产级合约开发经验
- 有协议审计或安全研究背景优先
- 能够解答深入的EVM和合约架构问题
- 准备好真实的踩坑经验和审计案例分享
- 能够进行高质量的Code Review
- 有耐心解答复杂的链上问题

### 学员筛选
- 设置明确的前置要求（Solidity/Rust基础+项目经验）
- 可要求提交GitHub和技术背景
- 控制学员水平差异，保证交流质量
- 鼓励带着具体的合约问题和代码来
- 提前了解学员的技术方向和诉求

### 实操设计
- 准备高质量的合约代码框架和Foundry测试套件
- 设计渐进式的练习任务（分析→修复→优化→审计）
- 预留Debug和Gas优化时间
- 准备应对各种链上异常情况
- 提供Code Review环节和互相审计机会
