# Web3 开发者聚会 活动策划知识库

## 场景概述

开发者聚会（Developer Meetup）是Web3行业中以社区交流、技术分享和人脉拓展为核心的社交型活动，氛围轻松、互动性强，是维护开发者社区活跃度、建立社区归属感的重要形式。

### 活动特征

- **规模**：30-150人
- **周期**：2-3小时
- **形式**：技术分享 + 社交交流 + 轻松互动
- **核心价值**：社区维护、人脉拓展、知识分享、归属感建立

## 线上Meetup补充指南

### 线上活动流程调整

| 环节 | 线下版本 | 线上版本 |
|------|----------|----------|
| 签到 | 现场签到+交流 | 提前5分钟进入+自我介绍环节 |
| 分享 | 现场演讲 | 屏幕共享（缩短至20-25分钟） |
| 社交 | 自由走动交流 | Breakout Room分组 |
| 互动 | 现场举手 | 弹幕/举手功能 |
| 抽奖 | 现场抽取 | 线上抽奖工具 |

### 线上活动时长调整

- 总时长控制在90分钟内
- 单个分享不超过25分钟
- 每30分钟设置互动环节
- 社交环节改为Breakout Room（15-20分钟）

### 线上活动平台选择

| 平台 | 特点 | 适用场景 |
|------|------|----------|
| Twitter Space | Web3原生、公开传播 | 开放式讨论、AMA |
| Discord Stage | 社群内、可互动 | 社区内部活动 |
| Zoom | 功能全面、分组支持 | 正式分享+社交 |
| Google Meet | 简单易用 | 小规模聚会 |

### 线上活动注意事项

- **互动设计**：增加投票、问答等互动环节
- **社交替代**：使用Breakout Room或会后Discord语音
- **录播分享**：提供录播供无法参与者回看
- **时区考虑**：选择多数人方便的时间

## 分享内容设计

### 分享类型

| 类型 | 时长 | 内容深度 | 适合 |
|------|------|----------|------|
| Lightning Talk | 5-10min | 概念/观点 | 快速分享、多人轮流 |
| 标准分享 | 20-30min | 技术/案例 | 有干货、可复用 |
| 深度分享 | 40-50min | 完整主题 | 需要时间铺垫的内容 |
| Live Coding | 30-45min | 实操演示 | 技术教程 |

### 常见分享主题

| 类别 | 主题示例 |
|------|----------|
| 技术分享 | 新技术介绍、开发经验、踩坑总结 |
| 项目分享 | 项目介绍、架构设计、成长历程 |
| 行业洞察 | 趋势分析、赛道解读、市场观察 |
| 经验分享 | Builder故事、创业心得、职业发展 |
| 工具推荐 | 开发工具、效率工具、资源推荐 |

### 分享者招募

| 来源 | 说明 |
|------|------|
| 社区成员 | 鼓励社区Builder分享 |
| 项目方 | 邀请项目做技术/产品分享 |
| 外部嘉宾 | 邀请行业专家 |
| 主办方团队 | 团队成员分享 |

## 嘉宾与主持配置

### 分享嘉宾画像

| 类型 | 画像 | 适合分享 |
|------|------|----------|
| Builder | 有实际项目经验 | 技术分享、踩坑经验 |

| DevRel | 开发者关系负责人 | 技术推广、工具介绍 |

### 主持人要求

| 维度 | 要求 |
|------|------|
| 社区熟悉度 | 了解社区、认识常客 |
| 表达能力 | 能调动氛围、控场 |
| 技术背景 | 能理解分享内容、引导讨论 |
| 亲和力 | 让参与者感到轻松 |

### 主持人职责

- 开场暖场、介绍流程
- 介绍嘉宾、引导分享
- 控制时间、管理Q&A
- 带动氛围、处理冷场
- 收尾总结、预告下期

## 社交环节设计

### 破冰活动

| 活动 | 形式 | 时长 |
|------|------|------|
| 快速自我介绍 | 每人30秒介绍 | 10-15min |
| 找相同点 | 两两交流找共同点 | 10min |
| 问题接龙 | 回答问题后指定下一人 | 10min |
| 技能交换 | 分享自己能教/想学的 | 15min |

### 主题桌讨论

设置不同主题的讨论桌，参与者自由选择：

| 桌号 | 主题示例 |
|------|----------|
| 1号桌 | DeFi开发 |
| 2号桌 | NFT/GameFi |
| 3号桌 | 公链生态 |
| 4号桌 | 新手入门 |
| 5号桌 | 自由交流 |

### 社交促进技巧

- **名牌设计**：包含姓名、角色、关注领域
- **标签贴纸**：技能/兴趣标签便于识别
- **桌签引导**：明确讨论主题
- **主持人引荐**：帮助对接相关的人

## 场地与氛围

### 场地选择

| 场地类型 | 特点 | 适用规模 |
|----------|------|----------|
| Co-working Space | 氛围好、设施全 | 30-80人 |
| 咖啡厅/酒吧 | 轻松、社交感强 | 20-50人 |
| 科技公司会议室 | 正式、设施好 | 30-100人 |
| 艺术空间 | 独特、有调性 | 50-150人 |

### 场地要求

| 要素 | 要求 |
|------|------|
| 布局 | 便于交流、不要太空旷 |
| 音响 | 分享能听清、不影响交流 |
| 灯光 | 明亮但不刺眼 |
| 网络 | 稳定WiFi |
| 位置 | 交通便利、好找 |

### 氛围营造

- **音乐**：轻松的背景音乐
- **灯光**：温暖、舒适
- **布置**：社区banner、打卡点
- **餐饮**：Pizza、饮料、小食
