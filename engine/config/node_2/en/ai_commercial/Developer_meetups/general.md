# AI Developer Meetup Planning Playbook

## Overview

AI Developer Meetups are community-driven events centered on networking, technical knowledge-sharing, and professional relationship-building. With a relaxed, interactive atmosphere, they serve as a powerful format for sustaining developer community engagement, driving product adoption, building developer loyalty, and gathering product feedback.

### Event Characteristics

- **Scale**: 30–150 attendees
- **Duration**: 2–3 hours
- **Format**: Technical talks + Networking + Demo showcases
- **Core Value**: Community building, technical evangelism, networking, product feedback collection

---

## Event Types & Selection

### Meetup Format Comparison

| Type | Characteristics | Best Use Case |
|------|-----------------|---------------|
| Technical Talk | Presentation-led, content-dense | Technical evangelism, best practices |
| Product Update | New feature launches, live demos | Product releases, feature rollouts |
| Networking-Led | Conversation-first, light content | Community maintenance, relationship building |
| Themed Discussion | Panel or AMA format | Hot-topic exploration, deep-dive conversations |
| Hybrid | Balanced talks + networking | Regular community events |

### Recommended Frequency by Community Stage

| Community Stage | Recommended Cadence | Notes |
|-----------------|---------------------|-------|
| Early-stage | Monthly | Establish rhythm, build habits |
| Growth-stage | Bi-weekly | Maintain momentum, expand reach |
| Mature | Weekly | Sustained operations, brand reinforcement |

## Virtual Meetup Supplement

### Virtual Format Adaptations

| Segment | In-Person Version | Virtual Version |
|---------|-------------------|-----------------|
| Check-in | On-site registration + networking | Join 5 min early + self-introduction round |
| Presentations | Live stage delivery | Screen share (shortened to 20–25 min) |
| Demo | On-site live demo | Screen share with real-time interaction |
| Networking | Open mingling | Breakout Rooms by topic |
| Audience engagement | Raise hands | Live chat / raise hand feature |
| Raffle | In-person draw | Online raffle tool |

### Virtual Event Duration Guidelines

- Total runtime: keep within 90 minutes
- Single presentation: no longer than 25 minutes
- Set an interactive touchpoint every 30 minutes
- Replace networking with Breakout Rooms (15–20 minutes)

### Platform Selection

| Platform | Strengths | Best Use Case |
|----------|-----------|---------------|
| Twitter Spaces | High discoverability, interactive | Open discussions, AMAs |
| Discord Stage | Community-controlled, contained | Internal community events |
| Zoom | Full-featured, breakout support | Formal talks + networking |
| YouTube Live | Large-scale broadcast | Product launches, wide-reach streaming |

### Virtual Engagement Design

| Segment | Virtual Design |
|---------|----------------|
| Icebreaker | Live chat self-introductions, poll-based engagement |
| Presentation | Embedded Q&A, real-time audience feedback |
| Demo | Invite participants to test remotely |
| Networking | Topic-specific Breakout Rooms, post-event Discord |
| Raffle | Chat-based raffle, engagement-linked prizes |

### Virtual Event Best Practices

- **Interaction design**: Integrate polls, Q&A, and engagement prompts throughout
- **Pace**: Online attention spans are shorter — keep transitions tight
- **Recording**: Make recordings available for those who couldn't attend live
- **Community continuity**: Direct attendees to Discord/Slack for ongoing engagement

## Talk Content Design

### Talk Formats

| Format | Duration | Depth | Best For |
|--------|----------|-------|----------|
| Lightning Talk | 5–10 min | Single insight or tip | Quick sharing, multi-speaker rotation |
| Standard Talk | 20–30 min | Technical or case-based | Substantive, reusable content |
| Deep-Dive Talk | 40–50 min | Comprehensive topic | Topics requiring context-building |
| Live Coding | 30–45 min | Hands-on demonstration | Technical tutorials |

### Common AI Meetup Talk Topics

| Category | Example Topics |
|----------|---------------|
| Technical Talks | Prompt engineering techniques, RAG implementation, Agent development |
| Application Case Studies | "How I built X with AI", efficiency gains at scale |
| Product Updates | New feature walkthroughs, API changelog highlights |
| Best Practices | Cost optimization, latency reduction, security practices |
| Industry Insights | AI trends, technology direction |
| Tool Recommendations | Developer tooling, productivity tools |

### Speaker Sourcing

| Source | Notes |
|--------|-------|
| Community developers | Encourage members to share projects and lessons learned |
| Internal team | DevRel, engineers sharing updates |
| External guests | Invite industry experts or thought leaders |
| Customer advocates | Invite customers to share real-world implementation stories |

## Speaker & MC Configuration

### Speaker Profiles

| Type | Profile | Best For |
|------|---------|----------|
| Community developer | Hands-on project experience | Application case studies, lessons learned |
| Developer Relations | Developer advocacy expertise | Product overviews, best practices |
| Technical expert | Senior engineer or researcher | Technical depth, cutting-edge insights |
| Customer representative | Enterprise AI lead | Implementation case studies, real-world experience |

### MC Requirements

| Dimension | Requirement |
|-----------|-------------|
| Community familiarity | Knows the community, recognizes regulars |
| Technical background | Understands talk content, can guide discussion |
| Communication skills | Can energize the room, maintain control of flow |
| Approachability | Makes attendees feel at ease |

### MC Responsibilities

- Open and warm up the room; walk through the agenda
- Introduce speakers and facilitate presentations
- Manage timing and moderate Q&A
- Drive energy and handle lulls
- Close out with summary and preview of next event

## Networking Segment Design

### Icebreaker Activities

| Activity | Format | Duration |
|----------|--------|----------|
| Speed introductions | 30 seconds each: name + current AI project | 10–15 min |
| AI tool share-out | Share the most useful AI tool discovered recently | 10 min |
| Question chain | Answer a question, then nominate the next person | 10 min |
| Skill exchange | Share what you can teach and what you want to learn in AI | 15 min |

### Themed Discussion Tables

Set up distinct tables by topic; attendees self-select based on interest:

| Table | Topic |
|-------|-------|
| Table 1 | Prompt Engineering |
| Table 2 | RAG & Knowledge Bases |
| Table 3 | Agent Development |
| Table 4 | AI Product Design |
| Table 5 | Getting Started with AI |

### Networking Facilitation Tips

- **Name badges**: Include name and current AI project
- **Interest stickers**: Skill/interest labels for easy identification
- **Table cards**: Clear topic labels for discussion tables
- **MC-led introductions**: Proactively connect attendees with relevant backgrounds

## Venue & Atmosphere

### Venue Selection

| Venue Type | Characteristics | Ideal Capacity |
|------------|-----------------|----------------|
| Company office | Brand presence, full facilities | 30–80 people |
| Co-working space | Good vibe, neutral environment | 30–100 people |
| Café or bar | Casual, socially inviting | 20–50 people |
| Tech campus conference room | Professional, well-equipped | 50–150 people |

### Venue Requirements

| Factor | Requirement |
|--------|-------------|
| Layout | Conversation-friendly; avoid overly open or sparse arrangements |
| Audio | Clear enough for presentations without disrupting side conversations |
| Wi-Fi | Stable, reliable network (required for live demos) |
| Location | Convenient access via public transit |
| Atmosphere | Tech-forward and relaxed feel |

### Atmosphere Creation

- **Music**: Light background music during networking
- **Lighting**: Warm and comfortable
- **Décor**: Brand elements, tech posters, photo-worthy backdrop
- **Refreshments**: Pizza, drinks, light snacks

## Promotion Strategy

| Phase | Timing | Content |
|-------|--------|---------|
| Teaser | 2 weeks out | Date, location, topic preview |
| Speaker announcement | 1 week out | Speaker bios and talk highlights |

## Contingency Planning

| Risk | Description | Mitigation |
|------|-------------|------------|
| Speaker cancellation | Scheduled speaker drops out last minute | Backup speaker on standby; extend networking time |
| Low attendance | Fewer registrants than expected | Ramp up promotion; reset expectations |
| Over-capacity | More attendees than anticipated | Flexible venue plan; manage overflow |
| Demo failure | Live demo breaks during presentation | Pre-recorded backup video; backup network connection |
| Dead air | Networking segment fails to ignite | Icebreaker activities; MC-facilitated warm-ups |
| Equipment failure | Projector or audio system fails | Backup equipment; pre-event equipment check |

- Use registration platforms such as Luma or Lu.ma equivalents to manage sign-ups
- Prepare QR code check-in or printed attendee list
- Set up an express check-in lane
- Record actual attendance data for post-event reporting

### Post-Event Retrospective

- Attendance rate tracking
- Attendee feedback collection
- Talk content effectiveness review
- Networking segment observation and notes
- Improvement items log
- API registration and usage conversion tracking (where applicable)
