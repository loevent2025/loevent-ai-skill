# Web3 Cafe Cursor - 技术类活动知识库

## 🎯 场景特征概述

| 特征维度 | 具体描述 |
|----------|----------|
| 形式 | 咖啡厅环境 + 自带电脑 Coworking + 技术闲聊 + 轻量 Demo |
| 时长 | 3-6 小时，分时段可选，随时来走 |
| 参与方 | 协议开发者、智能合约工程师、技术爱好者、项目方技术团队 |
| 核心目标 | 技术共创、开发者连接、项目交流、工具体验 |

---

## 🎨 语气与风格覆写

当检测到 Cafe Cursor 技术活动时：

✅ 轻松、Geek 感、咖啡厅氛围
✅ "一边喝咖啡一边写代码"的 vibe
✅ 技术交流是自然聊天，不是正式分享
✅ Demo 是"你看我写了个东西"，不是路演
✅ 时间灵活，来去自由

❌ 禁止写成正式技术会议或研讨会
❌ 不要有严格的议程安排
❌ 避免"技术干货"、"深度学习"等功利词汇
❌ 不制造技术门槛，让新手也能来

**核心原则**：
🎯 这是一个"带上电脑，来咖啡厅一起写代码"的技术社交空间，咖啡是媒介，代码是话题，连接自然发生

## 📝 内容必含元素

### 1. 咖啡厅氛围开场（必须包含）

以"咖啡 + 代码"的 vibe 作为开场：

**公式**：
```
[咖啡厅环境] + [带电脑来] + [一起 coding 的感觉]
```

**要点**：
- ❌ 不要写："我们将举办技术交流会"
- ✅ 要写："带上你的电脑，来咖啡厅一起写代码"
- 让人感受到这是一个轻松的 coworking 空间

---

### 2. 五感沉浸（至少包含3项）

**视觉**：
- 空间："阳光洒进来的咖啡厅"、"长桌上摆满了 MacBook"、"有人在白板上画架构图"
- 技术元素："屏幕上跑着代码"、"有人在调试合约"、"终端窗口闪烁"

**味觉**：
- 具体描述："手冲咖啡"、"拿铁"、"美式续杯"
- ❌ 避免："提供饮品"
- ✅ 要写："咖啡管够，写代码的燃料我们包了"

**听觉**：
- 音乐："lo-fi beats"、"轻声的背景音乐"
- 环境音："键盘声此起彼伏"、"偶尔有人说'这个 bug 我搞定了'"

**触觉**：
- 温度："咖啡厅的空调刚刚好"
- 连接感："和旁边的人聊起来发现在用同一个框架"

**氛围**：
- 能量等级："专注但不压抑"、"chill coding vibe"、"有人在肝代码有人在闲聊"
- 社交动态："有人戴着耳机沉浸 coding，有人在讨论技术方案"

---

### 3. 时间灵活框架（必须包含）

Cafe Cursor 的核心是时间灵活：

**格式**：
```
⏰ 时段选择（选一个适合你的时间）：
- [时段A] 上午场 - 适合早起 coding
- [时段B] 下午场 - 适合午后专注
- [时段C] 全天 - 想待多久待多久
- 随时来坐一会儿喝杯咖啡也行
```

**要点**：
- 强调"来去自由"
- 不设强制时长
- 可以只来喝杯咖啡打个招呼

---

### 4. 技术交流自然发生（推荐包含）

技术讨论是聊天，不是分享会：

**要点**：
- ❌ "技术分享环节"
- ✅ "聊着聊着就聊到最近在用什么框架了"

- ❌ "深度技术交流"
- ✅ "有人在调 bug，旁边的人凑过来说'我遇到过这个问题'"

---

### 5. 轻量 Demo 氛围（推荐包含）

Demo 是"给你看个东西"，不是路演：

**要点**：
- ✅ "想展示最近写的东西？随时可以，围一圈人看看"
- ✅ "不是正式 Demo，就是'你看我这个酷不酷'"
- ✅ "有人在展示刚写的合约，有人在旁边提建议"
- ❌ "Demo 展示环节"、"项目路演"

---

### 6. 带电脑来（必须包含）

这是 Cafe Cursor 的核心元素：

**要点**：
- ✅ "带上你的电脑"
- ✅ "有 coding 计划最好，没有也行，来 social 也很好"
- ✅ "可以继续你手头的项目，也可以开个新坑"

---

### 7. 新手友好氛围（必须包含）

让技术新手也能来：

**要点**：
- ✅ "不管你是写了十年代码还是刚开始学，都欢迎"
- ✅ "有问题可以问旁边的人，大家都很友好"
- ✅ "来感受一下 coding 的氛围也很好"
- ❌ 不设技术门槛

---

### 8. 合规提示（如涉及项目展示）

```
"⚠️ 技术交流活动，项目展示不构成投资建议（NFA）"
```
