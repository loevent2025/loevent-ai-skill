# Web3 Demo Day — Technical Events Knowledge Base

## 📌 Module Activation Criteria

Apply this module when the event theme involves the following keywords:
- Technical project showcase day, Protocol Demo Day, developer achievement exhibition
- Hackathon results showcase, technical prototype presentation, product launch event
- Grant project milestone report, technical milestone showcase

---

## 🎯 Scenario Overview

| Dimension | Description |
|-----------|-------------|
| Format | Project showcase + Technical review + Live demos + Q&A interaction |
| Duration | Half-day to full day, 5–15 minutes per project |
| Participants | Project teams, technical reviewers, protocol teams, developer audience |
| Core Objective | Technical validation, achievement showcase, feedback collection, partnership facilitation |

---

## 👥 Audience Profiles (by Priority)

| Priority | Audience Type | Key Focus |
|----------|---------------|-----------|
| P0 | Protocol / Project Teams | Ecosystem project quality, partnership opportunities, grant evaluation |
| P1 | Technical Reviewers | Technical feasibility, innovation, completeness |
| P2 | Developer Audience | Learning and inspiration, tech trends, collaboration opportunities |
| P3 | Tech Media | Editorial material, industry developments, project discovery |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Focus |
|------------|-------------------|------------|
| Official Protocol Demo Day | Formal & Professional (A) | Technical authority, ecosystem value |
| Hackathon Results Showcase | Energetic & Engaging (B) | Celebrating achievements, community spirit |
| Grant Project Milestone Report | Formal & Professional (A) | Milestone review, continued support |
| Technical Prototype Debut | Suspense-Driven (C) | Technical breakthrough, innovation highlights |

## 📝 Required Content Elements

### 1. Showcase Value (at least 2 instances)
- "[Number] technical projects showcased in one event"
- "First public demo of [technology/product]"

### 2. Review Panel (at least 1 instance)

### 3. Interaction Opportunities (at least 1 instance)
- "Hands-on experience with project demos on-site"

## ⚠️ Prohibited Practices

- ❌ Framing project demos as investment recommendations
- ❌ Overstating the technical maturity of projects
- ❌ Disclosing unreleased project information
- ❌ Making promises about future project development
