# AI DEMO DAY × TECHNICAL / AI项目展示日 × 技术类

## 适用场景
AI技术项目展示、开源项目Demo Day、AI工具展示日、Hackathon成果展示、技术产品发布

## 场景特征
- **形式**：技术路演 + Live Demo + 技术Q&A + 开发者交流
- **核心价值**：技术展示、方案评估、开发者采用、技术认可

## 受众定位
- **主要受众**：AI开发者 - 了解新工具、学习技术方案
- **次要受众**：技术负责人/CTO - 评估技术、寻找解决方案
- **第三受众**：技术型投资人 - 评估技术可行性
- **扩展受众**：开源社区 - 发现项目、参与贡献
- **受众行为**：评估技术、学习方案、寻找工具、技术交流

### 常用受众组合
- **开发者导向**：开发者 + 开源社区，适合工具展示型
- **评估导向**：技术负责人 + 投资人，适合技术深度展示

## 内容元素
- 技术Demo Day主题定位
- 展示项目技术方向
- 技术评审阵容
- Live Demo形式（代码、架构）
- 技术Q&A环节
- 开源/文档资源

## 语言处理
- 技术导向，但小红书风格表达
- 强调「能跑的代码」「Live Demo」
- 技术细节是卖点
- 开发者社群氛围
- 不炫技，讲实用

## 结构策略
- 开场信息块占比 15%
- 技术项目块占比 35%：项目方向 + 技术亮点
- 评审阵容块占比 20%：技术背景
- 参与价值块占比 20%：能学到什么
- 行动号召块占比 10%
- Emoji 密度控制：总量 8-12 个，技术感

## 情绪氛围
技术硬核、Demo说话。"Show me the code"的Builder氛围。

## 风格变体

### 变体A：开发工具型 / Dev Tools Showcase
- **适用**：AI开发工具、SDK、框架展示
- **开场格式**：🔧 {Demo Day名} | AI开发工具集中展示
- **语气**：实用、开发者导向

### 变体B：开源项目型 / Open Source Demo
- **适用**：开源AI项目展示
- **开场格式**：🌟 {Demo Day名} | 开源AI项目集中亮相
- **语气**：开源精神、社区导向

### 变体C：Hackathon成果型 / Hackathon Showcase
- **适用**：黑客松获奖项目展示
- **开场格式**：🏆 {Hackathon名} | 获奖项目技术展示
- **语气**：创新、快速验证

## 注意事项

### 技术类Demo Day要点
1. **Live Demo是核心**：必须现场跑代码
2. **技术细节要有**：架构、性能、代码
3. **评审要懂技术**：能问到点上
4. **开发者视角**：解决什么问题
5. **资源要给到**：GitHub、文档、社群
