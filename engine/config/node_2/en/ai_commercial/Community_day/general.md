# AI Brand Pop-up Café — Event Planning Knowledge Base

## 1. Overview

An AI brand pop-up café (Pop-up Café) is a new type of brand experience event that blends coffee culture with developer communities. Attendees write code, network, and explore products in a relaxed café setting. It is an innovative format for building brand awareness, cultivating user loyalty, and fostering community belonging.

### Event Characteristics

- **Scale**: 50–300 attendees
- **Duration**: Half-day to full-day (single event) / Multi-city tour
- **Format**: Complimentary coffee + co-working + open networking + branded giveaways
- **Core value**: Brand experience, community belonging, user loyalty, social media reach, real-world connection

## 4. Venue Selection & Setup

### Venue Type Comparison

| Venue Type | Advantages | Disadvantages | Recommended Scale |
|------------|------------|---------------|-------------------|
| Independent café buyout | Great atmosphere, full setup included | Limited capacity, coordination required | 50–100 |
| Co-working space | Full amenities, larger capacity | Requires atmosphere investment | 100–200 |
| Branded pop-up space | Professional, fully customizable | Higher cost | 100–300 |
| University café | Student convenience, low cost | Basic facilities | 30–80 |
| Tech campus common area | Highly targeted audience | Requires approvals | 50–150 |

### Venue Requirements

| Factor | Requirements |
|--------|-------------|
| Seating | Ample power outlets, suitable for laptop use |
| Network | Stable, high-speed Wi-Fi (minimum 100 Mbps) |
| Power | Multiple sockets and power strips to support many devices |
| Atmosphere | Warm, tech-forward feel, comfortable for extended stays |
| Location | Accessible by transit, in developer-dense areas |
| Capacity | 1.2× expected attendance |

### Space Layout Recommendations

| Zone | Function | Share |
|------|----------|-------|
| Co-working area | Core coding/work zone with ample outlets | 50% |
| Social lounge | Sofas and standing networking areas | 20% |
| Coffee bar | Drinks service and queuing area | 15% |
| Branded merchandise display | Giveaway showcase and photo ops | 10% |
| Check-in area | Entry check-in and materials distribution | 5% |

### Atmosphere & Ambiance

| Element | Recommendations |
|---------|----------------|
| Visual | Brand color palette, logo displays, tech-inspired décor |
| Lighting | Warm and comfortable, screen-friendly brightness |
| Music | Lo-fi / ambient music at low background volume |
| Scent | Natural coffee aroma (ambient) |
| Photo ops | Branded backdrop wall, signature installations |

## 7. Interactive Session Design (Optional)

### Light-Touch Engagement

A pop-up café centers on a relaxed, open atmosphere. Interactive sessions should be **lightweight, optional, and non-intrusive**.

| Session | Format | Duration | Notes |
|---------|--------|----------|-------|
| Product mini-class | Informal share | 15–30 min | New features or tips; attendance optional |
| Project showcase | Volunteer-led | 20–30 min | Community members share work in progress |
| Team AMA | Open Q&A | 20–30 min | Team members on-site and available for questions |
| Quick demo | Live demonstration | 10–15 min | Showcase key product features |

### Social Connection Facilitators

| Tool | Description |
|------|-------------|
| Name badge design | Name + "What I'm building with the product" |
| Project tag sticker | "Currently building: [Project Name]" |
| Skills exchange board | "I can teach / I want to learn" |
| Team member identifier | Clearly marked so attendees can find and approach team members |

### Build Challenge (Advanced Format)

| Element | Design |
|---------|--------|
| Format | On-site time-limited build challenge |
| Duration | 1–2 hours |
| Theme | Complete a designated task using the product |
| Prizes | API credits, limited-edition merchandise |
| Showcase | Best projects demonstrated live |

## 8. Staffing

### Core Team Roles

| Role | Headcount | Responsibilities |
|------|-----------|-----------------|
| Event lead | 1 | Overall coordination, contingency management |
| Community operations | 1–2 | Check-in, merchandise distribution, atmosphere management |
| Product / technical representatives | 2–3 | Engaging with attendees, answering questions, gathering feedback |
| Coffee service coordinator | 1 | Liaising with café, managing restocking |
| Photography / content | 1 | On-site photography, social media posting |

### Staff-to-Attendee Ratios

| Scale | Team Size |
|-------|-----------|
| Under 50 attendees | 3–4 staff |
| 50–100 attendees | 5–6 staff |
| 100–200 attendees | 7–10 staff |
| 200+ attendees | 10–15 staff |

### Team Conduct Guidelines

- **Proactive but unobtrusive**: Engage with attendees, but respect those who are focused on their work
- **Authentic**: Converse as a fellow user or developer, not as a salesperson
- **Gather feedback**: Stay alert to user pain points and suggestions during natural conversations
- **Make connections**: Actively facilitate introductions between attendees

## 11. Registration & Check-in

### Registration Management

| Factor | Recommendations |
|--------|----------------|
| Platform | Luma, Eventbrite, Huodongxing |
| Capacity | 80% of venue capacity (reserve space for walk-ins) |
| Data collected | Name, contact information, product usage |
| Reminders | Send reminders 3 days and 1 day before the event |

### Check-in Design

| Method | Description |
|--------|-------------|
| QR code check-in | Luma QR code for fast, efficient entry |
| Name badge distribution | Name badges and basic giveaways distributed at check-in |
| Walk-in lane | Reserved quota for on-site registration |

### Attendance Rate Optimization

| Strategy | Description |
|----------|-------------|
| Merchandise preview | Emphasize "check-in gifts" available to all attendees |
| Scarcity messaging | Highlight "limited merchandise, first come, first served" |
| Multiple reminders | Send multiple reminders in the days leading up to the event |
| Waitlist | Activate a waitlist when registration exceeds capacity |

## 12. Risk Management

| Risk | Description | Contingency Plan |
|------|-------------|-----------------|
| Higher-than-expected turnout | Attendance exceeds venue capacity | Build in flexibility, stagger entry by time slots |
| Lower-than-expected turnout | Sparse attendance, low energy | Adjust layout, team members actively foster atmosphere |
| Network outage | Unstable Wi-Fi | Backup mobile hotspots, pre-test all connections |
| Coffee supply shortage | Drink consumption exceeds forecast | Backup supplier, nearby purchase option |
| Merchandise shortage | Giveaways run out early | Stage distribution, manage pacing |
| Weather disruption | Adverse weather reduces attendance | Advance notice, backup date on standby |
| Venue issue | Last-minute venue change required | Pre-identified backup venue |
