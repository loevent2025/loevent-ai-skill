## 🔥 Cross-Stack Core Elements (unique to this module)

### Building Tech Stack Collisions
**Narrative formula:**
> "When a [Stack A developer] meets a [Stack B developer], what kind of code do they write?"
> "When [AI capabilities] merge with [X technology], what new possibilities emerge?"

**Required elements:**
- ✅ Clearly identify what each tech stack brings and how they complement each other
- ✅ Use collision-style phrasing: "A excels at…, B specializes in…, together they…"
- ✅ Emphasize the innovative potential of cross-stack combinations

### Cross-Stack Action Verbs
Use dynamic verbs when describing cross-stack collaboration:
- ❌ "Joint development," "co-participation," "coding together"
- ✅ "Fuse," "collide," "bridge," "chain together," "remix," "cross-stack connect"

### Complementary Tech Value
Answer the core questions:
- "Why can't a single stack build this, but a cross-stack team can?"
- "How do different technical perspectives inspire each other?"

## 🎭 Cross-Stack Atmosphere Building Guide

### 1. Capturing Tech Collision Moments
```
"The AI developer staring at the game dev's code: 'So that's how game AI is written'"
"The game developer trying to call an LLM: 'This NPC can actually hold a real conversation!'"
"2 AM — both sides still debating which approach makes more sense"
"During the demo, a player has a genuine conversation with an NPC for the first time"
```

### 2. Describing Stack Complementarity
```
❌ AI developer building a game solo: The model is powerful, but the game isn't fun
❌ Game developer adding AI alone: The game is fun, but the AI is dumb
✅ Cross-stack team: AI intelligence + game experience = a truly fun, intelligent game
```

### 3. Cross-Stack Emotional Vocabulary
- Technical interoperability: "So you can call it like that"
- Perspective exchange: "That's how you think about this problem"
- Fusion excitement: "This result couldn't be built by either side alone"

## ⚠️ Prohibited Actions

- ❌ Creating tech stack hierarchies or elitism
- ❌ Letting one tech stack dominate while ignoring the other's contributions
- ❌ Over-weighting judging criteria toward a single technical direction
- ❌ Overstating the technical outcomes of cross-stack fusion
- ❌ Disclosing technical implementation details of cross-stack projects without authorization
