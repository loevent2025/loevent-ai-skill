# Web3 Large-Scale Conference / Expo — Technical Events Knowledge Base

## 📌 Module Activation Criteria

Apply this module when the event theme involves the following keywords:
- Blockchain technology summit, developer conference, protocol technical launch
- Infrastructure showcase, Layer 1 / Layer 2 technology expo
- ZK technology summit, modular architecture conference, consensus mechanism workshop

---

## 🎯 Scenario Overview

| Dimension | Description |
|-----------|-------------|
| Format | Main stage + Technical breakout sessions + Demo exhibition area + Workshops |
| Content Depth | Core infrastructure, source-code-level analysis, technical roadmap reveals |
| Participants | Leading protocol core teams, prominent developers, technical researchers |

---

## 👥 Audience Profiles (by Priority)

| Priority | Audience Type | Key Focus |
|----------|---------------|-----------|
| P0 | Protocol Core Developers | Core infrastructure, consensus mechanisms, performance optimization |
| P1 | Infrastructure Builders | SDKs, development frameworks, DevOps |
| P2 | Technical Architects | System design, technology selection, security auditing |
| P3 | Security Researchers | Vulnerability analysis, security models, formal verification |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Focus |
|------------|-------------------|------------|
| Protocol Technical Launch | Formal & Professional (A) | Technical depth, authoritative release |
| Developer Technical Conference | Formal & Professional (A) | Technical talks, hands-on experience |
| Technical Workshop | Suspense-Driven (C) | Technical challenges, solution discovery |
| Hackathon-Adjacent Tech Expo | Value-Driven (D) | Prizes, resources, incubation |

---

## 📝 Required Content Elements

### 1. Technical Authority (at least 2 instances)
- "Bringing together [number]+ core contributors from leading global protocols"
- "Deep dive into the architectural evolution of [protocol/technology]"
- "First public reveal of the [protocol/project] technical roadmap"

### 2. Frontier Topics (at least 2 instances)
- "Focused on the latest breakthroughs in [ZK / modular / cross-chain] technology"
- "Exploring the future of [scalability / privacy / interoperability]"

### 3. Hands-On Value (at least 1 instance)
- "Live demos of the latest developer tools"
- "Access first-hand technical documentation and code samples"

### 4. Compliance Disclaimer (mandatory)
- "⚠️ This event is for technical exchange purposes only and does not constitute investment advice (NFA)"

## ⚠️ Prohibited Practices

- ❌ Promising investment returns or yield expectations
- ❌ Using hype language such as "get rich quick" or "alpha leak"
- ❌ Fabricating protocol names or technical details
- ❌ Exaggerating technical performance or making misleading claims
