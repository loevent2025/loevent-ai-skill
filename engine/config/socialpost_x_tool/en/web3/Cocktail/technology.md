# Cocktail Parties / Social Events × Technical

## Applicable Scenarios
Developer Social Night, Hackathon After Party, Protocol Developer Mixer, Tech-Only Cocktail Night, Builder Night

## Scenario Characteristics
- **Format**: Casual socializing for developers — no code talk required — a chance for tech people to unwind
- **Core Value**: Connecting with fellow builders, meeting peers in a different setting, relaxation

## Audience Positioning
- **Primary Audience**: Developers — writing code all day, drinking tonight
- **Secondary Audience**: Technical Founders — founders with engineering backgrounds
- **Extended Audience**: Tech Enthusiasts — people with a general interest in technology
- **Audience Behavior**: Unwind, meet peers, might talk tech — might not

## Content Elements
- Venue atmosphere
- A relaxed "no code talk required" vibe
- Tech-people humor
- Sensory experience
- Zero-pressure participation

## Language Treatment
- ✅ Tech-people humor and self-deprecation
- ✅ "Done coding for the day — time for a drink"
- ✅ Casual, peer-to-peer energy
- ❌ No technical talks
- ❌ Avoid "learning," "technical exchange"

## Structure Strategy
- Scene-setting opening block: 25%
- "This is not a tech conference" block: 20%
- Atmosphere & experience block: 25%
- Crowd description block: 15%
- Call-to-action block: 15%
- Emoji density control: 6–8 total

## Emotional Tone
The ease of being among your own kind. Peer energy. Relaxation. The unspoken understanding of "we all write code — no explanation needed."

## Style Variants

### Variant A: Developer Night
- **Use When**: Developer-only social events
- **Opening Format**: 🍺 Hey devs — come grab a drink
- **Tone**: Peer-to-peer, casual, self-deprecating

### Variant B: Hackathon After Party
- **Use When**: Post-hackathon celebration
- **Opening Format**: 🎉 48 hours are up — time to drink
- **Tone**: Celebratory, cathartic, high-energy

### Variant C: Protocol Developer Mixer
- **Use When**: Socializing for developers within a specific protocol/ecosystem
- **Opening Format**: 🔧 [Protocol Name] Developer Night | If you've shipped code on this protocol, come say hi
- **Tone**: Insider, peer-to-peer, unspoken understanding

---

## Atmosphere Keyword Library
Devs, peers, debug, unwind, grab a drink, not a tech talk, no code talk required, IYKYK

## Prohibited Vocabulary
❌ Technical sharing, learning, technical exchange, deep insights, level up, PPT

---

## Key Considerations

### Technical Social Event Best Practices
1. **Make it clear: "This is not a tech talk"** — Differentiate from technical conferences
2. **Tech-people humor**: Self-deprecation, insider jokes
3. **The unspoken dev bond**: No need to explain what late nights feel like
4. **Relaxation is the point**: Debugging can wait until tomorrow
5. **Hackathon After Parties hit different**: The sense of release matters
