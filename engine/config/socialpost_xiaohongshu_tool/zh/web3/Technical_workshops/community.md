# Web3技术工作坊场景知识库 - 社区类

## 场景概述
社区类Web3技术工作坊由开发者社区、DAO组织或技术公会发起，强调共同学习、链上实操和社群归属感。这类活动通常免费或由生态基金赞助，氛围轻松友好，是社区成员互相学习智能合约开发、分享链上踩坑经验和建立Builder连接的重要方式，也是Web3技术社区活力的核心体现。

---

## 场景特征
- **规模**：通常10-50人，便于交流互动和互助Debug
- **时长**：2-4小时，灵活安排
- **形式**：社区分享 + 动手实操 + 自由交流 + 互助学习
- **参与者**：社区Builder、技术爱好者、独立开发者、学习者
- **特点**：氛围友好、低门槛、互助性强、社交属性明显

## 社区运营建议

### 活动筹备
- 确定志愿讲师和引导者（社区OG或Grant获得者）
- 准备适合共同学习的链上实操内容
- 选择便于交流和写代码的场地（有WiFi和电源）
- 准备茶歇和小食
- 提前建Telegram/Discord群预热

### 氛围营造
- 强调"一起Build"而非"上课"
- 鼓励提问和互助Debug
- 志愿者热情欢迎新人
- 安排破冰和自我介绍环节
- 保持轻松友好的Builder氛围

### 互助机制
- 鼓励有经验的Builder帮助新手
- 建立合约Code Review互助机制
- 设置学习小组和Pair Programming
- 表扬和感谢帮助他人的成员（POAP/Shoutout）
- 营造WAGMI的互相支持文化

### 持续运营
- 保持固定的活动频率（每周/双周）
- 维护活跃的Telegram/Discord学习群
- 整理和分享学习资源、代码模板
- 庆祝成员的Build成果（部署了合约/拿了Hackathon奖）
- 鼓励成员组织分享和Demo
