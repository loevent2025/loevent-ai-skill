# Demo Day × 技术类（AI/Model 商业化版）

## 适用场景
AI 技术 Demo Day、LLM 架构展示、Agent 技术方案演示、AI Infra 工具发布、开源项目 Demo、技术方案评审会

## 场景特征
- **形式**：技术 Demo、架构讲解、代码展示、技术 Q&A
- **核心价值**：技术展示、架构分享、技术反馈、技术认可

## 受众定位
- **主要受众**：AI 工程师——想看技术实现的开发者
- **次要受众**：技术负责人——评估技术方案的决策者
- **第三受众**：开源贡献者——想了解项目技术的贡献者
- **延伸受众**：技术型投资人——关注技术创新的投资人
- **受众行为**：学习技术实现、评估技术方案、获取技术灵感、建立技术人脉

### 常用受众组合
- **工程师核心**：开发者为主，适合深度技术 Demo
- **技术决策**：技术负责人 + 工程师，适合方案评审型
- **开源社区**：贡献者 + 用户，适合开源项目 Demo

## 内容元素
- 技术主题定位
- Demo 项目技术方向
- 技术深度说明
- 代码/架构展示要求
- 技术评审阵容
- 技术 Q&A
- 技术资料分享

## 语言处理
- 技术术语精准
- 架构和实现导向
- 代码优先
- 性能数据说话
- 避免营销语言

## 结构策略
- 开场信息块占比 15%
- 技术展示块占比 40%
- 评审反馈块占比 20%
- 参与说明块占比 15%
- 行动号召块占比 10%
- Emoji 密度控制：总量 8-10 个

## 情绪氛围
技术深度、代码说话。"Show me the code"的 Builder 文化。

- **语气**：深度、架构导向

## 注意事项

### 技术类 Demo Day 要点
1. **技术深度是核心**：不是产品演示，是技术分享
2. **代码/架构要展示**：Show me the code
3. **评审要专业**：能给出技术层面的反馈
4. **性能数据要有**：不是空谈，要有数据
5. **技术资料分享**：Demo 后能获取技术文档
