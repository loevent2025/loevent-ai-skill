# Technical Workshops × General

## Applicable Scenarios
Developer skill training, hands-on technical courses, tooling tutorials, protocol integration guidance, live coding sessions, in-depth technical topic deep dives

## Scenario Characteristics
- **Format**: Instruction + hands-on practice, live coding, instructor guidance, interactive Q&A
- **Core Value**: Skill acquisition, hands-on application, problem solving, real-time feedback

## Audience Segmentation
- **Primary Audience:** Developers — engineers who want to learn a specific tech stack or tooling
- **Secondary Audience:** Technical product managers — product professionals who want to understand implementation
- **Extended Audience:** Tech enthusiasts — learners with strong interest in a specific technology
- **Audience Behavior:** Learn new skills, solve real-world problems, pick up best practices, build technical connections

### Common Audience Combinations
- **Developer-only:** Best for deep technical workshops
- **Mixed background:** Best for introductory or conceptual workshops

## Content Elements
- Workshop theme and learning objectives
- Prerequisite knowledge requirements
- Curriculum outline / agenda
- Instructor background
- Hands-on segment details
- Environment setup requirements
- Post-workshop resources

## Tone & Language Guidelines
- State learning outcomes clearly ("After this, you'll be able to...")
- Be upfront about prerequisites
- Specify the proportion of hands-on vs. instruction time
- Highlight the instructor's real-world experience
- Reduce the perceived barrier to entry
- Emphasize doing over listening

## Content Structure Strategy
- Opening info block: 15% — theme + date/venue
- Learning value block: 25% — what you'll learn + who it's for
- Curriculum content block: 30% — outline + hands-on segments
- Instructor resources block: 20% — instructor background + course materials
- Call to action block: 10%
- Emoji usage: 10–12 total

## Tone & Mood
Practical, learning-friendly, hands-on first. The no-nonsense feeling of "walk away and use it immediately."

## Style Variants

### Variant A: Skill Introduction
- **Best for:** Zero-background or entry-level workshops
- **Opening format:** 🌱 {Topic} Intro Workshop | Learn {skill} from scratch
- **Tone:** Welcoming, encouraging, progressive

### Variant B: Hands-On Advanced
- **Best for:** Workshops for participants with an existing foundation
- **Opening format:** 🔧 {Topic} Hands-On Workshop | {Core skill focus}
- **Tone:** Professional, applied, in-depth

### Variant C: Tool Tutorial
- **Best for:** Teaching a specific tool or framework
- **Opening format:** 🛠️ {Tool Name} Practical Tutorial | Get up and running fast
- **Tone:** Practical, efficient, immediately usable

## Notes & Best Practices

### Technical Workshop Checklist
1. **State learning objectives clearly**: What will participants be able to do after this? Say it explicitly
2. **Be honest about prerequisites**: Mismatched skill levels hurt everyone
3. **Keep the hands-on ratio high**: A workshop is not a lecture
4. **Send environment setup instructions in advance**: Don't waste session time on installations
5. **Instructor credibility comes from experience**: What they've built matters more than their title
