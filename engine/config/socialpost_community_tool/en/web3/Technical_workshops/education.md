# Web3 Technical Workshop – Education Event Knowledge Base

## 📌 Module Activation Criteria

Apply this module when the event theme involves the following keywords:
- Introductory coding workshop, zero-experience developer training
- Student technical workshop, university hands-on course
- Skill certification training, career skill workshop

---

## 🎯 Scenario Overview

| Dimension | Description |
|-----------|-------------|
| Format | Fundamentals lecture + Guided hands-on practice + Teaching assistant support + Project output |
| Duration | 2–6 hours, progressive learning |
| Participants | Learners, students, instructors, teaching assistants |
| Core Objective | Getting started, skill introduction, interest cultivation, portfolio building |

---

## 👥 Audience Profiles (by Priority)

| Priority | Audience Type | Key Concerns |
|----------|---------------|--------------|
| P0 | Zero-Experience Learners | Getting-started path, learning barrier, ease of entry |
| P1 | University Students | Hands-on opportunity, resume boost, industry awareness |
| P2 | Career Switchers | Transferable skills, learning efficiency, career direction |
| P3 | Hobbyists | Trying something new, understanding the tech, social connection |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Focus |
|------------|-------------------|------------|
| Zero-Experience Intro Workshop | Energetic & Engaging (B) | Relaxed atmosphere, encourage experimentation |
| University Partnership Workshop | Formal & Professional (A) | Academic value, structured learning |
| Experience-Based Workshop | Benefits-Oriented (D) | Low barrier, tangible output |
| Skill-Building Workshop | Suspense-Driven (C) | Solving questions, leveling up |

---

## 📝 Required Content Elements

### 1. Low-Barrier Promise (at least 2 instances)
- "Beginner-friendly — starting from your very first line of code"
- "Teaching assistants guide you throughout — no fear of falling behind"

### 2. Output Guarantee (at least 1 instance)
- "Build your first [project type]"
- "Walk away with a presentable project"

### 3. Support System (at least 1 instance)
- "[Number] teaching assistants available on-site"
- "Small-class format: 1 teaching assistant per [number] participants"

### 4. Disclaimer Note (recommended)
- "This event is educational in nature, aimed at skill introduction"

## ⚠️ Prohibited Actions

- ❌ Creating excessive pressure on zero-experience learners
- ❌ Exaggerating the learning outcomes of a single workshop
- ❌ Promising employment or skill certification
- ❌ Neglecting participants who are progressing more slowly
