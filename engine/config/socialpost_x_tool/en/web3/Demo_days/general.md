# DEMO DAYS × GENERAL

## Applicable Scenarios
Project Demo Days, pitch events, showcase days, accelerator Demo Days, ecosystem project showcases, cohort graduation events, investor showcase sessions

## Scenario Characteristics
- **Format**: Project pitches, judge commentary, Q&A, Networking
- **Core Value**: Project exposure, feedback gathering, investor connections, industry recognition

## Audience Segmentation
- **Primary Audience:** Project Teams — showcase their work, gather feedback, pursue opportunities
- **Secondary Audience:** Investors — discover projects, evaluate opportunities
- **Tertiary Audience:** Industry Practitioners — learn about new projects, identify collaboration opportunities
- **Extended Audience:** Media & Analysts — cover and report on new projects
- **Audience Behavior:** Present progress, gather feedback, build connections, discover opportunities

### Common Audience Combinations
- **Investment-focused:** Project teams + investors, suited for fundraising-oriented Demo Days
- **Ecosystem-focused:** Project teams + practitioners, suited for ecosystem showcase formats

## Content Elements
- Demo Day theme and positioning
- Pitching project overview / number of projects
- Judge and speaker lineup
- Pitch format details (duration, Q&A structure)
- Networking segment
- Post-event connection and follow-up mechanisms

## Tone & Language Guidelines
- Project-forward; emphasize the opportunity to showcase
- Judge lineup is a key selling point
- Highlight the value of structured feedback
- Clearly articulate connection and follow-up opportunities
- Investment-related content requires an NFA disclaimer

## Content Structure Strategy
- Opening info block: 15% — name + date/venue + scale
- Project overview block: 30% — pitching projects + vertical distribution
- Judge lineup block: 25% — judge backgrounds + feedback format
- Participation value block: 20% — who it's for + what attendees gain
- Call to action block: 10%
- Emoji usage: 8–12 total

## Tone & Mood
Highlight-driven, opportunity-oriented. The energy of "show what you've built." Supportive of builders, supportive of feedback.

## Style Variants

### Variant A: Multi-Track Showcase
- **Best for:** Multi-vertical comprehensive Demo Days
- **Opening format:** 🎯 {Demo Day Name} | {Number} projects taking the stage
- **Tone:** Comprehensive, opportunity-driven

### Variant B: Cohort Graduation
- **Best for:** Incubator / accelerator Demo Days
- **Opening format:** 🎓 {Incubator Name} Demo Day | Cohort {X} Graduation Pitches
- **Tone:** Milestone celebration, progress presentation

### Variant C: Ecosystem Showcase
- **Best for:** L1 / protocol ecosystem project showcases
- **Opening format:** ⚡ {Ecosystem Name} Project Showcase | Rising stars from the ecosystem
- **Tone:** Ecosystem-driven, discovery-focused

## Notes & Best Practices

### Demo Day Copywriting Checklist
1. **Projects are the core**: Clearly state how many projects are pitching and which verticals they cover
2. **Judge lineup matters**: It determines the quality of feedback and the value of post-event connections
3. **Clarify pitch format**: Duration, Q&A structure, and commentary style
4. **Bilateral value**: The event must be compelling for both project teams and attendees
5. **Networking segment**: The understated value of every Demo Day

### Accuracy Standards
- Confirm pitching projects before listing them in copy
- Verify judge lineup before publishing
- Investment-related content requires an NFA (Not Financial Advice) disclaimer
