# Web3 技术工作坊 - 技术类活动知识库

## 📌 模块激活条件

当活动主题涉及以下关键词时，应用本模块规则：
- 智能合约开发工作坊、协议技术 Workshop
- ZK/Layer2/跨链技术实操课、安全审计培训
- SDK/API 实战训练、开发工具深度教学

---

## 🎯 场景特征概述

| 特征维度 | 具体描述 |
|----------|----------|
| 形式 | 讲解 + 现场编码 + 导师辅导 + 答疑互动 |
| 时长 | 2-8 小时，深度实操 |
| 参与方 | 开发者、技术导师、协议方技术团队 |
| 核心目标 | 技能习得、实操掌握、问题解决、技术交流 |

---

## 👥 受众画像（按优先级）

| 优先级 | 受众类型 | 关注点 |
|--------|----------|--------|
| P0 | 协议开发者 | 新技术学习、最佳实践、性能优化 |
| P1 | 应用开发者 | 集成方案、SDK 使用、实战技巧 |
| P2 | 技术架构师 | 架构设计、技术选型、安全考量 |
| P3 | 技术学习者 | 入门实操、基础掌握、项目练手 |

---

## 🔀 风格选择指引

| 活动类型 | 推荐风格 | 核心侧重 |
|----------|----------|----------|
| 协议官方技术培训 | 正式专业型 (A) | 技术权威、官方认证 |
| 开发者实操工作坊 | 热情活力型 (B) | 动手实践、互动学习 |
| 高级技术深潜 | 悬念引导型 (C) | 技术难点、解决方案 |
| 工具/SDK 培训 | 福利导向型 (D) | 效率提升、资源获取 |

## 📝 内容必含元素

### 1. 技术价值（至少2处）
- "深度掌握 [技术/工具] 核心用法"
- "[协议/项目] 官方团队亲自授课"

### 2. 实操设计（至少1处）
- "全程 Coding，边学边练"
- "完成一个可运行的 [项目类型]"

### 3. 导师支持（至少1处）
- "[数量]+ 位导师现场辅导"
- "1v1 答疑，解决你的具体问题"

### 4. 合规提示（必须包含）
- "⚠️ 本活动为技术培训，不涉及投资建议（NFA）"
