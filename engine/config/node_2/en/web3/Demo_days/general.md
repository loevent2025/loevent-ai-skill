# Web3 Demo Day Event Planning Knowledge Base

## Scenario Overview

A Demo Day is a pitch-focused event in the Web3 industry centered on project showcases and investor introductions. Project teams present their products and business plans to investors, potential partners, and the community within a fixed time slot. It is a key channel for fundraising, project visibility, and ecosystem partnerships.

### Event Characteristics

- **Scale:** 50–300 attendees
- **Duration:** Half-day to 1 day
- **Format:** Project pitches + judge feedback + investor matchmaking + networking
- **Core Value:** Fundraising introductions, project exposure, industry exchange, ecosystem partnerships

| Type | Characteristics | Best Use Case |

| Accelerator Demo Day | Cohort graduation showcase, targeted investor invitations | Accelerator program graduation |

| Themed Demo Day | Focused on a specific vertical | DeFi / NFT / Infrastructure and other verticals |
| Closed-Door Demo Day | Investor-only attendance | High-quality curated project introductions |

### Project Selection Criteria

| Dimension | Key Evaluation Points |
|-----------|----------------------|
| Team Background | Founder track record, technical capabilities, industry experience |
| Product Stage | MVP / testnet / mainnet launch, user metrics |
| Market Positioning | Vertical focus, competitive analysis, differentiation |
| Funding Stage | Pre-seed / Seed / Pre-Series A / Series A, etc. |
| Presentation Readiness | Clear communication, demonstrable live product |

## Virtual Demo Day Guidelines

### Virtual Format Adaptations

| Element | In-Person | Virtual |
|---------|-----------|---------|
| Check-in | On-site registration | Join the livestream 10 minutes early |
| Pitch Presentation | Live on-stage with slides | Video livestream / pre-recorded video + live Q&A |
| Judge Interaction | In-room questions | Live chat / Q&A panel submission |
| Investor Matchmaking | On-site 1-on-1 meetings | Pre-scheduled video calls / post-event email introductions |
| Networking | Dinner / reception | Discord voice room / virtual networking session |

### Virtual Session Duration Guidelines

- Shorten individual pitch time to 8–10 minutes per project
- Maintain 3–5 minutes for Q&A per project
- Keep total event duration within 3 hours
- Schedule a mid-event break (10–15 minutes)

### Virtual Event Tools

| Purpose | Recommended Tools |
|---------|-------------------|
| Livestream Platform | YouTube Live, Twitter Spaces, Zoom Webinar |
| Engagement Tools | Slido (Q&A / polling), Discord |
| Project Materials | Notion pages, pitch deck links |
| Matchmaking System | Calendly, Luma 1:1 |

## Pitch Design Standards

### Individual Pitch Structure

**Time Allocation (12–15 Minutes)**

| Segment | Duration | Content |
|---------|----------|---------|
| Project Presentation | 8–10 min | Pitch deck walkthrough |
| Live Demo | 2–3 min | Product demonstration (optional) |
| Q&A | 3–5 min | Judge and investor questions |

### Pitch Deck Framework

| Section | Key Content | Suggested Slides |
|---------|-------------|------------------|
| Opening | One-line positioning, core value proposition | 1 |
| Problem | Pain point description, market opportunity | 1–2 |
| Solution | Product overview, technical architecture | 2–3 |
| Demo | Live product demo or screenshots | 1–2 |
| Market | Market size, target users | 1 |
| Business Model | Revenue model, monetization strategy | 1 |
| Competitive Analysis | Competitor comparison, differentiation | 1 |
| Traction | Key metrics, milestones achieved | 1 |
| Team | Core members, backgrounds | 1 |
| Funding Ask | Target raise, use of funds | 1 |
| Contact | Social media, email | 1 |

### Pitch Best Practices

- **Respect the time limit:** Presentations will be cut off if they run over
- **Demo over slides:** A live product demonstration is more persuasive than a pitch deck alone
- **Lead with data:** Highlight user numbers, TVL, transaction volume, and other key metrics
- **Tell a story:** Build in a memorable narrative hook
- **Prepare for Q&A:** Anticipate and rehearse answers to common investor questions

## Investor Matchmaking Design

### Matchmaking Process

| Phase | Content | Notes |
|-------|---------|-------|
| Needs Collection | Gather requirements from both sides before the event | Projects: funding stage, target raise; Investors: sector focus |
| Smart Matching | Generate recommendations based on stated needs | Prioritize high-compatibility pairings |
| Scheduling System | Mutual opt-in and time confirmation | Provide an online scheduling tool |
| On-Site Meetings | 1-on-1 sessions | 15–20 minutes per meeting |

### Meeting Area Setup

| Zone | Quantity | Configuration |
|------|----------|---------------|
| 1-on-1 Meeting Tables | 10–20 | Small round table + 2 chairs |
| Small Group Discussion Areas | 3–5 | Lounge seating |
| Open Networking Area | 1 | Standing, informal |

### Improving Matchmaking Efficiency

- Send project materials to investors in advance
- Share investor background profiles with project teams
- Set up automated meeting reminders
- Assign staff to guide participants to their meetings

|-------|--------|-------------|

| Pitch Highlights | Quote graphics, key moments | Real-time during event |

| Project Roundup | Long-form article / visual summary | After the event |

## Risk Management

| Risk Type | Description | Mitigation |
|-----------|-------------|------------|
| Project Withdrawal | A pitching team drops out last minute | Maintain a waitlist of backup projects; adjust the agenda accordingly |
| Time Overrun | Pitches significantly exceed their allotted time | Strict timekeeping, mandatory rehearsal |
| Demo Failure | Live product demo breaks during the presentation | Prepare a pre-recorded screen capture as backup |
| Investor No-Show | A key investor fails to attend | Confirmation protocol, backup investor list |
| Quiet Q&A | No one asks questions during the Q&A segment | Prepare a set of moderator-led seed questions |

## Execution Checklist

### Project Team Preparation

- Submit pitch deck for review in advance
- Test the demo environment thoroughly before the event
- Conduct a full run-through rehearsal (strongly recommended)
- Prepare answers to common investor Q&A questions

### Investor Experience

- Dedicated VIP check-in lane
- Distribute project materials before the event
- Open the matchmaking scheduling system in advance
- Assign a dedicated host for VIP investor hospitality

### Venue Setup

- **Main Stage:** Large display screen, countdown timer, teleprompter (optional)
- **Meeting Area:** Private, acoustically separated space
- **Rest Area:** Refreshments, power charging stations
- **Media Zone:** Dedicated interview and livestream area

### Equipment Checklist

- Projector / LED display screen
- Countdown timer (visible on the main display)
- Presentation clicker
- Backup laptop
- Video recording equipment
