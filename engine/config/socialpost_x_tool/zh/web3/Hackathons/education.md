# HACKATHONS × EDUCATION / 黑客松 × 教育类

## 适用场景
新手黑客松、学习型黑客松、训练营黑客松、大学黑客松、引导入门型黑客松、技能培养竞赛、导师重度参与的活动

## 场景特征
- **规模**：30-200 人
- **时长**：24-48 小时
- **形式**：学习 + 实战、导师重度参与、友好评审
- **核心价值**：技能学习、实战经验、导师指导、入门路径

## 受众定位
- **Primary Audience:** 新手开发者 / Beginner Developers - 想通过实战学习 Web3
- **Secondary Audience:** 学生 / Students - 计算机/相关专业学生，积累项目经验
- **Tertiary Audience:** 转型开发者 / Switching Developers - 从 Web2 转型
- **Extended Audience:** 非技术想学技术 / Non-tech Learners - 产品/运营想了解开发
- **Audience Behavior:** 学习新技能、积累项目经验、获得导师指导、建立开发者人脉

### 常用受众组合
- **Pure Beginner:** 新手为主，适合入门型 Hackathon
- **Mixed Level:** 新手 + 有经验者组队，适合带教型

## 内容元素
- 学习目标说明
- 导师配置（比例、背景）
- 前置学习资源
- 评审标准（学习友好）
- Workshop/教学环节
- 后续学习路径

## 语言处理
- 降低心理门槛
- 强调"学习"而非"竞争"
- 导师支持是核心卖点
- 评审标准友好（重参与、重学习）
- 鼓励试错

## 结构策略
- 开场信息块占比 15%
- 学习价值块占比 35%：能学到什么 + 导师配置
- 活动安排块占比 25%：Workshop + Hacking + 评审
- 参与说明块占比 15%：适合谁 + 要求
- 行动号召块占比 10%
- Emoji 密度控制：总量 12-14 个，友好鼓励

## 情绪氛围
热情、支持、以学习为中心。“你的第一场黑客松，我们为你保驾护航”的氛围。

## 风格变体

### 变体A：入门友好型 / Beginner Friendly
- **适用：** 新手专场、入门 Hackathon
- **开场格式：** 🌱 {Hackathon名} | 你的第一场 Web3 Hackathon
- **语气：** 欢迎、支持、无压力

### 变体B：学生专场型 / Student Track
- **适用：** 高校 Hackathon、学生开发者活动
- **开场格式：** 🎓 {Hackathon名} | 学生开发者专场
- **语气：** 青春、学习、成长

### 变体C：技能提升型 / Skill Building
- **适用：** 有一定基础想提升的开发者
- **开场格式：** 📈 {Hackathon名} | 从能跑到跑得好
- **语气：** 进阶、实战、提升

4. **导师配置要足**：这是核心卖点
