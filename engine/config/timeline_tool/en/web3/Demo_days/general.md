# Web3 Demo Day Knowledge Base

## Scenario Overview

A Demo Day is a professional roadshow event where Web3 projects showcase their products and technologies to investors, partners, and industry audiences. Typically hosted by accelerators, incubators, or investment firms, it is a crucial occasion for projects to secure funding and connect with resources.

## Tag Classification Extension

| Tag | Web3 Demo Day Specific Content |
|-----|-------------------|
| **venue** | Roadshow stage, investor zone, one-on-one meeting rooms, presentation equipment |
| **commercial** | Investor firm invitations, project screening, funding matching, strategic partnerships |
| **content** | Roadshow training, presentation slides, product demonstration, Q&A session |
| **others** | Project scheduling, time management, investor feedback collection, follow-up |

## Web3 Demo Day Specific Dependency Chains

### Chain E - Project Preparation
Project Screening (4d) → Roadshow Training (3d) → Presentation Optimization (3d) → Rehearsal (2d)

### Chain F - Investor Management
Investor List Organization (3d) → Targeted Invitations (4d) → Attendance Confirmation (2d) → Seating Arrangement (1d)

### Chain G - Product Demonstration
Demo Environment Setup (3d) → Demo Script Design (2d) → Technical Testing (2d) → Backup Plan (1d)

### Chain H - On-chain Presentation
On-chain Data Visualization (3d) → Real-time Transaction Demo (2d) → Smart Contract Interaction Display (2d)

## Time Compression Strategies

### Relaxed Mode (30+ days)
- Ample project screening and training cycles
- Multiple rounds of roadshow rehearsals
- Detailed one-on-one investor arrangements
- Complete data room preparation

### Compact Mode (15-29 days)
- Accelerate project screening pace
- Condense roadshow training into intensive sessions
- Simplify one-on-one matching process
- Handle investor invitations in parallel

### Urgent Mode (8-14 days)
- Screen only highly mature projects
- Rapid roadshow training
- Focus on inviting core investors
- Simplify demonstration segments

### Extreme Mode (1-7 days)
- Small-scale private roadshow
- Targeted invitations to known investors
- Utilize existing presentation materials
- Rapid matching mode

## Risk Warnings

- ⚠️ High risk of technical glitches during product demonstrations; prepare multiple backup plans
- ⚠️ Investor time is valuable; strictly control roadshow duration
- ⚠️ Project information involves commercial sensitivity; ensure confidentiality measures are in place
- ⚠️ On-chain demonstrations require stable testnet/mainnet environment
