# 跨界协作 × 通用类活动知识库

---

## 一、DEMO DAYS × 跨界协作

### 适用场景
Web2×Web3 项目路演、传统行业+区块链解决方案展示、跨生态联合 Demo Day、行业垂直+Crypto 路演、技术×商业双背景项目展示

### 场景特征
- **形式**：跨背景项目路演、双视角评审、跨界 Networking
- **核心价值**：跨界项目曝光、双领域资源对接、认知破壁、新赛道验证

### 受众定位
- **Primary Audience:** 跨界项目方——融合两个领域的创业团队
- **Secondary Audience:** 双背景投资人——关注跨界赛道的 VC/CVC
- **Tertiary Audience:** 传统行业决策者——探索区块链应用的企业代表
- **Extended Audience:** 跨界从业者——在两个领域都有涉猎的人
- **Audience Behavior:** 展示跨界方案、获取双视角反馈、寻找跨界资源、验证市场需求

#### 常用受众组合
- **资本对接型**：跨界项目 + 传统 VC + Crypto Fund
- **产业对接型**：跨界项目 + 行业企业 + 技术服务商

### 内容元素
- 跨界背景说明（为什么这两个领域需要融合）
- 路演项目的跨界属性
- 双背景评委配置
- 跨界评审标准
- 双边 Networking 机制
- 后续跨界资源对接

### 语言处理
- 突出"两个世界的交汇点"
- 用碰撞、融合等动态词汇
- 强调"只有这种组合才能解决的问题"
- 双边价值都要讲清
- 避免只对一方有吸引力

### 结构策略
- 开场信息块占比 10%
- 跨界背景块占比 20%：为什么这两个世界需要对话
- 项目说明块占比 30%：路演项目 + 跨界属性
- 评委阵容块占比 20%：双背景评委 + 评审视角
- 参与价值块占比 15%：对不同背景参与者的价值
- 行动号召块占比 5%
- Emoji 密度控制：总量 10-14 个

### 情绪氛围
碰撞的、发现的、开拓新边界的。"见证两个世界的交汇"的历史感。

### 风格变体

#### 变体A：行业×Crypto 型
- **适用：** 传统行业+区块链解决方案
- **开场格式：** 🔀 {行业} × Web3 Demo Day | 当 {行业} 遇见区块链
- **语气：** 产业、落地、破壁

#### 变体B：Web2×Web3 型
- **适用：** 互联网公司/项目的 Web3 转型
- **开场格式：** 🌉 Web2 → Web3 Project Showcase | 跨越的一步
- **语气：** 转型、迭代、融合

#### 变体C：生态联合型
- **适用：** 多个生态/协议的联合展示
- **开场格式：** ⚡ {生态A} × {生态B} Demo Day | 跨生态碰撞
- **语气：** 协作、互通、组合创新

### 注意事项

#### 跨界 Demo Day 要点
1. **双边价值要平衡**：对两个领域的人都要有吸引力
2. **评委配置要跨界**：双背景评审才能给出双视角反馈
3. **项目筛选要真跨界**：不是"贴标签"的跨界，是真正融合的
4. **Networking 要刻意混合**：打破背景分组，制造碰撞机会
5. **跨界叙事要真实**：说清为什么这个组合有意义
