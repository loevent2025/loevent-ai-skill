# WEB3 DEMO DAY × GENERAL / Web3项目展示日 × 通用类

## 适用场景
Web3加速器毕业展示、Crypto项目路演日、综合Web3 Demo Day、生态项目集中展示、赛道专场路演

## 场景特征
- **形式**：项目路演 + 产品Live Demo + 评委点评 + Networking + After Party
- **核心价值**：项目曝光、产品展示、获得反馈、投资对接、生态连接

## 受众定位
- **主要受众**：Web3创业团队 - 展示项目、获取反馈、寻求融资和合作
- **次要受众**：Crypto投资人 - 发现项目、评估机会、扩展Deal Flow
- **第三受众**：Web3从业者 - 了解创业方向、寻找合作、获取灵感
- **扩展受众**：大厂战投/交易所BD/媒体 - 发现标的、报道行业动态
- **受众行为**：展示产品、获取反馈、建立连接、发现机会

### 常用受众组合
- **投资导向**：创业团队 + 投资人，适合融资型Demo Day
- **生态导向**：创业团队 + 从业者 + 合作伙伴，适合生态展示型

## 内容元素
- Demo Day主题与定位
- 路演项目介绍（数量/方向/阶段分布）
- 评委/嘉宾阵容（投资人 + 行业OG + 技术专家）
- 路演 + Live Demo形式说明
- Networking环节 + After Party
- 后续对接机制

## 语言处理
- 项目导向，突出展示机会
- 强调「现场Live Demo」「链上能跑」的价值
- 评委阵容是重要卖点
- 小红书风格但保持专业
- 投资相关需NFA / DYOR声明

- 评委阵容块占比 25%：评委背景 + 点评形式

## 参会攻略建议

### 对于观众
- 提前了解路演项目和赛道分布，标记感兴趣的
- 准备好问题，Demo后可以在Q&A环节提问
- Networking和After Party环节主动找感兴趣的团队聊
- 在Twitter/小红书记录精彩项目和观点

### 对于投资人
- 提前看项目资料和链上数据，带着DD问题来
- 关注Demo的完成度、链上Traction和团队应变
- 会后及时跟进感兴趣的项目，约Office Hour
- 对比同赛道项目，验证自己的投资判断

### 对于创业者
- 重点准备Live Demo，这是最核心的环节
- 准备好回答技术、Token经济模型和商业问题
- 珍惜和其他创业者交流的机会，互相学习
- 不要只看投资人，生态伙伴同样重要
- 准备好Deck + One-pager + 合约地址 + 数据Dashboard

## 路演技巧指南（Web3专项）

### 对于路演者
- **开场**：30秒内抓住注意力，讲清楚解决什么链上痛点
- **痛点**：用链上数据或案例说明问题的严重性
- **方案**：简洁明了地解释你的链上解决方案
- **市场**：展示赛道规模（TAM/SAM）和增长潜力
- **产品**：Live Demo展示可运行的链上产品
- **Token**：简要说明Token经济模型和价值捕获逻辑
- **团队**：突出团队的Web3经验和执行力
- **Traction**：展示链上数据（TVL/用户数/交易量/增长率）
- **融资**：明确融资金额、估值和资金用途

### 常见问题准备
- 你和同赛道竞品的核心差异是什么？
- Token经济模型如何保证可持续性？
- 目前的链上Traction和增长趋势？
- 合约是否经过审计？安全措施是什么？
- 最大的技术风险和挑战是什么？
- 融资后12个月的里程碑计划？

### 演示技巧
- 控制在规定时间内，宁短勿长
- 多用链上数据和图表，少用大段文字
- Live Demo一定要提前测试稳定性（包括网络和Gas）
- 准备好Demo失败的备选方案（录屏/截图）
- 对项目的热情要溢于言表，但数据要扎实
- 诚实面对问题，不要回避安全和风险话题

## 注意事项

### 通用类Demo Day要点
1. **Demo是核心**：Web3项目必须有Live链上演示
2. **项目质量说清**：多少个、什么赛道、什么阶段、链上数据
3. **评委阵容重要**：投资人 + 技术专家 + 行业OG，决定反馈质量
4. **双边价值**：对创业者和观众/投资人都要有吸引力
5. **投资相关需声明**：NFA / DYOR / 对接≠承诺
6. **After Party很重要**：Web3行业，很多Deal在Party上谈成的
