# Cafe Coworking × Tech

## Applicable Scenarios
Developer cafes, Code & Coffee sessions, pre-hackathon warm-ups, protocol developer workdays, open-source contributor coworking, tech tool user coworking (Cursor, Figma, etc.)

## Scenario Characteristics
- **Format:** Developers coding together at a cafe, helping each other out, with optional short tech talks
- **Duration:** 3–6 hours
- **Core Value:** A focused coding atmosphere, peer-to-peer technical support, finding collaborators

## Audience Profile
- **Primary Audience:** Developers — looking for a space to focus and code
- **Secondary Audience:** Tech learners — learning to code and looking for people to ask questions
- **Tertiary Audience:** Technical founders — founders with an engineering background
- **Audience Behavior:** Coding, debugging, asking technical questions, exchanging ideas

## Content Elements
- Venue info (WiFi, power outlets, environment)
- Technical theme or focus (optional)
- Intro Circle (tech version: what you're building / where you're stuck / how you can help)
- Optional tech talks / workshops
- All skill levels welcome
- Tech community connections (Discord, etc.)

## Language Guidelines
- ✅ Developer-native, casual language
- ✅ "Bring your laptop and get coding," "stuck? ask the person next to you"
- ✅ All skill levels welcome
- ✅ Peer support and collaborative tone
- ❌ Avoid overly formal language
- ❌ Avoid elitist framing ("rockstar developers," "elite crowd")

## Content Structure
- Opening scene-setter: 15%
- Technical focus block: 20% (when a specific theme applies)
- Format overview block: 25%
- Peer support mechanism block: 20%
- Venue info block: 10%
- Call to action block: 10%
- Emoji density: 8–12 total

## Tone & Atmosphere
Focused, collaborative, and inclusive. The feeling of: *coding with others beats coding alone.*

## Style Variants

### Variant A: General Code & Coffee
- **Best for:** All developers
- **Opening format:** ☕💻 Code & Coffee | Bring your laptop, get things done
- **Tone:** Open, inclusive, all levels

### Variant B: Tool/Product User Coworking
- **Best for:** Users of specific tools (Cursor, Figma, VS Code, etc.)
- **Opening format:** ☕🔧 Cafe [Tool Name] | [Tool] users, let's build together
- **Tone:** Product community, user connection, team may be present

### Variant C: Tech-Themed Focus Session
- **Best for:** Specific technical stacks (Solidity, Rust, AI, etc.)
- **Opening format:** ☕⚡ [Tech] Developer Cafe | Building [Tech]? Come join us
- **Tone:** Focused, stack-specific, technically deeper

## Intro Circle Templates (Tech Version)

### Standard Version
```
🗣️ Intro Circle (2:30 PM):
- Who you are
- What you're building / working on today
- Where you're stuck / what help you need
- What you can help others with
(Free coworking after)
```

### Simplified Version
```
🗣️ Quick intro:
- What you're working on
- Need any help?
(30 seconds is fine, no need to go deep)
```

## Notes

### Key Principles for Tech Cafe Coworking

1. **Emphasize "All Skill Levels Welcome"**
   - Beginners are welcome
   - It's not about who's most skilled
   - Mutual support is the core vibe

2. **Peer-to-Peer Technical Support Is the Differentiator**
   - Intro Circle asks "where are you stuck?"
   - Encourage helping each other
   - Experienced devs can support newcomers

3. **Be Clear About Venue Requirements**
   - WiFi reliability
   - Number of power outlets
   - Noise level / working environment

4. **Tool/Product User Sessions May Offer Extra Perks**
   - Product credits
   - Swag
   - Team members may be on-site
   - Optional workshops

5. **Tech-Themed Sessions Are More Focused**
   - Same tech stack means easier peer support
   - Can mention a recommended baseline skill level
