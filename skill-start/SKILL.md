# LoEvent · 入门导览(start)

当用户对 LoEvent 一头雾水时,用它给一张清爽的上手地图。
**本 skill 没有脚本、不调任何 API**——你(agent)直接按下面内容组织一段中文回复即可。

## 何时使用(When to Use)
- 用户问"怎么用 / 这是什么 / 介绍一下 / help / 有哪些功能 / 从哪开始";
- 或用户明显没头绪、不知道该点哪个 skill(冷启动)。

## 第 0 步:先确认有可用模型(没配先配,再讲地图)

所有 skill 都要模型才能跑。展示地图**之前**先快速判一下(可 `python engine/doctor.py`):
- **已有可用模型**(有 `GEMINI_API_KEY`,或已配 `LOEVENT_TEXT_*`)→ 跳过,直接讲下面的地图,**一个字都别提配置**(别唠叨)。
- **都没配** → 先按 `references/API-KEY.md` **规则零**问一次"用哪个模型"(Gemini / 国内模型 GLM·DeepSeek / 自定义·网关),收好写进 `.env`,**再**讲地图。

国内用不了 Gemini 的用户尤其要在这步引导,否则地图讲得再好、一跑第一步就因没配模型报错。

## 输出什么(给用户的样子)
一句话定位 + 主线地图 + 支线清单 + 一个**具体的第一步**。例如:

> **LoEvent 是一套帮你从零策划活动的 AI 工具。** 用大白话描述活动,它就能一步步帮你做出完整方案。
>
> **主线(要一份完整策划方案,按这个走):**
> 1. **建档** —— 把你的活动描述整理成档案,所有功能的起点;
> 2. **受众画像** —— 这场该面向谁 + 他们的痛点;
> 3. **主办方调研 + 策略** —— 调研主办方/竞品,**给你 3 套活动策略方向供选**;
> 4. **选一套方向** → 5. **完整方案** —— 6 章节的成型策划方案。
>
> **支线(只想要单项,随时直接说):**
> 预算估算 / 筹备时间线 / 社媒文案 / Luma 发布文案 / 嘉宾简介 / 主办方简介 / 行业趋势调研 / 海报。
>
> **现在就开始**:把你的活动一句话告诉我(时间 / 地点 / 主题 / 主办方 / 规模),我先帮你建档。

## 要点(Pitfalls)
- **先模型、后地图**:按第 0 步,没配模型先引导配(国内用户引到 GLM 等),配好再讲地图;已配好就别提、别唠叨。
- **把黑话翻成产出**:别只甩 skill 代号(company/luma…),要说它**产出什么**(company = "给你 3 套活动策略方向")。
- 名称/字段用人话,别堆代号。
- 结尾**落到一个具体第一步**(通常是"描述你的活动 → 建档"),别只丢一堆并列选项让用户自己选。
- 用户已有明确目标(如"就要个海报")→ 别强推主线,直接进那个支线。
