# Web3 Cafe — Community Co-creation Event Marketing Strategy

## Event Overview

Web3 Cafe is a café-style co-creation event designed for local Web3 communities. Builders bring their laptops and spend the day building together, exchanging project ideas, and sharing experiences in a relaxed, social atmosphere. Events typically run for a full day (approximately 10 hours) with flexible drop-in time slots, emphasizing organic networking and builder culture. With approximately 100 attendees per edition, Web3 Cafe is a key format for growing local Web3 communities and fostering ecosystem exchange.

---

## Core Objectives

- Build a local Web3 Builder community
- Facilitate project exchange among Builders
- Attract new ecosystem participants and developers
- Drive community expansion and long-term retention

## Distribution Strategy

### Hashtag Strategy

- Primary tags: #Web3Cafe #[City]Web3
- Event tags: #Web3Cafe[City] #BuilderMeetup
- Engagement tags: #BuildInPublic #WAGMI

### Content Strategy

| Phase | Content Types | Posting Frequency |
|-------|---------------|-------------------|
| Recruitment | Event previews, venue sneak peeks, time slot introductions | Every 2 days |
| Warm-up | Featured project previews, Builder spotlights, perks announcements | Every 2 days |
| Final Push | Countdown posts, registration reminders, highlight teasers | Once daily |
| Event Day | On-site atmosphere, Builder check-ins, project showcases | Real-time updates |
| Recap | Event review, highlight moments, next-edition preview | 1–2 posts |

### Content Tone

- Relaxed and rooted in Builder culture
- Emphasis on community belonging
- Locally relevant and community-specific
- Welcoming to newcomers

### Twitter Space Planning

- **Pre-event:** Warm-up Space session covering the local Web3 ecosystem
- **During event:** Live on-site session with Builder project spotlights
- **Post-event:** Recap discussion and planning for the next edition

## Distribution Timeline

### T–2 Weeks: Recruitment Launch

- Publish event preview and establish hashtags
- Create dedicated Telegram event group / channel
- Go live on Luma and open registration
- Release first wave of key visual poster

### T–1 Week: Warm-up Phase

- Share venue atmosphere sneak peeks
- Announce participating projects and featured Builders
- Activate KOL reposts
- Drive community recruitment and amplification

### T–3 Days: Final Push

- Daily countdown posts
- Last-call registration reminders
- Announce event perks
- Publish venue directions and logistics

### Event Day

- Guide attendees through check-in
- Post real-time on-site atmosphere updates
- Encourage attendees to post and tweet check-ins
- Document Builder project showcases

### T+2 Days: Recap Phase

- Publish event highlight recap
- Collect attendee feedback
- Acknowledge and thank participating Builders
- Preview the next edition

### Target Community Types

| Community Type | Outreach Method | Content Focus |
|----------------|-----------------|---------------|

| Local Developer Groups | In-group posts, member invitations | Technical exchange, building opportunities |

### Community Retention Strategy

- **Dedicated Telegram Group:** Participants join upon registration; group is retained after the event
- **Discord Channel:** Ongoing technical discussions and resource sharing
- **Regular Cadence:** Establish a consistent event rhythm (e.g., monthly editions)
