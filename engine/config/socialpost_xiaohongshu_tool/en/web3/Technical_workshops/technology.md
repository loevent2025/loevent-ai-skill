# Web3 Tech Workshop Knowledge Base – Technical

## Scene Overview
Technical Web3 workshops are designed for developers with an established foundation in Solidity or Rust, focusing on deep learning and production-grade practice within specific technical domains. These events prioritize technical depth, contract security, gas optimization, and on-chain architecture design. They are the primary learning format for senior developers looking to push their expertise further and tackle complex on-chain engineering challenges.

---

## Scene Characteristics
- **Scale**: Typically 15–40 participants, optimized for deep technical exchange and code review
- **Duration**: 1–2 days — depth requires time
- **Format**: Conceptual deep dives + source code / contract analysis + advanced hands-on labs + code review
- **Participants**: Experienced smart contract developers, protocol engineers, security researchers, technical architects
- **Key Traits**: High technical bar, advanced content depth, emphasis on code quality and production readiness

## Curriculum Design Recommendations

### Content Planning
- Ensure content delivers real, substantive technical depth (EVM-level / source-level)
- Balance conceptual instruction with hands-on contract implementation time
- Prepare source-level walkthroughs of top-tier protocols (Uniswap / Aave / MakerDAO)
- Design challenging, non-trivial hands-on tasks (bug fixing / gas optimization / security audit)
- Build in dedicated time for technical discussion, code review, and Q&A

### Instructor Requirements
- Must have genuine mainnet production-grade smart contract development experience
- Prior protocol audit or security research background strongly preferred
- Capable of fielding and engaging with advanced EVM and contract architecture questions
- Prepared to share real lessons learned from actual audits and production deployments
- Able to conduct high-quality, actionable code review
- Patient and thorough when working through complex on-chain problems

### Participant Selection
- Publish clear, explicit prerequisites (Solidity / Rust foundation + real project experience)
- Consider requiring a GitHub submission and technical background review
- Minimize the experience gap between participants to ensure high-quality peer exchange
- Encourage participants to arrive with real contract problems and actual code
- Gather participant technical focus areas and goals before the session

### Hands-on Design
- Provide a high-quality, well-structured contract code framework and Foundry test suite to start from
- Design exercises that build progressively in difficulty (analyze → fix → optimize → audit)
- Reserve dedicated time for debugging, profiling, and gas optimization
- Be prepared to handle a wide variety of on-chain edge cases and unexpected issues
- Include a structured code review segment with opportunities for mutual peer audit
