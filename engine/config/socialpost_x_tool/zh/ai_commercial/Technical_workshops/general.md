# 技术工作坊 × 通用类（AI/Model 商业化版）

## 适用场景
LLM 应用开发工作坊、AI Agent 实战工作坊、RAG 系统搭建工作坊、Prompt Engineering 工作坊、AI 工具使用培训、模型微调实操工作坊

## 场景特征
- **形式**：讲授 + 实操、Live Coding、导师指导、即时答疑
- **核心价值**：技能获取、动手实践、问题解决、即时反馈

## 受众定位
- **主要受众**：AI 开发者——想学习特定技能的工程师
- **次要受众**：技术产品经理——需要理解技术实现的产品人
- **第三受众**：AI 学习者——通过实操学习的新手
- **延伸受众**：技术负责人——了解技术细节以便决策的管理者
- **受众行为**：学习技能、动手实践、解决问题、获取经验

### 常用受众组合
- **开发者核心**：工程师为主，适合技术深度工作坊
- **混合背景**：开发者 + 产品 + 管理者，适合概念+实操结合
- **学习为主**：学习者 + 入门开发者，适合入门工作坊

## 注意事项

### 技术工作坊要点
1. **实操是核心**：讲完就练，不能只讲不练
2. **前置要求要明确**：让参与者知道自己是否适合
3. **导师配比要够**：确保每个人都能得到帮助
4. **环境准备要提前**：避免现场花时间装环境
5. **产出物要具体**：学完能带走什么
