# Web3 Hackathon — Technical Events Knowledge Base

## 📌 Module Activation Criteria

Apply this module when the event theme involves the following keywords:
- Protocol development hackathon, smart contract programming marathon
- ZK / Layer 2 / Cross-chain technology track, infrastructure development competition
- Security audit challenge, developer tooling innovation competition

---

## 🎯 Scenario Overview

| Dimension | Description |
|-----------|-------------|
| Format | Pre-event training + Team development + Mentor guidance + Demo showcase + Judging and awards |
| Duration | 24–72 hours intensive development, or 1–4 weeks online competition |
| Participants | Developers, protocol teams, technical mentors, expert judges |
| Core Objective | Technical innovation validation, prototype development, talent discovery, ecosystem building |

---

## 👥 Audience Profiles (by Priority)

| Priority | Audience Type | Key Focus |
|----------|---------------|-----------|
| P0 | Full-Stack / Contract Developers | Technical challenges, prize pool, project incubation |
| P1 | Protocol Researchers | Technical exploration, innovation validation, academic value |
| P2 | Technical Founders | MVP development, team formation, funding opportunities |
| P3 | Student Developers | Hands-on practice, portfolio building, industry entry |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Focus |
|------------|-------------------|------------|
| Official Protocol Hackathon | Formal & Professional (A) | Technical depth, official endorsement, ecosystem value |
| Community Developer Competition | Energetic & Engaging (B) | Participatory atmosphere, learning and growth, community connection |
| Technical Challenge | Suspense-Driven (C) | Technical difficulty, innovation directions, breakthrough opportunities |
| High-Prize / Incubation Competition | Value-Driven (D) | Prize pool, grants, incubation resources |

---

## 📝 Required Content Elements

### 1. Event Value (at least 2 instances)
- "Total prize pool of [amount] across [number] technical tracks"
- "Outstanding projects receive an official grant from [protocol/organization]"

### 2. Technical Support (at least 1 instance)
- "[Number]+ technical mentors providing end-to-end guidance"
- "Official [SDK/API/documentation] and technical support provided"

### 3. Incubation Pathway (at least 1 instance)
- "Winning projects gain direct entry to [incubator/accelerator]"
- "Priority introductions to [investment institution]"

### 4. Compliance Disclaimer (mandatory)
- "⚠️ This event is a technical development competition and does not constitute investment advice (NFA)"

## ❓ FAQ Templates

| Question | Answer |
|----------|--------|

| Do I have to use a specific tech stack? | Track-relevant technologies are required — refer to official rules for details |
| Can existing projects participate? | Significant new development must be completed during the event period |

## ⚠️ Prohibited Practices

- ❌ Exaggerating prize amounts or grant commitments
- ❌ Implying token issuance or investment returns
- ❌ Plagiarism or reusing existing project code
- ❌ Misrepresenting incubation or funding resources
