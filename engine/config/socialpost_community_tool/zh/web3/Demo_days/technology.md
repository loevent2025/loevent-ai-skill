# Web3 Demo Day - 技术类活动知识库

## 📌 模块激活条件

当活动主题涉及以下关键词时，应用本模块规则：
- 技术项目展示日、协议 Demo Day、开发者成果展
- 黑客松成果路演、技术原型展示、产品发布会
- Grant 项目汇报、技术里程碑展示

---

## 🎯 场景特征概述

| 特征维度 | 具体描述 |
|----------|----------|
| 形式 | 项目路演 + 技术评审 + 现场 Demo + Q&A 互动 |
| 时长 | 半天至一天，每项目 5-15 分钟展示 |
| 参与方 | 项目团队、技术评审、协议方、开发者观众 |
| 核心目标 | 技术验证、成果展示、反馈获取、合作对接 |

---

## 👥 受众画像（按优先级）

| 优先级 | 受众类型 | 关注点 |
|--------|----------|--------|
| P0 | 协议/项目方 | 生态项目质量、合作机会、Grant 评估 |
| P1 | 技术评审 | 技术可行性、创新性、完成度 |
| P2 | 开发者观众 | 学习借鉴、技术趋势、合作机会 |
| P3 | 技术媒体 | 报道素材、行业动态、项目发现 |

---

## 🔀 风格选择指引

| 活动类型 | 推荐风格 | 核心侧重 |
|----------|----------|----------|
| 协议官方 Demo Day | 正式专业型 (A) | 技术权威、生态价值 |
| 黑客松成果展示 | 热情活力型 (B) | 成果庆祝、社区氛围 |
| Grant 项目汇报 | 正式专业型 (A) | 里程碑验收、持续支持 |
| 技术原型首发 | 悬念引导型 (C) | 技术突破、创新亮点 |

---

## 📝 内容必含元素

### 1. 展示价值（至少2处）
- "[数量] 个技术项目集中展示"
- "首次公开 [技术/产品] Demo 演示"

### 2. 评审阵容（至少1处）
- "[协议/机构] 核心团队技术评审"
- "[数量]+ 位技术专家现场点评"

### 3. 互动机会（至少1处）
- "现场体验项目 Demo"
- "与项目团队深度交流"

### 4. 合规提示（必须包含）
- "⚠️ 项目展示不构成投资建议（NFA）"
