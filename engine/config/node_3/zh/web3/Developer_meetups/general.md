# Web3 · Developer Meetup · 场地筛选维度知识库

## 规模速查

| 规模等级       | 人数范围    | 典型场景                         |
| ---------- | ------- | ---------------------------- |
| 小型技术沙龙     | 15–40人  | 闭门协议讨论、合约审计分享、核心贡献者聚会        |
| 标准Meetup   | 40–80人  | 技术分享 + 协议Demo + 社区交流         |
| 大型Meetup   | 80–150人 | 生态联合活动、公链开发者日、大会周边Side Event |
| Hack Night | 30–100人 | 合约开发、协议集成、链上工具实战             |

# 筛选维度

---

## 维度1：技术基础设施可靠性（硬性条件）

**定义：**
场地是否具备支撑技术分享和现场链上Demo的稳定基础设施。Web3 Demo的特殊风险在于：链上交互、节点连接、钱包签名对网络稳定性和访问权限极度敏感，网络被墙或延迟过高会直接导致演示失败。

**考察方向：**

网络
- 上行+下行带宽 ≥100Mbps（80人以上建议200Mbps）
- WiFi支持高并发连接（按参会人数80%同时在线估算）
- 演讲区可接有线网络（链上演示优先走有线）
- 无端口限制、无防火墙屏蔽（需可访问：主流RPC节点、Infura/Alchemy/QuickNode、IPFS、各链浏览器）
- 是否允许自带路由器或独立热点接入
- 有备用网络方案（4G/5G移动热点）

⚠️ Web3特殊风险：部分企业/园区网络会屏蔽境外节点访问，必须提前实测RPC连接，不接受口头承诺。

演示设备
- HDMI / Type-C 接口齐全，有常见转接头备用
- 投影或屏幕分辨率 ≥1080P（合约代码、交易哈希必须清晰可读）
- 麦克风稳定无啸叫（无线麦优先）
- 音频系统支持视频播放

电源
- 演讲区、Demo区独立供电插座
- 观众区每排可接电（开发者笔记本电量敏感）
- 总负荷支持同时运行10+台设备

---

## 维度2：空间结构与互动适配性（硬性条件）

**定义：**
Web3 Meetup包含两种状态：技术分享（聚焦、安静、视线集中）和社区交流（流动、自由、多点对话）。

**考察方向：**

分享模式
- 演讲区视线无遮挡，100%座位可看到屏幕
- 座椅可移动或可灵活重排
- Q&A环节观众可方便站起发言
- 演讲区与观众区有明确视觉焦点分隔

交流模式
- 有充足站立交流空间（Web3社区偏好走动式交流）
- 有自然聚集点：茶歇台、高脚桌、走廊区
- 有独立Demo角（允许多人围观链上互动）
- 空间层高 ≥3m（避免压抑感）
- 空间整体氛围鼓励横向交流，而非"台上讲台下听"

⚠️ 衡量标准：活动结束后，参与者是否有理由和空间自发聚在角落继续讨论协议细节。

---

## 维度3：Web3行业调性匹配（高权重条件）

**定义：**
空间气质是否与Web3开发者文化匹配。Web3社区的核心文化价值观是去中心化、开源精神、极客自由主义——对"过度商业化""企业感太强""形式主义"极度排斥，同时对"真实感""草根感""hack文化"有天然认同。

**Web3调性定位：**
> 有极客感，但不脏乱差
> 有社区感，但不像村委会
> 有自由感，但不像无组织散摊

**✅ 符合的空间类型：**
- 创意园区改造空间（工业风、仓库改造、Loft）
- 联合办公空间（开放式、无过度企业装潢）
- 区块链/Web3公司自有办公空间
- 高校技术社团场地（契合开源文化）
- 地下室或半地下空间（若设施完善，在Web3语境下反而加分）

**❌ 避免的空间类型：**
- 传统酒店宴会厅（过度商务感，与Web3文化强烈冲突）
- 过于精致的企业会议室（"打工人开会感"破坏社区氛围）
- 奢华场地（Web3社区对奢华感有本能排斥，容易引发"这钱哪来的"质疑）
- 过度品牌化的商业赞助场地（社区会感知到"被营销"）

---

## 维度4：链上Demo与协议演示支持能力（高权重条件）

**定义：**
Web3 Meetup的核心Demo场景与AI完全不同：链上交互、合约部署、钱包连接、节点实时响应是主要演示内容，对网络访问自由度和稳定性要求极高。

**考察方向：**

通用Demo支持
- 有独立Demo角，可同时运行2–5个展位
- 每展位独立供电（不共用同一插座）
- 支持音频/视频实时播放
- 允许现场录屏和直播
- 光线适中（昏暗环境在Web3场景可接受，但屏幕需清晰可见）

Web3特殊要求
- 网络可无障碍访问：主流公链RPC节点（Ethereum、Solana、BNB Chain等）、区块浏览器（Etherscan等）、IPFS节点
- 网络延迟稳定（链上交易广播和确认演示对延迟敏感）
- 支持MetaMask/硬件钱包等本地签名演示（无需云端依赖）
- 投屏支持同时展示多窗口（合约代码、交易记录、前端界面并排）
- 允许演示者连接自带节点或本地测试网

---

## 维度5：容量与舒适密度控制（硬性条件）

**定义：**
场地在目标人数下保持"适度热闹"状态。Web3 Meetup容忍略高的密度感，"挤在一起讨论"本身契合hack文化，但不能影响设备操作和基本舒适度。

**面积参考标准：**

| 模式        | 每人面积  |
| --------- | ----- |
| 剧场式分享     | 1–1.2㎡ |
| 站立社交交流    | 1.5㎡  |
| Hack Night | 2㎡    |

**考察方向：**
- 以舒适容量为准，而非场地标注最大容纳人数
- 空调和通风独立可控（高密度人群+多台设备发热，温控很重要）
- 夏季/冬季温控响应速度
- 卫生间数量（建议50人/1间）
- 如有户外或半户外区域，雨天备用方案是否可行

---

## 维度6：时间灵活性与使用弹性（硬性条件）

**定义：**
Web3开发者时区分散（常需配合海外社区时间）、讨论习惯延时，Hack Night更可能跨夜。时间弹性在Web3场景中的重要性普遍高于AI Commercial场景。

**考察方向：**
- 支持标准时段：18:00–22:00
- 允许延时使用（计费规则是否合理）
- 夜间出入无限制（保安/门禁配合）
- 明确最晚撤场时间（能否支持到23:00甚至更晚）
- 允许提前2–3小时进场布置
- 活动正式结束后允许自由交流延续

Hack Night场景：时间弹性升为第一硬性条件，需确认是否支持通宵使用，以及安保和空调是否全程配合。

---

## 维度7：交通与到达便利性（可权衡条件）

**定义：**
Web3开发者群体对"为了一个好的活动多走一段路"的接受度略高于AI Commercial受众，但交通便利性依然影响到场率，尤其对非核心社区成员。

**考察方向：**
- 距地铁站步行15分钟内（Web3可适当放宽至15分钟，AI Commercial建议10分钟）
- 打车可接送（有明确上下车点）
- 夜间离场安全（周边环境、照明）
- 场地入口标识清晰，易于导航（非标准场地尤其重要）
- 允许放置活动引导标识

⚠️ Web3社区核心成员对"偏僻但有意思的场地"接受度更高，可以用场地调性弥补交通劣势，但需在活动宣传中提前做好导航说明。

---

## 维度8：社区沉淀能力（可权衡条件）

**定义：**
场地是否利于Web3社区的长期建设和自然沉淀。与AI Commercial不同，Web3社区对"品牌背书感"要求低，更看重"这个地方是我们自己的据点"的归属感。

**考察方向：**
- 是否允许长期或定期复用（建立"固定社区据点"记忆）
- 是否有适合拍照的区域（用于社交媒体传播，风格偏真实感而非精致感）
- 允许活动全程拍摄和发布
- 支持社区留存动作（加群引导、钱包地址扫码等）
- 场地方是否本身是Web3行业从业者或友好方（生态协同加分）
- 是否允许社区自带装饰或改造（增强归属感）

⚠️ 与AI Commercial的核心差异：AI Commercial强调品牌背景板和视觉呈现；Web3社区对大型品牌背景板可能产生"被赞助商绑架"的负面观感，品牌呈现需更克制、更社区化。

---

## 维度9：运营合规与使用限制（可权衡条件）

**定义：**
场地是否存在影响活动正常运营的使用限制。Web3活动使用非标准场地（Loft、仓库、地下室）的频率高于AI Commercial，需重点核查使用限制。

**考察方向：**
- 最大承载人数是否明确
- 是否允许举办公开技术分享（部分写字楼有限制）
- 是否存在商业活动限制或需额外审批
- 是否允许外部餐饮和酒水进场
- 非标准场地的使用合规性（租赁合同是否允许此类活动）
- 场地责任险归属是否明确

# 维度默认优先级

## 硬性条件（不满足直接排除）
1. 技术基础设施可靠性（尤其：境外节点访问权限）
2. 空间结构与互动适配性
3. 容量与舒适密度
4. 时间灵活性与使用弹性

## 高权重条件（强烈影响决策）
5. Web3行业调性匹配
6. 链上Demo与协议演示支持能力

## 可权衡条件（结合具体情况判断）
7. 交通与到达便利性
8. 社区沉淀能力
9. 运营合规与使用限制

# 混合场景处理指引

### 技术分享 + 酒会
- 优先评估"剧场 → 社交"翻台时间（是否 ≤30分钟）
- 灯光是否支持明暗调节（Web3活动偏好较暗的氛围灯，避免荧光灯）
- 固定桌椅是否可快速移除或折叠

### Meetup + Hack Night
- 网络带宽标准提升至200Mbps
- 境外节点访问权限升为第一验证项
- 电源数量：每工位独立插座
- 时间弹性升为第一硬性条件（需确认通宵可行性）
- 面积按每人2㎡计算

### Meetup + Side Event（大会周边活动）
- 场地距主会场步行可达优先（参会者不愿远距离转场）
- 品牌呈现需克制（避免过度商业化影响社区观感）
- 时间卡点严格（大会期间参与者日程密集，撤场时间需精确）
- 网络独立性尤为重要（大会周边网络通常拥堵）
