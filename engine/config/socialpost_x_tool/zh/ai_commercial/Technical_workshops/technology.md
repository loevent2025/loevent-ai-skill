# 技术工作坊 × 技术类（AI/Model 商业化版）

## 适用场景
Agent 架构设计工作坊、LLM 性能优化实操、RAG 深度调优工作坊、模型微调实战、AI 安全攻防实操、LLMOps 工具链搭建

## 场景特征
- **形式**：深度讲解 + 高强度实操、代码 Review、架构讨论
- **核心价值**：深度技能、架构能力、性能优化、生产经验

## 受众定位
- **主要受众**：有经验的 AI 工程师——想深入特定领域的开发者
- **次要受众**：技术负责人——需要掌握核心技术的 Tech Lead
- **第三受众**：架构师——设计 AI 系统的架构师
- **延伸受众**：开源贡献者——想深入理解原理的贡献者
- **受众行为**：深度学习、解决难题、掌握原理、提升能力

### 常用受众组合
- **硬核技术**：有经验工程师为主，适合深度技术工作坊
- **架构设计**：架构师 + 技术负责人，适合架构工作坊
- **性能优化**：工程师 + 负责人，适合优化类工作坊

```

📍 上海浦东 [场地名]

---

**Agent 的 Memory 怎么设计？不只是会用，要知道为什么这样设计。**

🎯 学习目标：
- 深入理解 Agent Memory 的设计原理
- 掌握不同 Memory 方案的适用场景
- 学会针对业务场景选择和设计 Memory
- 了解 Memory 管理的性能和成本考量

**Part 1：Memory 原理（45分钟）**
- Memory 在 Agent 中的作用
- 短期 Memory vs 长期 Memory
- Memory 的存储和检索机制
- 不同框架的 Memory 实现对比

**Part 2：方案选型（60分钟）**
- 讲解：常见 Memory 方案（Buffer / Summary / Vector / Entity）
- 实操：在同一场景下对比不同方案
- 分析：每种方案的优缺点和适用场景
- 讨论：你的场景应该选哪种？

**Part 3：设计实践（60分钟）**
- 讲解：Memory 设计的关键决策点
- 实操：为一个复杂场景设计 Memory 方案
- Review：互相看看设计，讨论 Trade-off
- 优化：根据反馈调整设计

**Part 4：性能与成本（45分钟）**
- 讲解：Memory 的性能瓶颈和优化方法
- 实操：Memory 压缩和清理策略
- 分析：成本控制的实用技巧
- 案例：真实项目的 Memory 演进

**Q&A + 总结（30分钟）**

📋 前置要求：

🎁 参与者获得：

- 设计模板和参考架构
- 性能优化 Checklist
```
