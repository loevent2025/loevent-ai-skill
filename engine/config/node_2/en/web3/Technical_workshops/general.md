# Web3 Technical Workshop Event Planning Knowledge Base

## Scenario Overview

A technical workshop is an education-focused event in the Web3 industry centered on skill-building and hands-on technical training. Participants learn specific tech stacks or complete practical projects under expert instruction. It is a key channel for developer enablement, technology adoption, and ecosystem growth.

### Event Characteristics

- **Scale:** 20–100 participants
- **Duration:** 2–6 hours (single session) / 1–5 days (series)
- **Format:** Technical instruction + hands-on labs + guided Q&A
- **Core Value:** Skill development, hands-on experience, developer enablement, ecosystem building

---

## Workshop Types and Selection

### Workshop Format Comparison

| Type | Duration | Characteristics | Best Use Case |
|------|----------|-----------------|---------------|
| Introductory Workshop | 2–3 hours | Quick onboarding, concept overview | Beginner onboarding, technology outreach |
| Advanced Workshop | 4–6 hours | In-depth learning, end-to-end projects | Developers with existing foundations |
| Workshop Series | Multi-day / Multi-session | Structured curriculum, ongoing progression | Technical training programs, certification tracks |
| Themed Workshop | 3–4 hours | Deep dive into a specific topic | New technology launches, feature walkthroughs |

### Difficulty Levels

| Level | Target Audience | Prerequisites | Learning Outcomes |
|-------|-----------------|---------------|-------------------|
| L1 — Beginner | Non-technical / Web2 developers | Basic programming ability | Understand core concepts, complete a working demo |
| L2 — Foundational | Developers with basic knowledge | Familiarity with foundational concepts | Build a simple DApp |
| L3 — Intermediate | Experienced developers | Prior DApp development experience | Master advanced techniques |
| L4 — Advanced | Senior engineers | Deep development experience | Solve complex problems, apply best practices |

## Virtual Workshop Guidelines

### Virtual Format Adaptations

| Session Element | In-Person | Virtual |
|-----------------|-----------|---------|
| Check-in | On-site registration | Join 15 min early, self-serve environment check |
| Instruction | Live on-stage demo | Screen share + live coding |
| Lab | In-room facilitation | Breakout Rooms + TA floor monitoring |
| Q&A | Open floor questions | Chat / Raise Hand / TA direct message |
| Engagement | In-room interaction | Live polls / code sharing |

### Virtual Session Duration Guidelines

- Cap individual instruction segments at 45 minutes
- Schedule a 10-minute break every 45 minutes
- Extend lab segments by 15–20% to account for async troubleshooting
- Keep total session length to 4 hours or less

### Recommended Virtual Tools

| Purpose | Recommended Tools |
|---------|-------------------|
| Live Sessions / Meetings | Zoom (Breakout Rooms), Google Meet |
| Collaborative Coding | GitHub Codespaces, Replit, CodeSandbox |
| Live Q&A | Slido, Discord |
| Documentation Sharing | Notion, HackMD |
| Session Recording / Replay | YouTube, Loom |

### Virtual Workshop Best Practices

- **Environment Pre-check:** Distribute environment setup guide in advance
- **TA Coverage:** 1 TA per 15–20 participants
- **Engagement Cadence:** Build in an interaction touchpoint every 15 minutes
- **Session Recording:** Provide replay for post-event review
- **Community Support:** Create a participant group for ongoing Q&A

## Instructor and TA Configuration

### Instructor Requirements

| Dimension | Requirement |
|-----------|-------------|
| Technical Expertise | Hands-on experience in the domain; able to troubleshoot live issues |
| Communication | Clear, structured delivery; ability to simplify complex concepts |
| Teaching Experience | Prior instructional experience; awareness of common learner pain points |
| Approachability | Patient and encouraging, especially when handling beginner questions |

### Instructor Profiles

| Type | Best Fit |
|------|----------|
| Core Protocol Developer | Official technology outreach, in-depth technical sessions |
| DevRel Engineer | Introductory workshops, ecosystem onboarding |
| Experienced Builder | Hands-on project experience, real-world best practices |
| Security Researcher | Security-focused workshops, audit training |

### TA Coverage

| Participant Count | Number of TAs | Responsibilities |
|-------------------|---------------|------------------|
| Under 20 | 1 | Environment support, Q&A |
| 20–50 | 2–3 | Zone coverage, floor monitoring |
| 50–100 | 4–6 | Group-level facilitation |

**TA Responsibilities**
- **Pre-Event:** Prepare environment setup documentation, provide pre-check support
- **During Event:** Resolve environment issues, assist with Q&A, log common problems
- **Post-Event:** Compile FAQ, follow up on unresolved participant questions

## Risk Management

| Risk Type | Description | Mitigation |
|-----------|-------------|------------|
| Environment Issues | Large-scale participant environment failures | Cloud IDE fallback, TA rapid response |
| Pacing Issues | Instruction outpaces learner comprehension | Flexible pacing, tiered exercises |
| Network Issues | Testnet congestion, RPC outages | Backup nodes, local environment fallback |
| Instructor Issues | Last-minute instructor cancellation | Backup instructor, recorded session fallback |
| Low Engagement | Participants hesitant to ask questions or interact | Designed interaction moments, proactive facilitation |

## Execution Checklist

### Venue Requirements

- Individual workstations with power outlets for each participant
- Reliable, high-bandwidth Wi-Fi
- Clear, high-resolution projection or display screens
- Whiteboard or flip chart (optional)

### Materials Checklist

| Category | Items |
|----------|-------|
| Technical Supplies | Power strips, backup ethernet cables, whiteboard markers |
| Learning Materials | Course outline, code repository link, reference documentation |
| Event Supplies | Registration sheet, name badges, feedback forms |
| Refreshments | Coffee, beverages, snacks |

### Post-Event Follow-Up

- Send course materials and code repository link
- Set up and activate participant community (Discord / Telegram)
- Collect feedback and iterate on content
- Share recommended follow-on learning resources
