# Web3 Tech Workshop Knowledge Base – Education

## Scene Overview
Education-focused Web3 tech workshops are designed for professionals seeking a structured approach to blockchain development, Web2-to-Web3 developer transitions, and students. These programs emphasize systematic knowledge building and transferable skills. Through carefully designed curricula and extensive on-chain hands-on practice, participants build Web3 development competencies from the ground up — making this format a key entry point for smart contract education and Web2-to-Web3 career transitions.

---

## Scene Characteristics
- **Scale**: Typically 20–60 participants, optimized for learning outcomes and debugging support
- **Duration**: 1–5 days, structured for comprehensive learning
- **Format**: Systematic instruction + step-by-step practice + assignments + project-based assessment
- **Participants**: Web2-to-Web3 developers, early-career professionals, students, blockchain beginners
- **Key Traits**: Low barrier to entry, structured curriculum, progressive difficulty, project-backed outcomes

## Curriculum Design Recommendations

### Curriculum Architecture
- Define clear, outcome-oriented learning objectives for each module (what can participants do after each stage)
- Modular structure: fundamentals → contract development → testing and deployment → frontend interaction → capstone project
- Ensure coherent knowledge progression across modules — concepts should unlock progressively
- Scale difficulty gradually, with tangible on-chain deliverables at each milestone
- Use testnet deployment as the milestone marker for each learning phase

### Learning Support
- Staff sufficient TAs (recommended ratio: 1 TA per 10–15 participants)
- Establish a dedicated learner Telegram / Discord community
- Provide a comprehensive code repository and full learning materials package
- Design integrated on-chain hands-on assignments
- Schedule dedicated Office Hours for Q&A and one-on-one support

### Assessment & Evaluation
- Implement milestone-based testnet deployment assessments at each stage
- Provide substantive code review feedback on all assignments
- Use a capstone project as the final deliverable and graduation requirement
- Issue on-chain learning credentials (SBT / POAP) upon completion
- Offer re-audit opportunities and pathways to advanced programs

### Ongoing Community Management
- Maintain an active and engaging alumni Builder community
- Surface Hackathon opportunities and ecosystem grant announcements for alumni
- Host alumni events and Demo Days to showcase learner projects
- Collect participant feedback continuously to improve the curriculum
- Build and promote a library of standout alumni success stories
