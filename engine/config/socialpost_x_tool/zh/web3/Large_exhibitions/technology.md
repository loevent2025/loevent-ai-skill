# LARGE EXHIBITIONS × TECHNICAL / 大型展会 × 技术类

## 适用场景
大型科技展会、区块链峰会、协议展示会、基础设施展览、大型会议开发者专区、生态系统展馆、Web3技术博览会、年度行业大会、多日科技节

## 场景特征
- **规模**：500-10000+ 人
- **时长**：1-5 天
- **形式**：多展区、主论坛+分论坛、展位展示、Side Events、Workshop 专场
- **核心价值**：行业全景、技术趋势、生态展示、品牌曝光

## 受众定位
- **Primary Audience:** 技术决策者 / Tech Decision Makers - CTO、技术VP、架构师，评估技术方案和合作机会
- **Secondary Audience:** 核心开发者 / Core Developers - 协议核心贡献者、高级工程师，了解行业趋势和技术演进
- **Tertiary Audience:** 技术型投资人 / Technical Investors - 关注技术可行性的投资人，评估技术团队和方案
- **Extended Audience:** 技术媒体与分析师 / Tech Media & Analysts - 报道技术趋势，产出行业分析
- **Audience Behavior:** 受众行为：评估技术方案、寻找合作伙伴、了解竞争格局、建立技术人脉、获取一手信息

### 常用受众组合
- **Enterprise Focus:** 企业关注点：技术决策者 + 技术型投资人，适合 B2B 技术展示
- **Builder Focus:** 构建者关注点：核心开发者 + 技术决策者，适合开发者专区推广
- **Full Spectrum:** 全面覆盖：全受众覆盖，适合大型综合技术展会

## 内容元素
- 展会整体定位与规模说明（参展商数量、预计人流、覆盖赛道）
- 技术趋势全景图（行业技术演进方向、热门赛道解读）
- 核心展区/论坛亮点预告（技术专场、协议展示、基础设施区）
- 重点项目/协议技术解读（明星项目、首发技术、独家内容）
- 开发者专属福利说明（开发者通道、技术资料包）
- 技术社区 Side Event 信息（Builder Night、Protocol Party、Hackathon）
- 嘉宾/演讲者阵容预告（核心开发者、技术领袖、协议创始人）
- 技术社群入口（Discord、Telegram、活动后社群）

## 语言处理
- 宏观视角开场（"Web3 技术版图正在..."、"行业格局迎来..."）
- 趋势判断用数据支撑（"过去一年 TVL 增长 300%"、"开发者数量突破..."）
- 亮点用具体项目/技术名称，避免泛泛而谈
- 规模感营造（"10000+ 参与者"、"500+ 展商"、"50+ 技术论坛"）
- 稀缺性暗示（"年度最大"、"首次亮相"、"独家发布"）
- 避免过度营销语言，保持技术可信度
- 专业术语自然融入，不刻意堆砌
- 信息密度高，每段都有实质内容

## 结构策略
- 开场信息块占比 15%：展会名称 + Tagline + 规模 + 时间地点
- 行业背景块占比 20%：技术趋势 + 行业变化 + 为什么值得关注
- 亮点预告块占比 30%：展区介绍 + 论坛亮点 + 重点项目 + 嘉宾阵容
- 参与价值块占比 25%：适合谁 + 能获得什么 + 专属福利
- 行动号召块占比 10%：报名方式 + 社群入口
- Emoji 密度控制：总量 10-14 个

## 情绪氛围
宏大而充实，定义行业且技术扎实。整体氛围权威且前瞻——捕捉行业转型的规模，同时保持技术可信度。支持创新，支持构建者，反对炒作。

## 注意事项

### 展会文案特殊要点
1. **规模感是核心卖点**：参与人数、展商数量、论坛场次都要具体
2. **信息密度要高**：大展会信息多，文案要帮助读者快速定位
3. **导览价值**：告诉读者怎么逛、看什么、不要错过什么
4. **Side Events 很重要**：很多技术交流发生在主会场之外

### 真实性原则
- 嘉宾未确认的写"部分"或"持续更新"
- 规模数据用"预计"或往届数据
- 具体展商信息需核实后再列出
