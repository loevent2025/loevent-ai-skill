# Web3 Hackathon Event Planning Knowledge Base

## Scenario Overview

A hackathon is a competitive event in the Web3 industry centered on technical innovation and product building. Participants develop projects around specific themes or tech stacks within a fixed timeframe, culminating in demo presentations and winner selection. It is a key channel for developer ecosystem growth, talent discovery, and community engagement.

### Event Characteristics

- **Scale:** 50–500 participants (in-person) / 100–5,000 participants (virtual or hybrid)
- **Duration:** 24–72 hours (concentrated) / 2–4 weeks (distributed)
- **Format:** Coding Marathon + Workshops + Mentor Coaching + Demo Day
- **Core Value:** Technical innovation, talent discovery, ecosystem building, community engagement

### Hackathon Format Comparison

| Format | Duration | Characteristics | Best Use Case |

| Virtual Distributed | 2–4 weeks | Broad reach, low barrier to entry | Global developer participation |
| Hybrid | 1–2 weeks | Online development + in-person Demo Day | Balancing reach and depth |

## Virtual Hackathon Guidelines

### Virtual Event Adaptations

**Virtual 2-Week Hackathon Example**

| Phase | Timeline | Content |
|-------|----------|---------|
| Registration Period | Prior to Week 1 | Open registration, team matching |
| Opening Ceremony | Day 1 | Live virtual kickoff, rules overview (1–2 hours) |
| Workshop Week | Week 1 | One technical workshop per day (live-streamed) |
| Hacking Week | Week 2 | Open development, mentor office hours |
| Submission Deadline | Day 14 | Code + documentation + demo video submission |
| Demo Day | Day 15 | Virtual demo presentations + awards |

### In-Person vs. Virtual Format Comparison

| Session | In-Person | Virtual |
|---------|-----------|---------|
| Opening Ceremony | On-site gathering | Live or recorded broadcast (condensed to 60–90 min) |
| Team Formation | In-room networking | Discord / Telegram team formation channel |
| Workshop | On-site instruction | Live stream + recorded replay |
| Mentor Q&A | On-site office hours | Scheduled video call or async text Q&A |
| Progress Check-in | In-room floor rounds | Regular progress report submissions |
| Demo Presentations | On-stage live demo | Video submission + live virtual presentation |
| Social Event | After Party | Discord voice channel / online celebration |

### Recommended Tools for Virtual Events

| Purpose | Recommended Tools |
|---------|-------------------|
| Registration Management | Luma, Devfolio, DoraHacks |
| Community Communication | Discord (preferred), Telegram |
| Live Streaming | YouTube Live, Twitter Spaces |
| Code Submission | GitHub, Devfolio |
| Video Conferencing | Zoom, Google Meet |
| Project Showcase | Notion, Devpost |

### Virtual Event Best Practices

- **Time Zone Coverage:** Schedule key sessions at staggered times to accommodate global participants
- **Asynchronous Support:** Provide recorded replays and comprehensive documentation
- **Community Engagement:** Assign community managers to maintain active Discord / Telegram channels
- **Integrity Measures:** Conduct code originality checks; require live demos for finalist presentations
- **Contingency Planning:** Prepare screen-recording fallbacks for unstable demo connections

## Judging Framework

### Scoring Dimensions

| Dimension | Suggested Weight | Evaluation Criteria |
|-----------|------------------|---------------------|
| Innovation | 25% | Originality of concept, problem-solving approach |
| Technical Execution | 30% | Code quality, technical complexity, completeness |
| Product Experience | 20% | UI/UX design, user flow, usability |
| Business Potential | 15% | Market demand, business model viability |
| Presentation Quality | 10% | Demo clarity, communication effectiveness |

### Judging Process

| Phase | Content | Participants |
|-------|---------|--------------|
| Initial Screening | Code review, basic functionality verification | Technical mentors |
| Demo Presentations | Live demo + Q&A | All judges |
| Closed-Door Deliberation | Score discussion, final ranking | Judging panel |
| Results Announcement | Awards ceremony | All attendees |

### Judge Profiles

| Role | Profile | Recommended Count |
|------|---------|-------------------|
| Technical Judge | Senior engineer, CTO | 2–3 |
| Product Judge | Product Manager, Founder | 1–2 |
| Investment Judge | Venture capital investor | 1–2 |
| Ecosystem Judge | Sponsor or protocol representative | 1–2 |

|-------------|---------|-------------------|

| Area | Requirements | Notes |
|------|-------------|-------|
| Main Hall | Open workstations, ample power outlets | ~2–3 m² per participant |
| Rest Area | Sofas, bean bags | Essential for overnight events |
| Dining Area | Dedicated space, separate from workspace | Avoid disrupting development flow |

## Risk Management

| Risk Type | Description | Mitigation |
|-----------|-------------|------------|
| Technical Risk | Network outage, API unavailability | Backup network connection, offline development fallback |
| Health Risk | Participant medical emergency | On-site first aid kit, emergency contact list |
| Progress Risk | High rate of incomplete submissions | Lower minimum submission threshold, encourage partial submissions |
| Judging Risk | Judge disagreements or scoring disputes | Clear rubric, multi-round deliberation process |
| Integrity Risk | Code plagiarism, pre-built submissions | Code review, on-site verification during demos |

## Execution Checklist

### Pre-Event Preparation

- Publish technical documentation, SDKs, and APIs in advance
- Open team formation channels ahead of the event
- Announce the mentor roster in advance
- Conduct venue network load testing

### Day-of Support

- Maintain continuous technical support coverage
- Ensure timely catering replenishment
- Send regular progress reminders and countdown updates
- Monitor and maintain a positive event atmosphere

### Post-Event Follow-Up

- Encourage and facilitate open-sourcing of project repositories
- Track and incubate top-performing projects
- Maintain and grow the participant community
- Compile event recap and key performance data
