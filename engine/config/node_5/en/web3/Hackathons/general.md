# Web3 Industry — Hackathon Partner Reference Guide

> Applicable to: Blockchain hackathons, Web3 developer competitions, DeFi/NFT-themed programming marathons, and similar events

---

## Event Profile

| Attribute | Description |
|-----------|-------------|
| Scale | 50–500 developer participants |
| Duration | 24–72 hours (core sprint); extendable to 1–4 weeks (online + in-person) |
| Format | Team development + mentor sessions + Demo showcase + judging & awards |
| Core Value | Developer ecosystem growth, technical talent discovery, innovation incubation, brand thought leadership |
| Budget Level | Mid-range; emphasis on prize pools, venue, and technical infrastructure |

## Benefits Design Reference

### Co-organizer Benefits

**Brand Benefits:**
- Co-equal billing on all event branding
- Priority placement on all event materials
- Joint official announcements

**Content Benefits:**
- Participation in competition structure design
- Authority over technical track direction
- Speaking slots at opening and awards ceremonies
- Seat on the judging committee

**Project Benefits:**
- Priority investment / incubation rights for winning projects
- Full access to all participant project information
- Direct introductions to high-potential teams

### Track Sponsor Benefits

**Brand Benefits:**
- Track naming rights (e.g., "XX DeFi Innovation Track")
- Dedicated track visual identity
- Brand exposure during awards ceremony

**Content Benefits:**
- Primary authority over track judging
- Track direction and criteria setting
- Technical workshops and speaking sessions

**Project Benefits:**
- Priority access to all track projects
- First-mover investment / partnership rights for winning projects

### Technical Partner Benefits

**Brand Benefits:**
- "Technical Partner" designation on all materials
- Branded page in the developer resource package

**Content Benefits:**
- Technical workshops
- On-site technical support presence
- Participation as technical judges

**Resource Benefits:**
- Adoption rate data for products used in participant projects
- Structured developer feedback collection

## Hackathon Format Recommendations

### Track Design Principles
- Optimal range: 3–6 tracks
- Each track should have a defined technical focus and clear judging criteria
- Tracks may overlap; cross-domain innovation is encouraged

### Judging Criteria Reference

| Dimension | Suggested Weight | Description |
|-----------|-----------------|-------------|
| Technical Innovation | 25–30% | Novelty and complexity of the technical approach |
| Product Completeness | 20–25% | Demo usability and feature completeness |
| Business Potential | 20–25% | Market demand and commercial viability |
| Team Capability | 15–20% | Team background and execution ability |
| Ecosystem Fit | 10–15% | Alignment with organizer / track sponsor ecosystem |

## Execution Guidelines

### Priority Sequence
1. **Co-organizers** — Confirm during initial event planning (2–3 months out)
2. **Track Sponsors** — Confirm during competition structure design (1.5–2 months out)
3. **Technical Partners** — Confirm before developer recruitment launches (1–1.5 months out)
4. **Judges / Mentors** — Finalize 1 month before the event
5. **Media / Community Partners** — Activate in parallel with recruitment launch

### Key Considerations
- Track design requires deep alignment with sponsors to ensure direction consistency
- Technical partners must prepare developer documentation and training materials in advance
- Judging criteria must be published ahead of time to ensure fairness and transparency
- Post-event incubation pathways for outstanding projects should be planned early
- Online hackathons require a stable technical platform with contingency support
- In-person hackathons must account for logistics: venue, catering, and rest areas
- Intellectual property ownership must be clearly defined in the competition rules
