# Web3 Technical Workshop — Technical Events Knowledge Base

## 📌 Module Activation Criteria

Apply this module when the event theme involves the following keywords:
- Smart contract development workshop, protocol technical workshop
- ZK / Layer 2 / Cross-chain hands-on lab, security audit training
- SDK / API practical training, developer tooling deep-dive session

---

## 🎯 Scenario Overview

| Dimension | Description |
|-----------|-------------|
| Format | Instruction + Live coding + Mentor support + Q&A interaction |
| Duration | 2–8 hours, intensive hands-on practice |
| Participants | Developers, technical mentors, protocol technical teams |
| Core Objective | Skill acquisition, hands-on mastery, problem-solving, technical exchange |

---

## 👥 Audience Profiles (by Priority)

| Priority | Audience Type | Key Focus |
|----------|---------------|-----------|
| P0 | Protocol Developers | Learning new technologies, best practices, performance optimization |
| P1 | Application Developers | Integration solutions, SDK usage, practical techniques |
| P2 | Technical Architects | Architecture design, technology selection, security considerations |
| P3 | Technical Learners | Introductory hands-on practice, foundational skills, project-based learning |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Focus |
|------------|-------------------|------------|
| Official Protocol Technical Training | Formal & Professional (A) | Technical authority, official certification |
| Developer Hands-On Workshop | Energetic & Engaging (B) | Learning by doing, interactive experience |
| Advanced Technical Deep Dive | Suspense-Driven (C) | Technical challenges, solution discovery |
| Tool / SDK Training | Value-Driven (D) | Productivity gains, resource access |

---

## 📝 Required Content Elements

### 1. Technical Value (at least 2 instances)
- "Master the core usage of [technology/tool] in depth"
- "Taught directly by the [protocol/project] official team"

### 2. Hands-On Design (at least 1 instance)
- "Code along the entire session — learn by building"
- "Complete a working [project type] by the end"

### 3. Mentor Support (at least 1 instance)
- "[Number]+ mentors providing on-site guidance"
- "1-on-1 Q&A to solve your specific issues"

### 4. Compliance Disclaimer (mandatory)
- "⚠️ This event is a technical training session and does not constitute investment advice (NFA)"
