# AI Agent & LLM Industry — Hackathon Partnership Reference Guide

> Applicable to: AI application hackathons, Agent development competitions, LLM innovation marathons, prompt engineering challenges, and similar events

---

## Event Profile

| Attribute | Description |
|-----------|-------------|
| Scale | 50–500 participating developers |
| Duration | 24–72 hours (core competition), extendable to 1–4 weeks (online + offline) |

## Hackathon Competition Format Recommendations

### Track Design Principles
- 3–6 tracks is ideal
- Each track should have clearly defined use cases and judging criteria
- Cross-track overlap is acceptable; creative combinations should be encouraged

### Judging Criteria Reference

| Dimension | Suggested Weight | Description |
|-----------|-----------------|-------------|
| Innovation | 25–30% | Novelty of the proposed solution |
| Technical Implementation | 20–25% | Proficiency in AI technology application |
| Product Completeness | 20–25% | Demo usability and feature completeness |
| Commercial Potential | 15–20% | Market demand and business viability |
| User Experience | 10–15% | Product usability and overall experience |

### Technical Support Arrangements
- **Pre-event Training**: API usage, framework introductions, best practices
- **On-site Mentors**: Technical Q&A, directional guidance
- **Online Support**: Documentation, FAQ, technical support channels

## Execution Priorities

### Timeline
1. **Co-host** — Confirmed during initial planning phase (2–3 months before the event)
2. **Track Naming Sponsors** — Confirmed during competition format design (1.5–2 months before)
3. **Technical Partners** — Confirmed before developer recruitment begins (1–1.5 months before)
4. **Judges / Mentors** — Confirmed 1 month before the event
5. **Media / Community Partners** — Launched in parallel with recruitment kickoff

### Key Considerations
- API credits must be prepared in advance to ensure sufficient supply
- Track design requires in-depth alignment with naming sponsors to ensure directional consistency
- Technical partners must prepare developer documentation and training materials ahead of time
- Judging criteria must be published in advance to ensure fairness and transparency
- Post-event incubation pathways for top projects should be planned ahead
- Online hackathons require a stable and reliable technical platform
- On-site hackathons must account for venue, catering, and network infrastructure
- Intellectual property ownership must be clearly stated in the competition rules
- API call costs must be estimated and managed in advance
