# 黑客松 × 通用类（AI/Model 商业化版）

## 适用场景
AI 应用开发黑客松、LLM 应用创新大赛、AI Agent 开发马拉松、大模型应用挑战赛、AI 产品创意黑客松、AI 工具开发竞赛

## 场景特征
- **形式**：限时开发、团队协作、Demo 展示、评审答辩
- **核心价值**：快速验证想法、技术实践、展示能力、团队组建、获得资源

## 受众定位
- **主要受众**：AI 开发者——想动手做 AI 项目的工程师
- **次要受众**：AI 产品建设者——有想法想快速验证的产品人
- **第三受众**：AI 创业者——想组建团队或验证方向的创始人
- **延伸受众**：AI 学习者——想通过实战学习的新手
- **受众行为**：动手实践、验证想法、展示能力、组建团队、获取资源

### 常用受众组合
- **Builder 核心**：开发者 + 产品建设者，适合技术导向黑客松
- **创业导向**：开发者 + 创业者，适合产品化导向黑客松
- **学习实践**：开发者 + 学习者，适合入门友好型黑客松

## 内容元素
- 黑客松主题与方向
- 赛道设置
- 评审标准
- 技术资源支持
- 导师阵容
- 奖励说明（泛化处理）
- 时间安排
- 报名要求

## 语言处理
- Builder 文化氛围
- 技术术语保持英文（Agent、RAG、LLM）
- 强调"动手做"而非"听讲"
- 赛道方向要具体
- 奖励泛化表述（如有奖励但无具体信息时）
- 资源支持是卖点

## 结构策略
- 开场信息块占比 15%
- 主题赛道块占比 30%
- 资源支持块占比 25%
- 参与价值块占比 20%
- 行动号召块占比 10%
- Emoji 密度控制：总量 10-14 个

## 情绪氛围
Builder 热情、快速行动。"想法不值钱，做出来才算"的实干感。

## 风格变体

### 变体A：综合创新型
- **适用**：多赛道综合黑客松
- **开场格式**：🚀 {黑客松名} | 把你的 AI 想法做出来
- **语气**：开放、创新、Builder 文化

### 变体B：主题聚焦型
- **适用**：特定主题/方向黑客松
- **开场格式**：🔧 {主题} Hackathon | {核心挑战}
- **语气**：聚焦、深度、问题导向

### 变体C：生态共建型
- **适用**：平台/协议生态黑客松
- **开场格式**：⚡ {生态名} Builder Hackathon | 一起建设生态
- **语气**：生态、共建、资源丰富

## 背景痛点模板

### AI Agent 方向
**趋势描述：**
AI Agent 是当前最热门的方向之一，但大多数 Agent 还停留在 Demo 阶段。我们想看到真正能用的 Agent 应用。

**挑战问题：**
❓ 能不能做出一个真正解决具体问题的 Agent？
❓ 如何让 Agent 在复杂场景下保持可靠？
❓ Agent 的用户体验怎么设计？

### LLM 应用方向
**趋势描述：**
LLM 能力越来越强，但好的应用还是稀缺。我们想看到不只是"套壳"的 LLM 应用创新。

**挑战问题：**
❓ 除了聊天，LLM 还能做什么有价值的应用？
❓ 如何设计让用户持续使用的 AI 产品？
❓ 怎么在有限的成本内做出好体验？

## 注意事项

### 黑客松要点
1. **主题要清晰**：让参赛者知道做什么方向
2. **资源支持是卖点**：API Credits、导师、技术支持
3. **评审标准明确**：让参赛者知道怎么评
4. **奖励泛化处理**：如无具体信息，用"丰厚奖励"表述
5. **强调动手**：不是 PPT 大赛，是 Demo 大赛
