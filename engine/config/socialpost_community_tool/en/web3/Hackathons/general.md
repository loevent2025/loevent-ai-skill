# Web3 Hackathon — General Event Knowledge Base

## 📌 Module Activation Conditions

Apply this module when the event is a comprehensive hackathon covering multiple tracks or themes:
- Comprehensive Web3 Hackathons, Multi-track coding marathons
- City/regional hackathons, Annual Builder challenges
- Multi-host joint hackathons

---

## 🎯 Scenario Overview

| Dimension | Description |
|-----------|-------------|
| Format | Multi-track parallel competition + Mentor support + Demo Day + Awards ceremony |
| Duration | 48–72 hours on-site, or 2–4 weeks online |
| Participants | Developers, Founders, Investors, Project teams, Mentors |
| Core Objectives | Ecosystem building, Talent discovery, Project incubation, Community engagement |

---

## 👥 Audience Profiles (by Priority)

| Priority | Audience Type | Key Focus |
|----------|---------------|-----------|
| P0 | Full-stack developers | Technical challenges, Prize pool, Project opportunities |
| P1 | Startup teams | Deal sourcing, Incubation resources, Exposure |
| P2 | Students / Newcomers | Hands-on learning, Industry onboarding, Networking |
| P3 | Investors / Project teams | Talent discovery, Project screening, Ecosystem partnerships |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Focus |
|------------|-------------------|------------|
| Large-scale comprehensive hackathon | Formal Professional (A) | Scale impact, Multi-track diversity |
| City Builder challenge | Energetic & Engaging (B) | Community atmosphere, Participant experience |
| High-prize / Resource-rich competition | Benefits-Driven (D) | Prize pool, Incubation resources |
| Themed innovation challenge | Suspense-Driven (C) | Innovation direction, Problem-solving |

## ❓ FAQ Templates

| Question | Answer |
|----------|--------|
| Can I enter multiple tracks? | Each project can only select one primary track |
| Can I participate online? | This edition is on-site only — in-person attendance required |
| Can I enter with an existing project? | Yes, but new development during the competition period is required |
| How are prizes distributed? | Within 30 days of judging — USDT supported |

---

## ⚠️ Prohibited Actions

- ❌ Overstating prizes or investment commitments
- ❌ Implying token launch opportunities
- ❌ Plagiarizing or reusing existing code
- ❌ Misrepresenting the judging panel or mentor lineup
