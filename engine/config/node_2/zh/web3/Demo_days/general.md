# Web3 Demo Day 活动策划知识库

## 场景概述

Demo Day是Web3行业中以项目展示和投资对接为核心的路演型活动，项目方在限定时间内向投资人、合作伙伴和社区展示产品和商业计划，是项目融资、曝光和生态合作的重要场景。

### 活动特征

- **规模**：50-300人
- **周期**：半天至1天
- **形式**：项目路演 + 评委点评 + 投资配对 + 社交环节
- **核心价值**：融资对接、项目曝光、行业交流、生态合作

---

## 活动类型与选择

### Demo Day类型对比

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| 加速器Demo Day | 毕业项目展示、投资人定向邀请 | 加速器结业、批次毕业 |
| 黑客松Demo Day | 赛事成果展示、评审颁奖 | 黑客松收尾 |
| 开放式Demo Day | 公开报名、多元项目 | 行业活动、生态展示 |
| 主题Demo Day | 特定赛道聚焦 | DeFi/NFT/基础设施等垂直领域 |
| 闭门Demo Day | 仅限投资人参与 | 高质量项目定向对接 |

### 项目筛选标准

| 维度 | 筛选要点 |
|------|----------|
| 团队背景 | 创始人履历、技术能力、行业经验 |
| 产品进度 | MVP/测试网/主网、用户数据 |
| 市场定位 | 赛道选择、竞争分析、差异化 |
| 融资阶段 | 种子轮/Pre-A/A轮等 |
| 展示能力 | 表达清晰、Demo可演示 |

## 线上Demo Day补充指南

### 线上活动流程调整

| 环节 | 线下版本 | 线上版本 |
|------|----------|----------|
| 签到 | 现场签到 | 提前10分钟进入直播间 |
| 路演展示 | 现场演讲+投屏 | 视频直播/预录视频+实时Q&A |
| 评委互动 | 现场提问 | 弹幕/Q&A面板提问 |
| 投资配对 | 现场1对1 | 预约视频会议/会后邮件对接 |
| 社交环节 | 晚宴/酒会 | Discord语音房/线上Networking |

### 线上活动时长调整

- 路演时间缩短至8-10分钟/项目
- Q&A时间保持3-5分钟
- 整体活动时长控制在3小时内
- 安排中场休息（10-15分钟）

### 线上活动工具

| 用途 | 推荐工具 |
|------|----------|
| 直播平台 | YouTube Live、Twitter Space、Zoom Webinar |
| 互动工具 | Slido（Q&A/投票）、Discord |
| 项目展示 | Notion页面、Pitch Deck链接 |
| 配对系统 | Calendly、Luma 1:1 |

## 路演设计规范

### 单个项目路演结构

**时长分配（12-15分钟）**

| 环节 | 时长 | 内容 |
|------|------|------|
| 项目介绍 | 8-10min | Pitch展示 |
| Live Demo | 2-3min | 产品演示（可选） |
| Q&A | 3-5min | 评委提问 |

### Pitch内容框架

| 板块 | 内容要点 | 建议页数 |
|------|----------|----------|
| 开场 | 一句话定位、核心价值 | 1页 |
| 问题 | 痛点描述、市场机会 | 1-2页 |
| 解决方案 | 产品介绍、技术架构 | 2-3页 |
| Demo | 产品演示/截图 | 1-2页 |
| 市场 | 市场规模、目标用户 | 1页 |
| 商业模式 | 盈利方式、收入来源 | 1页 |
| 竞争分析 | 竞品对比、差异化 | 1页 |
| Traction | 数据指标、里程碑 | 1页 |
| 团队 | 核心成员、背景 | 1页 |
| 融资需求 | 融资金额、资金用途 | 1页 |
| 联系方式 | 社交媒体、邮箱 | 1页 |

### 路演注意事项

- **严格控时**：超时会被打断
- **Demo优先**：有可演示产品比PPT更有说服力
- **数据说话**：用户数、TVL、交易量等
- **讲故事**：有记忆点的叙事
- **准备Q&A**：预判常见问题

## 投资配对设计

### 配对机制

| 阶段 | 内容 | 说明 |
|------|------|------|
| 需求收集 | 活动前收集双方需求 | 项目方：融资阶段、金额；投资人：关注赛道 |
| 智能匹配 | 按需求匹配推荐 | 优先匹配度高的组合 |
| 预约系统 | 双向选择、确认时间 | 提供在线预约工具 |
| 现场配对 | 1对1洽谈 | 每场15-20分钟 |

### 配对区设置

| 区域 | 数量 | 配置 |
|------|------|------|
| 1对1洽谈桌 | 10-20个 | 小圆桌+2椅 |
| 小组讨论区 | 3-5个 | 沙发区 |
| 开放交流区 | 1个 | 站立交流 |

### 配对效率提升

- 提前发送项目材料给投资人
- 提供投资人背景给项目方
- 设置配对提醒
- 安排工作人员引导

## 目标人群分析

### 受众分级

| 层级 | 人群 | 占比 | 核心动机 |
|------|------|------|----------|
| 核心受众 | 投资人、投资机构 | 40% | 发现投资标的、行业洞察 |
| 核心受众 | 路演项目方 | 30% | 融资、曝光、合作 |
| 次要受众 | 潜在合作方、服务商 | 20% | 商务合作、客户获取 |
| 扩展受众 | 媒体、观察者 | 10% | 内容产出、行业了解 |

### 投资人画像

- **关注阶段**：种子轮/Pre-A/A轮
- **关注赛道**：DeFi/NFT/基础设施/GameFi等
- **决策周期**：快速判断、后续DD
- **期望收获**：优质项目、行业洞察

### 项目方画像

- **融资需求**：明确金额和估值预期
- **准备程度**：产品可演示、数据可展示
- **目标多元**：融资、曝光、合作、招聘
- **后续需求**：持续跟进、资源对接
