## Demo Day Format Recommendations

### Pitch Session Design
| Segment | Suggested Duration | Description |
|---------|-------------------|-------------|
| Project Pitch | 5–8 minutes per project | Product demo + Business model |
| Investor Commentary | 3–5 minutes per project | 2–3 judges rotating commentary |
| Q&A | 3–5 minutes per project | Audience and judge questions |
| Changeover | 2 minutes | Transition and setup for next project |

### Project Category Guide
| Category | Recommended Showcase Content |
|----------|------------------------------|
| AI Application Layer | Product demo, User metrics, Business model |
| AI Tooling Layer | Technical architecture, Performance benchmarks, Developer ecosystem |
| AI Infrastructure | Technical advantages, Cost efficiency, Customer case studies |

### Supporting Segments
- **AI Sector Insights Keynote**: 1–2 presentations (15–20 minutes each)
- **Technical Deep Dive**: In-depth exchange between technical experts and project teams
- **1-on-1 Meetings**: Appointment-based, 10–15 minutes per session
- **Networking Break / Lunch**: Informal conversation time
- **After Party**: Evening in-depth networking

## Execution Priorities

### Timeline & Sequencing
1. **Co-Host** — Confirm during event planning phase (2–3 months before the event)
2. **Investor Invitations** — Begin outreach 1.5–2 months before the event
3. **Project Selection** — Complete 1–1.5 months before the event
4. **Strategic Partners** — Confirm 1 month before the event
5. **Service Providers** — Finalize 2–3 weeks before the event

### Key Considerations
- Project selection criteria must be deeply aligned with the co-host
- Investor invitations should be targeted to avoid homogeneity
- Pitch order should ensure diversity across verticals
- 1-on-1 meeting preferences should be collected from both sides in advance
- Project material confidentiality terms must be agreed upon beforehand
- An investment intent tracking mechanism should be established in advance
- Media coverage scope must be confirmed with project teams
- Demo environment must be tested in advance (network, devices, APIs)
- The after party is a critical networking opportunity and should be thoughtfully planned
