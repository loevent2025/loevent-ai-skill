# Web3 路演日知识库

## 一、场景定义与特点

### 场景概述
路演日（Demo Day）是Web3行业中项目与资本对接的核心场景，通过结构化的项目展示为创业者提供曝光机会，为投资人提供项目发现渠道，是加速器、孵化器和投资机构的标志性活动。

### 核心特征
| 特征 | 描述 |
|------|------|
| 规模 | 50-300人，以投资人和项目方为主 |
| 周期 | 半天至1天，紧凑高效 |
| 形式 | 项目Pitch+投资人点评+一对一对接 |
| 定位 | 融资对接、项目曝光、投资发现 |

### 典型活动形式
- 加速器毕业路演（Y Combinator、a]Alliance Demo Day）
- 投资机构项目日（VC Portfolio Day）
- 黑客松决赛路演
- 垂直赛道项目路演（DeFi、GameFi、AI+Crypto等）
- 区域性创业路演

## 二、目标人群

### 核心参会群体
| 人群类型 | 占比参考 | 核心诉求 |
|----------|----------|----------|
| 投资人/VC | 35-50% | 项目发掘、投资机会、行业研判 |
| 项目创始人 | 25-35% | 融资对接、曝光机会、同行交流 |
| 企业/机构代表 | 10-15% | 合作机会、生态投资、市场洞察 |
| 媒体/分析师 | 5-10% | 项目报道、行业分析、内容生产 |
| 行业从业者 | 5-10% | 趋势了解、求职机会、社交拓展 |

### 投资人画像
- 机构类型：头部VC、CVC、天使基金、家族办公室
- 投资阶段：种子轮、Pre-A、A轮为主
- 决策权限：合伙人、投资总监、投资经理
- 关注赛道：与路演主题匹配

## 五、关键成功要素

### 项目筛选
- **质量把控**：严格筛选标准，确保项目质量
- **多元覆盖**：不同赛道、阶段的合理组合
- **信息披露**：提前向投资人同步项目信息
- **排序设计**：根据质量和热度安排出场顺序

### Pitch培训
- **内容辅导**：商业模式清晰度、数据呈现、差异化表达
- **表达训练**：时间控制、节奏把握、Q&A应对
- **材料优化**：Deck设计、Demo准备、补充材料
- **模拟演练**：多轮模拟Pitch和反馈迭代

### 对接机制
- **精准匹配**：基于赛道、阶段、偏好的投资人-项目匹配
- **预约系统**：一对一对接预约、时间管理
- **对接空间**：独立会议室、安静对话区域
- **信息同步**：项目材料提前共享、联系方式交换

### 后续跟进
- **对接数据**：收集整理对接情况、投资意向
- **持续撮合**：活动后继续推进对接
- **进度追踪**：跟踪融资进展、成功案例包装
- **投资人维护**：定期项目推送、关系维护

## 六、注意事项

### 项目保护
- 签署保密协议，保护项目核心信息
- 控制材料传播范围
- 尊重项目融资进度和信息披露意愿

### 投资人体验
- 避免信息过载，控制路演数量
- 提供充足的一对一对接时间
- 高效的会议安排和时间管理

### 公平性
- 透明的项目筛选标准
- 平等的曝光机会
- 避免举办方利益冲突

### 信息真实性
- 核实项目信息准确性
- 避免虚假宣传和夸大
- 建立信用记录和反馈机制
