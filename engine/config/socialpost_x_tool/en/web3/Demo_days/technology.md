# DEMO DAYS × TECHNICAL

## Applicable Scenarios
Technical demo days, protocol showcase events, infrastructure project presentations, developer tool demos, hackathon demo sessions, technical product launches

## Scenario Characteristics
- **Format:** Technical demos, architecture walkthroughs, live demos, technical Q&A
- **Core Value:** Technical showcase, architecture feedback, developer adoption, technical credibility

## Audience Profile
- **Primary Audience:** Technical Reviewers — CTOs and architects evaluating technical solutions
- **Secondary Audience:** Developers — exploring new tools and finding technical solutions
- **Tertiary Audience:** Technical Investors — assessing technical feasibility
- **Extended Audience:** Protocol Teams — seeking technical partnerships
- **Audience Behavior:** Evaluating technical solutions, learning new tools, finding collaborators, exchanging ideas

### Common Audience Combinations
- **Builder Focus:** Developers + protocol teams — ideal for tool and infrastructure demos
- **Evaluation Focus:** Technical reviewers + investors — ideal for in-depth technical showcases

## Content Elements
- Technical theme and positioning
- Technical direction of presenting projects
- Technical reviewer lineup
- Demo format (live demo, code walkthrough)
- Technical Q&A session
- Technical community entry points

## Language Guidelines
- Technical and architecture-first — lead with implementation, not vision
- Live demo is the core differentiator
- Reviewers prioritize technical depth
- Let code and data do the talking
- Avoid over-marketing language

## Content Structure
- Opening info block: 15%
- Technical projects block: 35% — project direction + technical highlights
- Reviewer lineup block: 20% — technical backgrounds
- Value proposition block: 20%
- Call to action block: 10%
- Emoji density: 8–10 total

## Tone & Atmosphere
Technically rigorous, demo-focused. "Show me the code" energy.

## Style Variants

### Variant A: Tool Showcase
- **Best for:** Developer tools, SDKs, frameworks
- **Opening format:** 🔧 {Topic} Tool Demo Day | Developer Tools in the Spotlight
- **Tone:** Practical, developer-oriented

### Variant B: Protocol Technical Showcase
- **Best for:** Protocol solutions and architecture presentations
- **Opening format:** ⚡ {Topic} Technical Showcase | See How They Build
- **Tone:** In-depth, architecture-driven

## Copy Snippets
- No slide decks. Just live demos.
- Code that runs is the only proof that matters.
- The technical details are what count.
- Architecture diagrams + performance data — everything you need at a glance.

## Notes

### Key Principles for Technical Demo Days
1. **Live Demo Is the Core:** This is not a slide presentation — it's about seeing something that runs
2. **Reviewer Lineup Determines Quality:** The technical depth of feedback depends on who's reviewing
3. **Show the Technical Details:** Architecture, code, and data — all three
4. **Developer Perspective First:** Always answer the question — what does this do for developers?
