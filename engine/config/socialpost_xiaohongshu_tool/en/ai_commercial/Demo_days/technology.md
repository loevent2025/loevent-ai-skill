# AI DEMO DAY × TECHNICAL

## Applicable Scenarios
AI technical project showcases, open-source project demo days, AI tool exhibitions, hackathon result showcases, technical product launches

## Scenario Characteristics
- **Format:** Technical demos + live code walkthroughs + technical Q&A + developer exchange
- **Core Value:** Technical showcase, solution evaluation, developer adoption, technical credibility

## Audience Profile
- **Primary Audience:** AI developers — discovering new tools, learning technical solutions
- **Secondary Audience:** Tech leads / CTOs — evaluating technology, finding solutions to real problems
- **Tertiary Audience:** Technical investors — assessing technical feasibility
- **Extended Audience:** Open-source community — discovering projects, contributing
- **Audience Behavior:** Evaluating technology, learning approaches, finding tools, exchanging technical insights

### Common Audience Combinations
- **Developer Focus:** Developers + open-source community — ideal for tool showcase events
- **Evaluation Focus:** Tech leads + investors — ideal for in-depth technical showcases

## Content Elements
- Technical demo day theme and positioning
- Presenting project technical directions
- Technical reviewer lineup
- Live demo format (code walkthroughs, architecture deep dives)
- Technical Q&A session
- Open-source and documentation resources

## Language Guidelines
- Technically grounded, but accessible and direct
- Emphasize "code that actually runs" and "live demo"
- Technical details are the differentiator
- Developer community atmosphere
- Practical over impressive — solve real problems

## Content Structure
- Opening info block: 15%
- Technical projects block: 35% — project direction + technical highlights
- Reviewer lineup block: 20% — technical backgrounds
- Value proposition block: 20% — what you'll learn
- Call to action block: 10%
- Emoji density: 8–12 total, clean and technical

## Tone & Atmosphere
Technically rigorous, demo-driven. "Show me the code" builder energy.

## Attendee Tips

### For Developers
- Come with your own technical problems in mind
- Focus on the projects most relevant to your work
- Go deep on the technical details after the demo
- Connect with the project maintainers directly

### For Tech Leads
- Focus on projects that solve real, practical problems
- Evaluate technical maturity and maintainability
- Understand how each solution differs from what you're already using
- Consider potential for integration or pilot testing

## Notes

### Key Principles for Technical Demo Days
1. **Live Demo Is Non-Negotiable:** Code must run in real time, on stage
2. **Technical Depth Is Expected:** Architecture, performance data, and code all matter
3. **Reviewers Must Understand the Tech:** They need to ask the right questions
4. **Developer Perspective First:** Always answer — what problem does this actually solve?
5. **Resources Must Be Provided:** GitHub links, documentation, and community access
