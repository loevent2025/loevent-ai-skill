## 📝 Required Content Elements

### 1. Contrast & Tension Building (mandatory)

**Formula**:
```
"What happens when [architecture/solution representing Tech System X] meets [capability/paradigm representing Tech System Y]?"
```

**Guidelines**:
- Clearly articulate the differences and respective strengths of each tech stack
- Use contrast phrasing: "X represents…, Y represents…"
- Emphasize the innovation and unique value of this particular tech combination

---

### 2. Chemical Reaction Description (mandatory)

**Dynamic verbs**:
- ✅ Use: "collide," "converge," "catalyze," "spark," "reshape," "bridge"
- ❌ Avoid: "cooperate," "combine," "join," "connect"

**Tech fusion formula**:
```
[Traditional tech strength] × [Web3-native capability] = [Unique technical outcome]
```

**Guidelines**:
- Describe what problems cross-stack fusion solves that neither stack can address alone
- Explain why this specific tech combination creates irreplaceable value

---

### 3. Participant Collision Description (mandatory)

**Technical background collision**:
```
"Here, enterprise architects seek to validate the engineering feasibility of on-chain solutions, while Web3 developers seek to absorb enterprise-grade architecture thinking"
```

**Reviewer perspective collision**:
```
"Enterprise tech experts evaluate from a stability and maintainability lens; protocol core developers review through a decentralization and security lens"
```

---

### 4. Scene & Atmosphere Building (select at least 2)

**a) Vivid Scene Setting**
- Don't write: "Projects will deliver technical presentations"
- Do write: Concretely describe developers from different tech backgrounds gathered around the same demo, debating
- Convey the tension of two technical mindsets colliding over a single solution

**b) Flow of Time**
- Describe key breakthrough moments during the tech fusion process
- Convey the cognitive shift from "Can this actually work?" to "So that's how it's done"

**c) Dynamic Interaction**
- Don't write: "Attendees can interact with project teams on-site"
- Do write: Concretely describe developers from different backgrounds debating and exploring the same technical solution

**d) Emotional Energy**
- Demo Day: discovery, surprise, breakthrough, validation
- Tech discussions: debate, collision, realization, recognition
- Networking: curiosity, deep-diving, connecting, continuation

---

### 5. Unique Tech Fusion Value Proposition (mandatory)

**Describe outcomes only possible through cross-discipline tech fusion**:
- List solutions that only this specific tech combination can deliver
- Explain why both traditional tech teams and Web3 teams should pay attention
- Emphasize the irreplaceability of the fusion approach

---

### 6. Transformation Arc (recommended)

**Technical cognition journey**:
- **Start**: Arrive with [curiosity about / skepticism toward the other tech paradigm]
- **Process**: Witness [a specific demo], hear [a specific solution], have [a technical assumption] challenged
- **End**: Leave with [a new technical perspective / fusion approach / collaboration possibilities]

---

### 7. Compliance Disclaimer (mandatory)

```
"⚠️ Project demos do not constitute investment advice (NFA)"
```

## ⚠️ Prohibited Practices

- ❌ Disparaging either tech paradigm ("traditional tech is outdated" or "Web3 lacks engineering rigor")
- ❌ Framing project demos as investment recommendations
- ❌ Overstating the maturity or production-readiness of tech fusion solutions
- ❌ Disclosing unreleased technical solutions or project information
- ❌ Making promises about project roadmaps or future development
- ❌ Ignoring the cross-discipline fusion value and writing generic Demo Day copy
- ❌ Using passive language like "cooperate" or "combine" in place of high-energy collision language
- ❌ Over-hyping cross-discipline concepts without substantive technical depth to back them up
