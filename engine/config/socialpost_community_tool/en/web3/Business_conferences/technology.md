# Web3 B2B Meeting — Technical Events Knowledge Base

## 📌 Module Activation Criteria

Apply this module when the event theme involves the following keywords:
- Technical executive sessions, protocol workshops, architecture review meetings
- CTO roundtables, technical strategy meetings, security audit workshops
- Enterprise blockchain solution meetings, technical partnership discussions

---

## 🎯 Scenario Overview

| Dimension | Description |
|-----------|-------------|
| Format | Executive sessions + Panel discussions + Technical workshops + Solution reviews |
| Content Depth | Technical decision-making, architecture selection, partnership evaluation, strategic planning |
| Participants | CTOs / Technical executives, Protocol core teams, Enterprise technology departments |
| Core Objective | Establish technical partnerships, finalize solutions, achieve strategic alignment |

---

## 👥 Audience Profiles (by Priority)

| Priority | Audience Type | Key Focus |
|----------|---------------|-----------|
| P0 | CTO / VP of Engineering | Technical strategy, architecture decisions, team planning |
| P1 | Protocol Core Developers | Technical partnerships, integration solutions, ecosystem collaboration |
| P2 | Enterprise Technology Leaders | Blockchain implementation, technology selection, cost assessment |
| P3 | Technical Investors | Technical due diligence, team evaluation, feasibility analysis |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Focus |
|------------|-------------------|------------|
| CTO Executive Roundtable | Formal & Professional (A) | Strategic discussion, high-level dialogue |
| Technical Partnership Meeting | Formal & Professional (A) | Solution evaluation, partnership advancement |
| Architecture Workshop | Suspense-Driven (C) | Technical challenges, solution discovery |
| Technical Review Session | Value-Driven (D) | Expert resources, review opportunities |

---

## 📝 Required Content Elements

### 1. Professional Positioning (at least 1 instance)
- "Invitation-only technical workshop exclusively for [role/background]"
- "In-depth dialogue at the CTO / VP of Engineering level"

### 2. Topic Value (at least 2 instances)
- "Focused on solutions for [specific technical challenge]"
- "Exploring strategic options for [technical direction]"

### 3. Outcome-Oriented (at least 1 instance)
- "Post-event technical assessment report to be shared"
- "Facilitate technical partnership intent"

### 4. Compliance Disclaimer (mandatory)
- "⚠️ This event is for technical exchange purposes only and does not constitute investment advice (NFA)"

## ❓ FAQ Templates

| Question | Answer |
|----------|--------|
| What are the eligibility requirements? | Open to CTO / VP of Engineering / Technical Co-Founder level only |
| Is the session confidential? | Chatham House Rule applies — content is not to be shared externally |
| Can I bring a team member? | In principle, one representative per company; exceptions may be requested |
| Will materials be shared afterward? | A redacted session summary will be distributed to attendees |

---

## ⚠️ Prohibited Practices

- ❌ Publicly disclosing the identities of attendees
- ❌ Sharing content discussed during executive sessions
- ❌ Framing B2B meetings as investment opportunities
- ❌ Exaggerating speakers' or attendees' credentials
