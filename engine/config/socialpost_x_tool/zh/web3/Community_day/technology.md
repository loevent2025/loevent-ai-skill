# 咖啡馆共建 / Cafe Coworking × 技术类

## 适用场景
开发者咖啡馆、Code & Coffee、Hackathon 前热身、协议开发者工作日、开源贡献者共建、技术工具用户共建（Cursor/Figma 等）

## 场景特征
- **形式**：开发者在咖啡馆一起写代码/工作、互相帮助、可能有简短技术分享
- **时长**：3-6 小时
- **核心价值**：专注写代码的氛围、技术互助、找到技术伙伴

## 受众定位
- **Primary Audience:** 开发者——想找个地方专注写代码
- **Secondary Audience:** 技术学习者——在学编程想找人问问题
- **Tertiary Audience:** 技术创业者——有技术背景的创始人
- **Audience Behavior:** 写代码、debug、问技术问题、技术交流

## 内容元素
- 场地说明（WiFi、插座、环境）
- 技术方向/主题（可选）
- Intro Circle（技术版：在做什么/卡在哪里/能帮什么）
- 可能的技术分享/workshop（可选）
- All skill levels welcome
- 技术社群连接（Discord 等）

## 语言处理
- ✅ 技术人的口语
- ✅ "带电脑来写代码"、"卡住了问旁边的人"
- ✅ All skill levels welcome
- ✅ 技术互助氛围
- ❌ 不要太正式
- ❌ 避免"高手云集"、"大牛"

## 结构策略
- 场景开场块占比 15%
- 技术方向块占比 20%（如果有特定方向）
- 活动形式块占比 25%
- 互助机制块占比 20%
- 环境说明块占比 10%
- 行动号召块占比 10%
- Emoji 密度控制：总量 8-12 个

## 情绪氛围
专注写代码、技术互助、同行陪伴。"一起写代码比一个人写代码有意思"的感觉。

## 风格变体

### 变体A：通用 Code & Coffee
- **适用：** 面向所有开发者
- **开场格式：** ☕💻 Code & Coffee | 带电脑来写代码
- **语气：** 开放、inclusive、all levels

### 变体B：工具/产品用户型 / Tool Cafe
- **适用：** 特定工具用户（Cursor、Figma、VS Code 等）
- **开场格式：** ☕🔧 Cafe [工具名] | [工具] 用户一起 build
- **语气：** 产品社区、用户连接、可能有团队在场

### 变体C：技术主题型 / Tech Focus
- **适用：** 特定技术方向（Solidity、Rust、AI 等）
- **开场格式：** ☕⚡ [技术] 开发者咖啡馆 | 写 [技术] 的一起来
- **语气：** 聚焦、同方向、技术深度

## Intro Circle 技术版模板

### 标准版
```
🗣️ 开场 Intro（2:30pm）：
- 你是谁
- 今天在写什么 / 做什么项目
- 卡在哪里 / 需要什么帮助
- 能帮别人什么
（之后自由活动）
```

### 简化版
```
🗣️ 简单说一下：
- 在做什么
- 需要帮助吗
（30 秒就行，不用太详细）
```

## 技术分享模板（可选）

### 闪电分享
```
⚡ 闪电分享（可选参加）：
- 15:00 [分享者]：[主题]（15 分钟）
- 不参加也完全没问题
- 分享的时候其他人可以继续工作
```

### Workshop
```
🔧 Workshop（可选）：
- 11:00 [主题] Workshop（45 分钟）
- 适合：[适合谁]
- 不参加的可以继续自己工作
```

## 注意事项

### 技术类咖啡馆共建要点

1. **All skill levels welcome 要强调**
   - 新手也能来
   - 不是比谁技术好
   - 互相帮助的氛围

2. **技术互助是特色**
   - Intro Circle 问"卡在哪里"
   - 鼓励互相帮助
   - 有经验的可以带新手

3. **环境要求要说清**
   - WiFi 稳定性
   - 插座数量
   - 是否安静

4. **工具/产品用户型有额外福利**
   - 产品 credits
   - 周边
   - 团队可能在场
   - 可选的 workshop

5. **技术主题型更聚焦**
   - 同技术栈的人
   - 问题更容易互相帮
   - 可以说明建议的基础水平
