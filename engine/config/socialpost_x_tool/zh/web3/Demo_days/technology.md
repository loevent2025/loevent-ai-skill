# DEMO DAYS × TECHNICAL / 路演日 × 技术类

## 适用场景
Technical demo days, protocol showcase events, infrastructure project presentations, developer tool demos, hackathon demo sessions, technical product launches

## 场景特征
- **形式**：技术路演、架构讲解、Live Demo、技术 Q&A
- **核心价值**：技术展示、架构反馈、开发者采用、技术认可

## 受众定位
- **Primary Audience:** 技术评审者 / Technical Reviewers - CTO、架构师，评估技术方案
- **Secondary Audience:** 开发者 / Developers - 了解新工具、寻找技术方案
- **Tertiary Audience:** 技术型投资人 / Technical Investors - 评估技术可行性
- **Extended Audience:** 协议团队 / Protocol Teams - 寻找技术合作
- **Audience Behavior:** 评估技术方案、学习新工具、寻找合作、技术交流

### 常用受众组合
- **Builder Focus:** 开发者 + 协议团队，适合工具/基础设施展示
- **Evaluation Focus:** 技术评审者 + 投资人，适合技术深度展示

## 内容元素
- 技术主题定位
- 路演项目技术方向
- 技术评审阵容
- Demo 形式（Live Demo、代码展示）
- 技术 Q&A 环节
- 技术社群入口

## 语言处理
- 技术导向，强调架构和实现
- Live Demo 是核心卖点
- 评审重技术深度
- 代码和数据说话
- 避免过度营销

## 结构策略
- 开场信息块占比 15%
- 技术项目块占比 35%：项目方向 + 技术亮点
- 评审阵容块占比 20%：技术背景
- 参与价值块占比 20%
- 行动号召块占比 10%
- Emoji 密度控制：总量 8-10 个

## 情绪氛围
Technically rigorous, demo-focused. "Show me the code" energy.

## 风格变体

### 变体A：工具展示型 / Tool Showcase
- **适用：** 开发工具、SDK、框架展示
- **开场格式：** 🔧 {主题} Tool Demo Day | 开发者工具集中展示
- **语气：** 实用、开发者导向

### 变体B：协议技术型 / Protocol Technical
- **适用：** 协议技术方案、架构展示
- **开场格式：** ⚡ {主题} Technical Showcase | 看看他们怎么 Build
- **语气：** 深度、架构导向

## 注意事项

### 技术类 Demo Day 要点
1. **Live Demo 是核心**：不是讲 PPT，是看能跑的东西
2. **技术评审阵容**：决定反馈的专业度
3. **技术细节要展示**：架构、代码、数据
4. **开发者视角**：对开发者有什么用
