# 开发者 Cafe Coworking 活动策划知识库

## 场景概述

Cafe Coworking（咖啡馆共工聚会）是开发者社区中以轻松共工、自然社交和产品交流为核心的新型社区活动，参与者自带电脑在咖啡馆环境中一起工作，在轻松氛围中建立连接。以 Cursor 社区的 "Cafe Cursor" 为典型代表，这种形式正在 AI 工具、开发者工具、Web3 等技术社区中快速流行。

### 活动特征

- **规模**：20-100人
- **周期**：半天至全天（3-10小时）
- **形式**：共同工作 + 自由交流 + 项目展示 + 团队互动
- **核心价值**：社区归属感、自然社交、产品反馈、用户连接

## 活动类型与选择

### Cafe Coworking 类型对比

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| 官方 Cafe | 产品团队主办、团队成员出席 | 品牌建设、用户反馈收集 |
| 社区 Cafe | 社区成员自发组织 | 本地社区维护、日常连接 |
| 主题 Cafe | 聚焦特定话题（如AI编程、Web3开发） | 垂直领域交流 |
| 联合 Cafe | 多个社区/产品联合举办 | 跨社区合作、扩大影响 |
| 新手友好 Cafe | 专门欢迎新用户 | 用户增长、降低入门门槛 |

### 与传统Meetup的区别

| 维度 | 传统Meetup | Cafe Coworking |
|------|------------|----------------|
| 核心形式 | 演讲分享为主 | 共同工作为主 |
| 议程结构 | 固定时间表 | 开放时段、自由选择 |
| 参与方式 | 听讲+提问 | 带电脑工作+随机交流 |
| 社交模式 | 安排的社交环节 | 自然发生的对话 |
| 氛围调性 | 偏正式、有主持 | 轻松、咖啡馆感 |
| 时长 | 2-3小时 | 3-9小时（可选时段） |

### 活动频率建议

| 城市活跃度 | 建议频率 | 说明 |
|------------|----------|------|
| 核心城市 | 月度1-2次 | 北上深、纽约、旧金山等 |
| 活跃城市 | 季度1-2次 | 成都、杭州、伦敦、柏林等 |
| 新兴城市 | 半年1次 | 测试市场、建立种子用户 |

## 线上 Cafe Coworking 补充指南

### 线上活动流程调整

| 环节 | 线下版本 | 线上版本 |
|------|----------|----------|
| 场地 | 咖啡馆 | Discord/Gather.town |
| 共工 | 同一空间工作 | 开摄像头静音工作 |
| 交流 | 面对面聊天 | 语音频道/Break Room |
| 展示 | 现场Demo | 屏幕共享 |
| 签到 | 现场签到 | 提前5分钟进入+自我介绍 |

### 线上活动时长调整

- 总时长控制在2-3小时
- 设置"安静工作时段"（45分钟）和"交流时段"（15分钟）交替
- 提供"随时可走"的氛围，不强制全程参与

### 线上活动工具

| 用途 | 推荐工具 |
|------|----------|
| 虚拟空间 | Gather.town、SpatialChat |
| 语音频道 | Discord Stage、Zoom |
| 社区直播 | Twitter Space |
| 报名管理 | Luma、Calendly |

## 场地与氛围设计

### 场地类型对比

| 场地类型 | 特点 | 适用规模 |
|----------|------|----------|
| 独立咖啡馆包场 | 完全控制、氛围好、成本高 | 30-60人 |
| 咖啡馆大区域 | 自然氛围、成本适中、有噪音 | 15-30人 |
| Co-working Space | 设施全、座位充足、缺咖啡馆感 | 40-80人 |
| 书店咖啡区 | 调性好、有文化感、座位有限 | 15-25人 |
| 酒店大堂吧 | 高端感、设施好、价格高 | 20-40人 |

### 场地要求

| 要素 | 要求 |
|------|------|
| WiFi | 稳定高速、能承载人数 |
| 电源 | 充足插座、每桌至少2个 |
| 座位 | 舒适、适合长时间工作 |
| 噪音 | 适度背景音、不影响交流 |
| 光线 | 自然光或柔和灯光 |
| 位置 | 交通便利、好找 |

### 座位布置

| 区域 | 布置方式 | 用途 |
|------|----------|------|
| 主工作区 | 长桌或4-6人圆桌 | 共工、方便交流 |
| 安静区 | 独立小桌、角落位置 | 需要专注的参与者 |
| 交流区 | 沙发、高脚桌 | 站立交流、休息 |
| 展示区 | 靠墙或中心位置 | 项目展示（如有） |

### 氛围营造

- **音乐**：轻松的背景音乐
- **灯光**：温暖、舒适
- **布置**：社区banner、打卡点
- **餐饮**：咖啡、茶饮、小食

## 嘉宾配置方案

### 团队出席配置

| 角色 | 画像 | 数量 |
|------|------|------|
| 社区负责人 | 全程在场、总体协调 | 1人 |
| 产品/技术代表 | 回答用户问题、收集反馈 | 1-2人 |
| 开发者关系 | 技术支持、深度交流 | 1-2人 |
| 流动角色 | 不同时段轮流出席 | 根据情况 |

### 团队互动原则

| 原则 | 说明 |
|------|------|
| 主动但不打扰 | 主动自我介绍，但不打断正在工作的人 |
| 平等姿态 | 以用户身份一起工作，非"官方代表" |
| 真诚交流 | 回答问题要真诚，不知道的承认不知道 |
| 收集反馈 | 自然地了解用户需求和痛点 |

### 主持人要求

| 维度 | 要求 |
|------|------|
| 社区熟悉度 | 了解社区、认识常客 |
| 表达能力 | 能调动氛围、控场 |
| 技术背景 | 能理解产品、引导讨论 |
| 亲和力 | 让参与者感到轻松 |

## 社交环节设计

### 破冰活动

| 活动 | 形式 | 时长 |
|------|------|------|
| 快速自我介绍 | 每人30秒介绍 | 10-15min |
| 找相同点 | 两两交流找共同点 | 10min |
| 问题接龙 | 回答问题后指定下一人 | 10min |
| 技能交换 | 分享自己能教/想学的 | 15min |

### 主题桌讨论

按主题设置讨论桌，参与者自由选择：

| 桌号 | 主题 | 适合人群 |
|------|------|----------|
| 1号桌 | AI应用开发 | AI工具使用者、开发者 |
| 2号桌 | 独立开发者 | Indie Hacker、Side Project |
| 3号桌 | 找合伙人 | 创业者、想组队的人 |
| 4号桌 | 新手入门 | 初学者、想了解的人 |
| 5号桌 | 自由交流 | 什么都聊 |

### 社交促进技巧

- **名牌设计**：包含姓名、角色、关注领域
- **标签贴纸**：技能/兴趣标签便于识别
- **桌签引导**：明确讨论主题
- **主持人引荐**：帮助对接相关的人

## 可选环节设计

### 项目展示（Lightning Demo）

| 要素 | 建议 |
|------|------|
| 时长 | 每人3-5分钟 |
| 数量 | 3-5个展示 |
| 形式 | 站立展示、屏幕投影 |
| 内容 | 展示正在做的项目 |
| 召集 | 提前1小时问谁想展示 |

**展示框架**
- 我在做什么（30秒）
- Demo演示（2分钟）
- 我需要什么帮助/反馈（30秒）

### AMA环节

| 要素 | 建议 |
|------|------|
| 时长 | 20-30分钟 |
| 形式 | 围坐或站立、非正式 |
| 主题 | 产品路线图、功能答疑、使用技巧 |
| 氛围 | 像朋友聊天、不是发布会 |

## 风险预案

| 风险类型 | 风险描述 | 预案 |
|----------|----------|------|
| 人数过少 | 报名人数不足 | 降低预期、调整场地、加强推广 |
| 人数过多 | 超出场地容量 | 候补名单、增加时段、限制报名 |
| WiFi故障 | 网络不稳定 | 提前测试、准备移动热点 |
| 电源不足 | 插座不够用 | 携带排插、提前沟通场地 |
| 场地变卦 | 咖啡馆临时取消 | 备选场地、提前签协议 |
| 冷场风险 | 没人交流 | 主动破冰、安排"气氛组" |
| 团队缺席 | 团队成员临时有事 | 确保至少1人在场 |

## 执行提醒

### 场地要求

- WiFi稳定高速，能承载预期人数
- 电源充足，每桌至少2个插座
- 座位舒适，适合长时间工作
- 有独立交流区域
- 交通便利、好找

### 物料清单

- 签到表/名牌/记号笔
- 排插（至少3个）、备用充电线
- WiFi密码牌、区域指引
- 产品贴纸、小礼品
- 移动热点（备用）

### 服务标准

- 提前1小时到场布置
- 测试WiFi和电源
- 团队成员主动但不打扰
- 活动结束清理场地
- 拍摄现场照片用于回顾
