# Web3 技术工作坊 活动策划知识库

## 场景概述

技术工作坊（Technical Workshop）是Web3行业中以技术学习和实操训练为核心的教育型活动，参与者在导师指导下学习特定技术栈或完成实操项目。是开发者培育、技术推广和生态建设的重要形式。

### 活动特征

- **规模**：20-100人
- **周期**：2-6小时（单场）/ 1-5天（系列）
- **形式**：技术讲解 + 实操练习 + 答疑指导
- **核心价值**：技能学习、实操经验、开发者培育、生态建设

## 线上工作坊补充指南

### 线上活动流程调整

| 环节 | 线下版本 | 线上版本 |
|------|----------|----------|
| 签到 | 现场签到 | 提前15分钟进入、环境自检 |
| 讲解 | 现场演示 | 屏幕共享+实时编码 |
| 实操 | 现场指导 | 分组Breakout Room+助教巡场 |
| 答疑 | 现场提问 | 弹幕/举手/私聊助教 |
| 互动 | 现场走动 | 在线投票/代码分享 |

### 线上活动时长调整

- 单次讲解不超过45分钟
- 每45分钟安排10分钟休息
- 实操环节增加15-20%时间
- 总时长建议不超过4小时

### 线上活动工具推荐

| 用途 | 推荐工具 |
|------|----------|
| 直播/会议 | Zoom（分组功能）、Google Meet |
| 代码协作 | GitHub Codespaces、Replit、CodeSandbox |
| 实时问答 | Slido、Discord |
| 文档分享 | Notion、HackMD |
| 录播回放 | YouTube、Loom |

### 线上活动注意事项

- **环境预检**：提前发送环境配置指南
- **助教配比**：建议每15-20人配1名助教
- **互动频率**：每15分钟设置互动点
- **录播提供**：提供录播回放供复习
- **社群支持**：建立学员群持续答疑

## 讲师与助教配置

### 讲师要求

| 维度 | 要求 |
|------|------|
| 技术能力 | 该领域有实战经验、能解决现场问题 |
| 表达能力 | 讲解清晰、能把复杂概念简单化 |
| 教学经验 | 有授课经验、了解学员痛点 |
| 耐心程度 | 能处理入门问题、不急躁 |

### 讲师画像

| 类型 | 适合场景 |
|------|----------|
| 项目核心开发者 | 官方技术推广、深度技术 |
| DevRel工程师 | 入门教学、生态推广 |
| 资深Builder | 实战经验分享、最佳实践 |
| 安全研究员 | 安全主题、审计培训 |

### 助教配置

| 规模 | 助教数量 | 职责 |
|------|----------|------|
| 20人以下 | 1人 | 环境支持、答疑 |
| 20-50人 | 2-3人 | 分区负责、巡场 |
| 50-100人 | 4-6人 | 分组指导 |

**助教职责**
- 活动前：环境配置文档、预检支持
- 活动中：解决环境问题、辅助答疑、记录问题
- 活动后：整理FAQ、跟进学员问题
