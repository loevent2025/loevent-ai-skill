# Web3 Developer Meetup — Event Planning Knowledge Base

## Overview

Developer Meetups are community-driven events in the Web3 industry centered around technical knowledge sharing, professional networking, and casual interaction. With a relaxed atmosphere and strong interactivity, they are a vital format for sustaining developer community engagement and fostering a sense of belonging.

### Event Characteristics

- **Scale**: 30–150 attendees
- **Duration**: 2–3 hours
- **Format**: Technical talks + Networking + Casual interaction
- **Core Value**: Community cultivation, Professional networking, Knowledge sharing, Sense of belonging

## Online Meetup Supplement

### Online Agenda Adjustments

| Element | Offline Version | Online Version |
|---------|-----------------|----------------|
| Check-In | On-site registration + Networking | Join 5 minutes early + Self-introduction round |
| Talks | Live on-stage presentation | Screen sharing (shortened to 20–25 min) |
| Networking | Free-roaming conversations | Breakout rooms |
| Interaction | Raise hand in person | Chat / Raise-hand feature |
| Raffle | On-site draw | Online raffle tool |

### Online Duration Adjustments

- Keep total duration under 90 minutes
- Cap individual talks at 25 minutes
- Schedule an interactive segment every 30 minutes
- Replace networking with breakout rooms (15–20 minutes)

### Online Platform Selection

| Platform | Features | Use Cases |
|----------|----------|-----------|
| X Spaces (formerly Twitter Spaces) | Web3-native; Public reach | Open discussions, AMAs |
| Discord Stage | Community-native; Interactive | Internal community events |
| Zoom | Full-featured; Breakout room support | Formal talks + Networking |
| Google Meet | Simple and easy to use | Small-scale gatherings |

### Online Event Best Practices

- **Interaction Design**: Add polls, Q&A, and other interactive elements
- **Social Alternative**: Use breakout rooms or post-event Discord voice channels
- **Recording**: Provide a recording for those who could not attend live
- **Time Zones**: Choose a time that works for the majority of attendees

## Speaker & Host Configuration

### Speaker Profiles

| Type | Profile | Best Suited For |
|------|---------|-----------------|
| Builder | Hands-on project experience | Technical talks, Lessons learned |
| Founder | Project founder / Co-founder | Project overviews, Entrepreneurial stories |
| Researcher | Research institution / Analyst | Industry insights, Trend analysis |
| DevRel | Developer Relations lead | Technology promotion, Tool introductions |

### Host Requirements

| Dimension | Requirement |
|-----------|-------------|
| Community Familiarity | Knows the community; Recognizes regulars |
| Communication Skills | Can energize the room and manage flow |
| Technical Background | Understands talk content; Can guide discussion |
| Approachability | Makes attendees feel at ease |

### Host Responsibilities

- Warm up the room; Walk through the agenda
- Introduce speakers; Set up each talk
- Manage time; Facilitate Q&A
- Keep energy up; Address dead moments
- Wrap up; Tease the next event

## Social & Networking Design

### Icebreaker Activities

| Activity | Format | Duration |
|----------|--------|----------|
| Speed Introductions | 30-second intro per person | 10–15 min |
| Find Common Ground | Pair up and discover shared interests | 10 min |
| Question Relay | Answer a question, then nominate the next person | 10 min |
| Skill Swap | Share what you can teach / Want to learn | 15 min |

### Themed Table Discussions

Set up discussion tables by theme; Attendees choose freely:

| Table | Example Theme |
|-------|---------------|
| Table 1 | DeFi Development |
| Table 2 | NFT / GameFi |
| Table 3 | L1/L2 Ecosystems |
| Table 4 | Getting Started |
| Table 5 | Open Chat |

### Tips for Facilitating Networking

- **Name Badges**: Include name, Role, Areas of interest
- **Tag Stickers**: Skill / Interest labels for easy identification
- **Table Signs**: Clearly indicate discussion topics
- **Host Introductions**: Help connect people with shared interests

## Venue & Atmosphere

### Venue Selection

| Venue Type | Features | Suitable Scale |
|------------|----------|----------------|
| Co-Working Space | Great vibe, Full amenities | 30–80 people |
| Café / Bar | Casual, Strong social atmosphere | 20–50 people |
| Tech Company Meeting Room | Formal, Good facilities | 30–100 people |
| Art Space | Unique, Distinctive character | 50–150 people |

### Venue Requirements

| Criteria | Requirement |
|----------|-------------|
| Layout | Encourages interaction; Not too cavernous |
| Sound | Talks are audible; Does not overpower conversations |
| Lighting | Bright but not harsh |
| Network | Stable Wi-Fi |
| Location | Easy to reach by transit; Easy to find |

### Atmosphere Design

- **Music**: Light background music
- **Lighting**: Warm and comfortable
- **Décor**: Community banners, Photo-op spots
- **Catering**: Pizza, Drinks, Snacks

| Content | Format | Publishing Timeline |
|---------|--------|---------------------|

## Risk Contingency Plan

| Risk Type | Description | Contingency |
|-----------|-------------|-------------|
| Speaker Cancellation | Speaker drops out last minute | Backup speaker on standby; Extend networking time |
| Low Turnout | Insufficient registrations | Boost promotion; Adjust expectations |
| Overcrowding | Actual attendance exceeds expectations | Flexible venue capacity; Overflow plan |
| Dead Room | No one networking during social segments | Deploy icebreaker activities; Host-led introductions |
| Equipment Failure | Projector / Sound system malfunction | Backup equipment; Pre-event testing |

- Prepare a check-in QR code / Attendance list

- Track actual attendance data

### Post-Event Review Checklist

- Attendance rate analysis
- Attendee feedback collection
- Talk content effectiveness assessment
- Networking segment observation
- Improvement notes for next event
