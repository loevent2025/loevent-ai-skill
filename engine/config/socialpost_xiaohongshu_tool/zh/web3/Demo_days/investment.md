# WEB3 DEMO DAY × INVESTMENT / Web3项目展示日 × 投资类

## 适用场景
Web3融资路演日、Crypto VC专场Demo Day、加速器毕业路演、天使投资人对接会、赛道专场Pitch Day、战投专场展示

## 场景特征
- **形式**：融资路演 + 产品Live Demo + 投资人点评 + 1v1对接
- **核心价值**：融资对接、投资人反馈、Deal Flow、项目筛选

## 受众定位
- **主要受众**：Crypto投资人/VC - 发现早期项目、评估投资机会、高效看Deal
- **次要受众**：Web3创始人 - 展示项目、获取反馈、寻求融资
- **第三受众**：大厂战投/交易所投资部 - 寻找投资/上币/合作标的
- **扩展受众**：FA/OTC - 获取项目信息、对接资源
- **受众行为**：筛选项目、评估Demo和链上数据、对接创始人、做投资决策

### 常用受众组合
- **机构导向**：Crypto VC + 创始人，适合正式融资路演
- **天使导向**：天使投资人 + 早期创始人，适合Seed/Pre-seed对接

## 内容元素
- Demo Day投资定位（融资阶段 + 赛道方向）
- 路演项目信息（赛道/阶段/链上数据/团队）
- 投资人/评委阵容（Fund + AUM + Portfolio）
- Pitch + Live Demo形式说明
- 1v1对接 / Office Hour机制
- 合规声明（必须 / NFA / DYOR）

## 语言处理
- 投资视角，强调项目质量和链上数据
- 投资人阵容是核心卖点
- Demo完成度、链上Traction和Token设计是重点
- 对接机制要清晰
- 必须包含NFA / DYOR声明
- 不做任何收益承诺

## 结构策略
- 开场信息块占比 15%
- 项目说明块占比 25%：项目质量 + 链上数据亮点
- 投资人阵容块占比 25%：投资人背景 + Portfolio
- 对接机制块占比 20%：怎么对接 + Office Hour
- 合规行动块占比 15%
- Emoji 密度控制：总量 8-12 个，专业克制

## 情绪氛围
专业、高效、Deal导向。"发现下一个Web3独角兽"的期待。

## 风格变体

### 变体A：融资路演型 / Fundraising Pitch
- **适用**：明确融资导向的Demo Day
- **开场格式**：🚀 {Demo Day名} | Web3创始人直面投资人
- **语气**：融资导向、高效对接

### 变体B：投资人专场型 / Investor Exclusive
- **适用**：仅限投资人参加的闭门专场
- **开场格式**：💼 {Demo Day名} | 投资人专属Web3项目展示
- **语气**：筛选导向、质量背书

### 变体C：赛道专场型 / Track Focus
- **适用**：特定赛道项目路演（DeFi/Infra/DePIN/AI×Crypto等）
- **开场格式**：🔥 {赛道}专场 Demo Day | 赛道最新项目一次看完
- **语气**：垂直、专业、深度

## 参会攻略建议

### 对于创始人
- 准备好针对Crypto VC的Pitch版本（强调链上数据和Token设计）
- Demo要稳定，提前多次测试（包括链上交互部分）
- 准备好核心数据：TVL、用户数、链上指标、增长率
- 明确融资需求：轮次、金额、估值、Token结构
- 准备好Deck + One-pager + 链上数据Dashboard

### 对于投资人
- 提前看项目资料和链上数据，带着DD问题来
- 关注Demo的真实完成度和链上Traction
- 现场感受团队的应变能力和技术深度
- 感兴趣的项目及时约Office Hour跟进
- 对比同赛道项目的差异

### 会后跟进
- 24小时内发送跟进TG/邮件
- 承诺的材料（Deck/数据/合约审计报告）及时发送
- 链上数据持续跟踪
- 保持适度跟进，不要过度Push

## 注意事项

### 投资类Demo Day要点
1. **合规声明必须有**：NFA / DYOR / 不构成投资建议 / 对接≠承诺
2. **项目质量要背书**：链上数据、融资阶段、完成度、审计情况
3. **投资人阵容是核心**：要有决策权、活跃出手、代表Portfolio
4. **对接机制清晰**：怎么约1v1、Office Hour规则、跟进流程
5. **Demo是关键**：Web3项目必须现场演示链上交互
6. **注意监管红线**：不涉及Token推荐、价格预测、收益承诺
