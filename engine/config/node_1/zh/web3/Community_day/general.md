# Web3 开发者社区咖啡日知识库

## 一、场景定义与特点

### 场景概述
开发者社区咖啡日（Developer Cafe Day / Community Cafe）是 Web3 行业中以咖啡馆为载体的轻量级社区活动形式，通过共创（Co-work / BUIDL）、自由社交、协议答疑等方式，营造轻松氛围，拉近协议团队与开发者之间的距离，是社区培育、用户留存和开发者关系深化的重要场景。尤其适合在黑客松间隙、主网上线后、新协议冷启动阶段等关键节点落地。

### 核心特征
| 特征 | 描述 |
|------|------|
| 规模 | 30-100人，小而精 |
| 周期 | 3-10小时，分时段入场 |
| 形式 | BUIDL共创 + 自由社交 + 协议答疑 + 周边发放 |
| 定位 | 社区培育、开发者连接、协议生态深化 |

### 典型活动形式
- 协议社区咖啡日（Uniswap Builder Day、Solana Hacker Cafe）
- 开发者 BUIDL 日（Builder Cafe、Hacker House Lite）
- 团队开放日（Office Hour Cafe、Meet the Core Devs）
- 城市巡回咖啡聚（City Tour Cafe — ETH Global 同期、Token2049 周边）
- 主网/测试网上线后社区见面会
- DAO 治理线下协作日

## 五、关键成功要素

### 场地选择
- **氛围契合**：有调性的独立咖啡馆或联合办公空间，避免连锁快餐感；优先选择 Web3 友好社区场所
- **容量适中**：预留 20% 余量，确保舒适 BUIDL 体验
- **网络可靠**：稳定 WiFi + 备用热点，开发者刚需
- **电源充足**：多插座，支持长时间工作（Laptop + 硬件钱包 + 手机）
- **位置便利**：靠近主会议场馆（ETHGlobal、Token2049、Devcon 等周边尤佳）

### 活动设计
- **时段灵活**：上午 / 下午 / 晚上分时段，开发者自选入场，避免强制打卡
- **结构松散**：无固定议程，自由交流与 BUIDL 为主；可预留 15-30 分钟闪电分享窗口
- **核心团队在场**：协议工程师、研究员、DevRel 全程参与，主动交流
- **标识清晰**：团队成员佩戴名牌（含 ENS / 链上身份），便于识别
- **工作友好**：允许纯 BUIDL，不强制社交；设置安静工作区与交流区

### 体验打造
- **餐饮免费**：包场或提供餐饮券，降低参与门槛
- **周边精致**：硬件钱包贴纸、协议徽章、限定 NFT POAP，增强仪式感与链上归属感
- **拍照友好**：布置协议品牌元素与 Web3 文化装置，方便打卡分享
- **技术展示区**：演示节点运行、合约交互、链上数据看板，营造 Builder 氛围
- **惊喜环节**：现场 Gas 补贴、测试网代币发放、Grant 快速通道资格、内测白名单

### 传播策略
- **预热期**：官宣推文 + POAP 报名链接 + 限时名额福利（早鸟特权）
- **活动期**：实时推文直播、Builder UGC 引导（#BUIDL #协议名）、核心团队互动展示
- **长尾期**：POAP 发放、照片回顾 Thread、Builder 故事专访、下期预告

## 六、注意事项

### 规模把控
- 首次举办建议 30-50 人，验证模式后再扩大
- 宁可超额报名限制入场，不可现场冷清
- 时段分流避免人流高峰拥挤；黑客松期间注意与其他活动时间错峰

### 团队参与
- 确保协议核心成员（工程师 / 研究员）全程在场，非走过场
- 团队成员主动出击，不等 Builder 来找
- 避免团队成员扎堆聊天，分散在人群中
- **工程师 / 研究员参与比 BD / 市场更有价值**，技术深度对话是核心资产

### 开发者体验
- 不强制社交，尊重独处 / 专注 BUIDL 的开发者
- 不过度打扰，观察状态后再切入交流
- 不推销协议，以支持和倾听为主；警惕变成硬广场所
- 技术问题认真对待，当场能解决的尽量解决；解决不了的当场记录跟进
- 尊重链上隐私习惯，不强制索要真实身份信息

### 反馈收集
- 自然对话中收集，避免正式问卷打扰
- 记录高频技术痛点，同步反馈给协议研究团队
- 识别核心 Builder / 贡献者，后续重点维护与 Grant 跟进
