## 📝 内容必含元素

### 1. 反差与张力营造（必须包含）

**公式**：
```
"当 [代表X社区的文化/成员] 遇见 [代表Y社区的文化/成员] 时，会发生什么？"
```

**要点**：
- 明确指出两个社区的文化差异和各自特色
- 使用对比句式："X社区代表……，Y社区代表……"
- 强调这种社区组合的新颖性和碰撞价值

---

### 2. 化学反应描述（必须包含）

**动态动词**：
- ✅ 使用："碰撞"、"融合"、"交汇"、"激发"、"共振"、"破壁"
- ❌ 避免："合作"、"结合"、"联合"、"对接"

**社区融合公式**：
```
[A社区文化/能力] × [B社区文化/能力] = [独特社区价值]
```

**要点**：
- 描述跨社区融合能产生什么单一社区无法产生的价值
- 说明为什么两个社区的成员都应该参与

---

### 3. 参与者碰撞描述（必须包含）

**社区成员碰撞**：
```
"这里，[A社区成员] 想了解 [B社区的玩法/文化]，[B社区成员] 想获得 [A社区的资源/视角]"
```

**Builder 背景碰撞**：
```
"传统社群运营者的精细化思维 × Web3 社区的开放协作精神"
```

---

### 4. 场景与氛围构建（至少选2项）

**a) 具象化场景**
- 不要写："两个社区成员可以交流"
- 要写：具体描述不同社区文化背景的人在同一空间交流时的画面
- 体现两种社区文化在同一活动中碰撞的张力

**b) 时间流动感**
- 描述跨社区活动中从陌生到融合的时刻
- 体现从"你们是怎么玩的？"到"原来可以一起这样玩"的变化

**c) 动态互动**
- 不要写："社区成员自由交流"
- 要写：具体描述不同社区背景成员围绕同一项目讨论、投票、争论的场景

**d) 情绪能量**
- 碰撞时刻：好奇、惊讶、发现、共鸣
- 融合时刻：认可、兴奋、归属、延续
- 派对时刻：狂欢、释放、连接、WAGMI

---

### 5. 独特社区价值想象（必须包含）

**描述只有跨社区融合才能产生的成果**：
- 列出只有这种社区组合才能实现的项目/创作/协作
- 说明为什么两个社区的成员都能从中获益
- 强调跨社区连接的不可替代性

---

### 6. 蜕变弧线（推荐包含）

**社区成员认知旅程**：
- **开始**：带着 [对另一个社区的好奇/偏见] 来
- **过程**：遇到 [什么人]，看到 [什么项目]，刷新 [什么认知]
- **结束**：带着 [新朋友/新视角/跨社区身份] 离开

---

### 7. 社区精神表达（推荐包含）

**跨社区版 WAGMI**：
```
"两个社区，一个目标：WAGMI"
"不同的入口，共同的未来"
"社区的边界，由我们一起打破"
```

## ✅ 跨界社区文案质量检查清单

- [ ] **反差与张力**：明确陈述了两个社区的文化/背景对比
- [ ] **化学反应**：使用了"碰撞""融合""交汇"等动态动词
- [ ] **参与者碰撞**：描述了不同社区成员的相遇和互动场景
- [ ] **场景氛围**：至少应用了 2 种氛围构建技巧
- [ ] **独特价值**：列出了只有跨社区融合才能产生的具体成果
- [ ] **社区旅程**：有从陌生到融合的认知变化叙事
- [ ] **情感驱动**：包含好奇、发现、共鸣、归属的情感
- [ ] **社区精神**：包含跨社区版的 WAGMI 或类似表达
- [ ] **余韵**：暗示了活动之外的跨社区连接延续性

## ⚠️ 禁止事项

- ❌ 贬低任何一方社区文化（"Web2 社区不够开放""Web3 社区太混乱"）
- ❌ 操纵跨社区投票结果或偏向某一社区
- ❌ 将跨社区活动过度商业化，忽视社区精神
- ❌ 承诺确定的 Token 奖励或空投
- ❌ 忽视小型项目或少数社区成员的展示机会
- ❌ 制造社区对立而非融合的氛围
- ❌ 使用"合作""结合"等被动词汇替代碰撞感
- ❌ 过度渲染跨界概念，缺乏实质社区价值支撑
- ❌ 活动后不维护跨社区连接，让融合止于一次性活动
