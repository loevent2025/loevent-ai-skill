# HACKATHONS × GENERAL

## Use Cases
General hackathons, multi-track hackathons, ecosystem hackathons, themed coding competitions, innovation challenges, builder events, protocol-sponsored hackathons

## Scene Characteristics
- **Scale:** 50–500 people
- **Duration:** 24–72 hours (typically 48 hours)
- **Format:** Intensive development, team formation, pitching, judging, awards
- **Core Value:** Hands-on capability, project incubation, networking, technical growth

## Audience Targeting
- **Primary Audience:** Developers — developers of all levels looking for a real-world challenge
- **Secondary Audience:** Product & Design — team up with developers to build more complete projects
- **Tertiary Audience:** Entrepreneurs — validate ideas, find technical co-founders
- **Extended Audience:** Students — learn by doing, build experience
- **Audience Behavior:** Push their limits, meet collaborators, validate ideas, learn new tech, demonstrate capabilities

### Common Audience Configurations
- **Builder Core:** Primarily developers — best for purely technical hackathons
- **Full Team:** Developers + product + design — best for comprehensive hackathons

## Content Elements
- Hackathon theme and positioning
- Track structure
- Sponsor and judge lineup
- Timeline (start / end / submission / demo)
- Technical support details (mentors, APIs, resources)
- Team formation mechanism
- Judging criteria

## Language Tone
- Challenge-driven, action-oriented
- Emphasize "hands-on building" over "come and listen"
- Moderate sense of urgency
- Frame technical challenges as questions to solve
- Encouraging without applying pressure
- Generalized phrasing for prizes (when specifics are unavailable)

## Content Structure
- Opening info block: 15% — name + time/location + scale
- Challenge overview block: 30% — theme + tracks + technical challenges
- Resource support block: 25% — mentors + technical resources + sponsors
- Participation details block: 20% — who it's for + how to join + team formation
- Call-to-action block: 10% — registration + deadline
- Emoji density: 10–14 total

## Emotional Tone
Energetic, challenging, and builder-focused. "Prove yourself with code" energy. Pro-action, pro-building, anti-talk.

## Style Variants

### Variant A: Multi-track Challenge
- **Best for:** Multi-track comprehensive hackathons
- **Opener format:** 🚀 {Hackathon Name} | {Duration} intensive challenge
- **Tone:** Motivating, comprehensive, open

### Variant B: Theme Focus
- **Best for:** Single-theme hackathons (DeFi, AI, Gaming, etc.)
- **Opener format:** 🔧 {Theme} Hackathon | Answer this question with code
- **Tone:** Focused, challenging, deep

### Variant C: Ecosystem Building
- **Best for:** Protocol or L1-sponsored ecosystem hackathons
- **Opener format:** ⚡ Build on {Protocol Name} | {Hackathon Name}
- **Tone:** Ecosystem-oriented, resource-rich

## Notes

### Hackathon Copywriting Principles
1. **Make tracks clear:** Participants should be able to find their fit immediately
2. **Be specific about technical support:** Mentors, APIs, documentation
3. **State the timeline explicitly:** Start / end / submission deadline
4. **Clarify team formation:** Can people register solo and team up on-site?
5. **Handle prize information carefully:** If details aren't confirmed, use general phrasing or omit

### Accuracy Principles
- Only list sponsors once confirmed
- Use "partial list" or "to be updated" for mentor lineups
- Verify prize information before publishing; generalize or omit if unconfirmed
