# Web3 Industry — Hackathon Marketing Strategy

## Event Definition

A hackathon is a competitive technical event designed for developers, where participants form teams and build innovative projects within a defined timeframe. Web3 hackathons typically center on a specific protocol, public chain, or technical theme, with the goals of surfacing high-quality projects, nurturing developer ecosystems, and driving technical innovation.

## Core Objectives

- Attract high-caliber developers to participate
- Discover innovative projects and teams
- Build and grow the developer community
- Demonstrate the strength of the technical ecosystem
- Cultivate potential ecosystem projects for future development

| Channel | Priority | Role |

## Communications Strategy

### Hashtag Strategy
- Primary: #[EventName]Hackathon #BUIDLWeek
- Technical: #Web3Dev #BlockchainDevelopment
- Track-specific: #DeFi #NFT #Infra, etc.

### Content Strategy

| Phase | Content Types | Publishing Frequency |
|-------|---------------|----------------------|
| Recruitment | Event overview, track descriptions, judge announcements | 1–2 times daily |
| Registration | Technical resources, mentor introductions, FAQ | Every 2 days |
| Competition | Progress updates, standout project spotlights | Daily updates |
| Judging | Judging updates, shortlist announcements | Key milestones |
| Results | Winner announcements, project showcases | Concentrated release |

### Discord Operations Focus
- Establish a clear and well-organized channel structure
- Set up participant-exclusive roles
- Keep the technical Q&A channel active and responsive
- Maintain a dedicated team-matching channel
- Post real-time progress updates throughout the event

## Thought Leader Partnership Strategy

### Thought Leader Profiles

| Type | Profile | Engagement Format |
|------|---------|-------------------|
| Technical Authorities | Notable protocol developers, open-source contributors | Judges / Mentors |
| Ecosystem Developers | Active builders, developer advocates | Technical talks / Judges |
| Project Founders | Well-known DApp founders | Experience sharing, endorsements |
| Developer Community Leaders | Leads of DevDAOs and similar communities | Community mobilization |

### Engagement Formats
- Judges / Mentors: provide technical authority and credibility
- Workshops: deliver technical training and knowledge sharing
- Community Mobilization: developer recruitment and outreach
- Project Incubation: connect standout projects with follow-on support

## Distribution Timeline

### T-4 Weeks: Recruitment Launch
- Publish hackathon announcement
- Open Discord and registration channels
- Announce tracks and judging criteria
- Begin outreach across developer communities

### T-3 Weeks: Deep Warm-Up Phase
- Announce judges and mentor lineup
- Release technical documentation and resources
- Host a pre-event technical warm-up workshop
- Activate team-matching channel

### T-2 Weeks: Registration Push
- Launch registration deadline countdown
- Publish consolidated FAQ
- Showcase standout projects from past editions
- Final recruitment push

### T-1 Week: Pre-Competition Preparation
- Confirm registered participants
- Intensive technical Q&A support
- Publish development environment setup guide
- Re-communicate judging criteria

### Competition Phase: Live Operations
- Post daily progress updates
- Provide ongoing technical Q&A support
- Spotlight notable team progress
- Encourage participants to share updates publicly

### Judging + Results Phase
- Maintain transparency throughout the judging process
- Announce shortlisted projects
- Feature winning projects with in-depth showcases
- Publish post-event incubation and support plans

## Developer Community Outreach

### Target Communities

| Community Type | Outreach Method | Content Focus |
|----------------|-----------------|---------------|
| Technical Discord Servers | Event posting | Technical challenges, learning opportunities |
| Developer DAOs | Community partnership | Ecosystem opportunities, project incubation |
| University Tech Clubs | Targeted invitations | Skill development, industry entry point |
| Open-Source Communities | Technical angle | Tech stack, creative freedom |

### Recruitment Approach
- Emphasize the appeal of the technical challenges
- Highlight the mentor lineup and learning opportunities
- Showcase post-event incubation support and resources
- Feature success stories from previous editions

## Technical Support Framework

### Workshop Schedule
- Pre-event: tech stack onboarding, tooling walkthroughs
- During event: technical Q&A office hours
- Post-event: project refinement and optimization guidance

### Resource Support
- Comprehensive technical documentation
- API and SDK support
- Testnet access and resources
- Recommended development tooling
