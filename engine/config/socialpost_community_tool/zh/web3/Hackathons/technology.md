# Web3 Hackathons - Technology Activities Knowledge Base

## 📌 Module Activation Conditions

Apply this module's rules when the event theme involves the following keywords:
- Protocol development hackathon, smart contract programming marathon
- ZK/Layer2/Cross-chain technology track, infrastructure development competition
- Security audit challenge, development tool innovation competition

---

## 🎯 Scenario Features Overview

| Feature Dimension | Specific Description |
|----------|----------|
| Format | Pre-competition training + team development + mentor guidance + Demo presentation + judging and awards |
| Duration | 24-72 hours intensive development, or 1-4 weeks online competition |
| Participants | Developers, protocol teams, technical mentors, judging experts |
| Core Goals | Technical innovation verification, prototype development, talent discovery, ecosystem building |

---

## 👥 Audience Profile (by Priority)

| Priority | Audience Type | Focus Areas |
|--------|----------|--------|
| P0 | Full-stack/Contract Developers | Technical challenges, prize pool, project incubation |
| P1 | Protocol Researchers | Technical exploration, innovation verification, academic value |
| P2 | Tech Entrepreneurs | MVP development, team building, funding opportunities |
| P3 | Student Developers | Learning practice, resume enhancement, industry entry |

---

## 🔀 Style Selection Guide

| Event Type | Recommended Style | Core Emphasis |
|----------|----------|----------|
| Official Protocol Hackathon | Formal Professional (A) | Technical depth, official endorsement, ecosystem value |
| Community Developer Competition | Enthusiastic & Vibrant (B) | Participatory atmosphere, learning & growth, community connection |
| Technical Challenge | Suspense-Driven (C) | Technical difficulties, innovative directions, breakthrough opportunities |
| High Prize/Incubation Competition | Benefit-Oriented (D) | Prize pool, Grant, incubation resources |

---

## 📝 Essential Content Elements

### 1. Event Value (at least 2 points)
- "Total prize pool [Amount], [Number of tracks] major technical tracks"
- "Outstanding projects receive official Grant from [Protocol/Organization]"

### 2. Technical Support (at least 1 point)
- "[Number]+ technical mentors provide full guidance"
- "Official provision of [SDK/API/Documentation] technical support"

### 3. Incubation Channel (at least 1 point)
- "Winning projects directly enter [Incubator/Accelerator]"
- "Receive priority matching opportunities with [Investment Institution]"

### 4. Compliance Notice (must include)
- "⚠️ This event is a technical development competition and does not involve investment advice (NFA)"

## ❓ Frequently Asked Questions Template

| Question | Answer |
|------|------|
| Can I participate alone? | Yes! Individuals or teams (≤5 people) are welcome. |
| Do I have to use a specific tech stack? | Relevant track technologies must be used; refer to event rules for specifics. |
| Can existing projects participate? | Significant new development must occur during the event. |
| How will prizes be awarded? | Prizes will be awarded within 30 days after judging, supporting USDT/USDC. |

## ⚠️ Prohibited Items

- ❌ Exaggerating prize money or Grant promises
- ❌ Implying Token issuance or investment returns
- ❌ Plagiarism or reusing existing project code
- ❌ False advertising of incubation/funding resources
